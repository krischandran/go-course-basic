Логические операторы

Объединяем несколько условий с помощью И, ИЛИ, НЕ.

ЛОГИЧЕСКИЕ ОПЕРАТОРЫ НА САМОМ ДЕЛЕ

В прошлом уроке мы научились проверять одно условие. Но часто нужно проверить несколько условий одновременно:

  • Число больше 5 И меньше 10
  • Пользователь администратор ИЛИ модератор
  • Пароль НЕ пустой

Для объединения условий в Go есть три логических оператора:

  • && — логическое И (AND)
  • || — логическое ИЛИ (OR)
  • ! — логическое НЕ (NOT)

Логические операторы работают с типом bool и возвращают bool.

ЛОГИЧЕСКОЕ И (&&)

Оператор && возвращает true только когда оба условия истинны:

ABA && B
truetruetrue
truefalsefalse
falsetruefalse
falsefalsefalse
Go 
package main

import "fmt"

func main() {
    var x, y int
    fmt.Scan(&x, &y)

    if x > 5 && y < 10 {
        fmt.Print("YES")
    } else {
        fmt.Print("NO")
    }
}

ЛОГИЧЕСКОЕ ИЛИ (||)

Оператор || возвращает true когда хотя бы одно условие истинно:

ABA || B
truetruetrue

Go 
package main

import "fmt"

func main() {
    var x, y int
    fmt.Scan(&x, &y)

    if x > 5 || y < 10 {
        fmt.Print("YES")
    } else {
        fmt.Print("NO")
    }
}
✓

Мнемоника для ||: чтобы результат был false, нужно чтобы оба условия были false. Как сложение: 0 + 0 = 0, но 1 + 0 = 1.

ЛОГИЧЕСКОЕ НЕ (!)

Оператор ! инвертирует значение — меняет true на false и наоборот:

A!A
truefalse
falsetrue
Go 
package main

import "fmt"

func main() {
    a := true
    b := false

    fmt.Println(!a)  // false
    fmt.Println(!b)  // true
}

Оператор ! — унарный, он применяется к одному операнду и ставится перед ним.

ПРИОРИТЕТ ОПЕРАТОРОВ

Логические операторы имеют разный приоритет выполнения:

  1. ! — выполняется первым
  2. && — выполняется вторым
  3. || — выполняется последним
Go 
// Эти записи эквивалентны:
a || b && c
a || (b && c)  // && выполняется раньше ||

// Эти записи эквивалентны:
!a && b
(!a) && b  // ! выполняется раньше &&

Для явного указания порядка используй скобки ():

Go 
// Инвертировать результат всего выражения
!(a && b)  // сначала a && b, потом инверсия

// Без скобок инвертируется только a
!a && b    // сначала !a, потом && b
!

Если сомневаешься в приоритете — используй скобки. Они делают код понятнее.

ЛОВУШКИ

Ловушка: && vs ||

Частая ошибка — путаница при проверке диапазона:

Go 
// ❌ НЕПРАВИЛЬНО: любое число подходит!
if x > 1 || x < 100 {
    fmt.Println("В диапазоне")
}

Это условие всегда истинно! Любое число либо больше 1, либо меньше 100 (а обычно — и то, и другое).

Go 
// ✅ ПРАВИЛЬНО: число в диапазоне от 1 до 100
if x > 1 && x < 100 {
    fmt.Println("В диапазоне")
}

Правило: для проверки диапазона используй &&.

Ловушка: неполное условие

Go 
// ❌ ОШИБКА: не скомпилируется
if x >= 7 && <= 9 {
    fmt.Println("OK")
}

// ✅ ПРАВИЛЬНО: оба условия полные
if x >= 7 && x <= 9 {
    fmt.Println("OK")
}

Каждое условие должно быть полным выражением с переменной и оператором.

Объединение нескольких условий

Можно объединять произвольное количество условий:

Go 
if x > 5 && y < 10 && x*y > 15 {
    fmt.Println("Все три условия истинны")
}

if x > 5 || y < 10 || x*y > 15 {
    fmt.Println("Хотя бы одно условие истинно")
}

Сохранение результата в переменную

Логические выражения можно сохранять в переменные типа bool:

Go 
package main

import "fmt"

func main() {
    var age int
    fmt.Scan(&age)

    fromAge := age >= 21
    toAge := age <= 30
    isDiscount := fromAge && toAge

    if isDiscount {
        fmt.Println("Скидка равна 30%")
    } else {
        fmt.Println("Скидка равна 5%")
    }
}

При вводе 27:

  • fromAge = 27 >= 21 = true
  • toAge = 27 <= 30 = true
  • isDiscount = true && true = true

Результат: "Скидка равна 30%"

03 Что выведет?
Go
package main

import "fmt"

func main() {
    var x1, x2 int
    x1 = 3
    x2 = 10

    if x1 < x2 {
        fmt.Print(1)
    }
    if x1 > x2 {
        fmt.Print(2)
    }
    if x1 == x2 {
        fmt.Print(3)
    }
    if x1 != x2 {
        fmt.Print(4)
    }
}
Показать решение

Вывод: 14

Почему:

  1. x1 < x2 → 3 < 10 → true → выводим 1
  2. x1 > x2 → 3 > 10 → false → пропускаем
  3. x1 == x2 → 3 == 10 → false → пропускаем
  4. x1 != x2 → 3 != 10 → true → выводим 4
04 Трёхзначное число

Напиши условие, которое проверяет, является ли число n трёхзначным (от 100 до 999 включительно).

Показать решение
Go
if n >= 100 && n <= 999 {
    fmt.Println("YES")
} else {
    fmt.Println("NO")
}

Почему:

  • Трёхзначные числа начинаются с 100
  • Максимальное трёхзначное число — 999
  • Оба условия должны выполняться одновременно → используем &&

РЕЗЮМЕ

Key Ключевые идеи
  • && — истинно когда оба условия истинны
  • || — истинно когда хотя бы одно условие истинно
  • ! — инвертирует значение (true ↔ false)
  • Приоритет: ! → && → ||

Главная ловушка: для проверки диапазона нужен &&, а не ||

Правило: && = "И то, И другое", || = "ИЛИ то, ИЛИ другое"
