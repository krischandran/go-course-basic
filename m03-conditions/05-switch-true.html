switch без выражения

Проверяем диапазоны и сложные условия с помощью switch.

ПРОБЛЕМА: ДИАПАЗОНЫ В SWITCH

В прошлом уроке мы узнали, что обычный switch работает только с конкретными значениями:

Go
// Работает
switch month {
case 1, 2, 3:
    fmt.Println("Первый квартал")
}

// Не работает
switch score {
case score >= 90:  // Ошибка!
    fmt.Println("Отлично")
}

Но иногда нужно проверить диапазоны или сложные условия. Для этого в Go есть switch без выражения.

SWITCH БЕЗ ВЫРАЖЕНИЯ (SWITCH TRUE)

Если не указать выражение после switch, он работает как цепочка if-else if:

Go
package main

import "fmt"

func main() {
    var score int
    fmt.Scan(&score)

    switch {  // <-- нет выражения после switch
    case score >= 90:
        fmt.Println("Отлично")
    case score >= 75:
        fmt.Println("Хорошо")
    case score >= 60:
        fmt.Println("Удовлетворительно")
    default:
        fmt.Println("Неудовлетворительно")
    }
}

Как это работает:

  1. Каждый case содержит логическое выражение
  2. Проверяются по порядку сверху вниз
  3. Первый case с результатом true выполняется
  4. Остальные case пропускаются

Это эквивалентно записи switch true { ... } — каждый case сравнивается с true.

СРАВНЕНИЕ С IF-ELSE

switch без выражения — альтернативный синтаксис для цепочки if-else if:

Go
// С помощью if-else
if score >= 90 {
    fmt.Println("Отлично")
} else if score >= 75 {
    fmt.Println("Хорошо")
} else if score >= 60 {
    fmt.Println("Удовлетворительно")
} else {
    fmt.Println("Неудовлетворительно")
}

// С помощью switch без выражения
switch {
case score >= 90:
    fmt.Println("Отлично")
case score >= 75:
    fmt.Println("Хорошо")
case score >= 60:
    fmt.Println("Удовлетворительно")
default:
    fmt.Println("Неудовлетворительно")
}

Оба варианта делают одно и то же. Выбирай тот, который кажется более читаемым.

Пример: возрастные группы

Go
package main

import "fmt"

func main() {
    var age int
    fmt.Scan(&age)

    switch {
    case age <= 13:
        fmt.Println("детство")
    case age <= 24:
        fmt.Println("молодость")
    case age <= 59:
        fmt.Println("зрелость")
    default:
        fmt.Println("старость")
    }
}

При вводе 27:

  1. 27 <= 13? Нет
  2. 27 <= 24? Нет
  3. 27 <= 59? Да -> выводим "зрелость"

ЛОВУШКА: ПОРЯДОК CASE ВАЖЕН!

Как и в if-else if, порядок проверок имеет значение:

Go
// НЕПРАВИЛЬНО: первое условие всегда истинно для score >= 60
switch {
case score >= 60:
    fmt.Println("Удовлетворительно")  // score = 95 попадёт сюда!
case score >= 75:
    fmt.Println("Хорошо")  // недостижимо для score >= 75
case score >= 90:
    fmt.Println("Отлично")  // недостижимо для score >= 90
}
Go
// ПРАВИЛЬНО: от строгого к менее строгому
switch {
case score >= 90:
    fmt.Println("Отлично")
case score >= 75:
    fmt.Println("Хорошо")
case score >= 60:
    fmt.Println("Удовлетворительно")
default:
    fmt.Println("Неудовлетворительно")
}
!

Располагай условия от строгих к менее строгим — сначала >= 90, потом >= 75, потом >= 60.

Когда использовать switch без выражения

Используй switch без выражения когда:

  • Проверяешь диапазоны
  • Условия сложные (несколько переменных)
  • Много веток (4+) — код читается легче

Используй обычный switch когда:

  • Сравниваешь с конкретными значениями
  • Значения можно перечислить

Используй if-else когда:

  • 2-3 простых условия
  • Условия очень разные по структуре

ПРОВЕРЬ СЕБЯ

Вопросы

01Чем switch без выражения отличается от обычного switch?
Показать ответ

В обычном switch сравнивается значение переменной с конкретными значениями в case.

В switch без выражения каждый case содержит логическое выражение, которое проверяется на истинность.

02Что означает switch { ?
Показать ответ

Это switch без выражения (эквивалентно switch true). Каждый case проверяется как логическое условие, первый истинный выполняется.

03Почему важен порядок case в switch без выражения?
Показать ответ

Потому что проверки идут сверху вниз, и первый истинный case выполняется. Если менее строгое условие стоит выше, более строгие никогда не сработают.

Задачи

01Оценка по баллам

Напиши программу с switch без выражения: 90-100 -> "5", 80-89 -> "4", 70-79 -> "3", 60-69 -> "2", 0-59 -> "1"

Показать решение
Go
package main

import "fmt"

func main() {
    var score int
    fmt.Scan(&score)

    switch {
    case score >= 90:
        fmt.Println("5")
    case score >= 80:
        fmt.Println("4")
    case score >= 70:
        fmt.Println("3")
    case score >= 60:
        fmt.Println("2")
    default:
        fmt.Println("1")
    }
}
02Что выведет при вводе 85?
Go
package main

import "fmt"

func main() {
    var score int
    fmt.Scan(&score)

    switch {
    case score >= 60:
        fmt.Println("Зачёт")
    case score >= 90:
        fmt.Println("Отлично")
    default:
        fmt.Println("Незачёт")
    }
}
Показать решение

Вывод: Зачёт

Почему:

  1. score = 85
  2. Проверяем 85 >= 60? Да -> выводим "Зачёт"
  3. Остальные case не проверяются

Проблема: условие score >= 90 никогда не выполнится, потому что любое число >= 90 также >= 60, и первый case сработает раньше.

03Исправь ошибку

Программа должна выводить "Отлично" для оценок 90+, "Хорошо" для 70-89, "Плохо" для остальных. Исправь порядок:

Go
switch {
case score >= 70:
    fmt.Println("Хорошо")
case score >= 90:
    fmt.Println("Отлично")
default:
    fmt.Println("Плохо")
}
Показать решение
Go
switch {
case score >= 90:
    fmt.Println("Отлично")
case score >= 70:
    fmt.Println("Хорошо")
default:
    fmt.Println("Плохо")
}

Порядок: сначала проверяем >= 90, потом >= 70.

04Знак числа

Используя switch без выражения, напиши программу, которая выводит: 1 если число положительное, -1 если отрицательное, 0 если равно нулю.

Показать решение
Go
package main

import "fmt"

func main() {
    var x int
    fmt.Scan(&x)

    switch {
    case x > 0:
        fmt.Println(1)
    case x < 0:
        fmt.Println(-1)
    default:
        fmt.Println(0)
    }
}
