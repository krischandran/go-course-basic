fallthrough

Редкое, но полезное ключевое слово для "проваливания" в следующий case без проверки условия.

Поведение switch по умолчанию

В Go после выполнения блока case управление автоматически выходит из switch. Это отличается от некоторых других языков (C, Java), где нужно явно писать break.

Go 
package main

import "fmt"

func main() {
    day := 2

    switch day {
    case 1:
        fmt.Println("Понедельник")
    case 2:
        fmt.Println("Вторник")  // Выполнится только это
    case 3:
        fmt.Println("Среда")
    }
}
Вывод
Вторник

После вывода "Вторник" switch завершается. Блок "Среда" не выполняется.

Ключевое слово fallthrough

Иногда нужно, чтобы после выполнения одного case управление перешло в следующий. Для этого используется fallthrough:

Go 
package main

import "fmt"

func main() {
    day := 2

    switch day {
    case 1:
        fmt.Println("Понедельник")
    case 2:
        fmt.Println("Вторник")
        fallthrough  // <-- перейти к следующему case
    case 3:
        fmt.Println("Среда")
    }
}
Вывод
Вторник
Среда

fallthrough заставляет выполнить следующий блок case, независимо от его условия.

Особенности fallthrough

1. fallthrough безусловный

fallthrough выполняет следующий блок без проверки условия:

Go 
package main

import "fmt"

func main() {
    x := 5

    switch x {
    case 5:
        fmt.Println("Пять")
        fallthrough
    case 10:
        fmt.Println("Десять")  // Выполнится, хотя x != 10
    case 15:
        fmt.Println("Пятнадцать")
    }
}
Вывод
Пять
Десять

Блок case 10 выполнился, хотя x != 10. fallthrough не проверяет условие следующего case.

2. fallthrough только в конце блока

fallthrough должен быть последней инструкцией в блоке case:

Go 
// ✅ Правильно
case 1:
    fmt.Println("Один")
    fallthrough

// ❌ Ошибка компиляции
case 1:
    fallthrough
    fmt.Println("Один")  // Код после fallthrough

3. fallthrough не работает в последнем case/default

Go 
// ❌ Ошибка: некуда "проваливаться"
switch x {
case 1:
    fmt.Println("Один")
default:
    fmt.Println("Другое")
    fallthrough  // Ошибка!
}

Когда использовать fallthrough?

fallthrough используется редко. В большинстве случаев можно обойтись объединением case:

Go 
// Вместо fallthrough — объединение case
switch day {
case "Суббота", "Воскресенье":
    fmt.Println("Выходной")
}

Используй fallthrough когда:

  • Нужно выполнить разный код в нескольких case, но с общим продолжением
  • Логика требует "накопительного" поведения

Не используй fallthrough когда:

  • Нужно просто объединить несколько значений — используй запятую
  • Можно обойтись if-else
✓
Tip

Если в коде много fallthrough, вероятно, стоит переписать логику с помощью if-else или функций.

Проверь себя

Вопросы

01 Что делает fallthrough?
Показать ответ

fallthrough заставляет выполнить следующий блок case после текущего, без проверки его условия.

02 Нужно ли писать break в конце case в Go?
Показать ответ

Нет. В Go switch автоматически выходит после выполнения блока case. break не нужен (он есть, но используется для других целей — выход из циклов).

03 Проверяется ли условие следующего case при fallthrough?
Показать ответ

Нет. fallthrough выполняет следующий блок безусловно, независимо от того, подходит ли значение под case.

Задачи

01 Что выведет код?
Go 
package main

import "fmt"

func main() {
    n := 1

    switch n {
    case 1:
        fmt.Print("A")
        fallthrough
    case 2:
        fmt.Print("B")
        fallthrough
    case 3:
        fmt.Print("C")
    }
}
Показать решение

Вывод: ABC

Почему:

  1. n = 1, срабатывает case 1
  2. Выводим A, затем fallthrough
  3. Переходим в case 2, выводим B, затем fallthrough
  4. Переходим в case 3, выводим C
  5. switch завершается
02 Что выведет код?
Go 
package main

import "fmt"

func main() {
    x := 10

    switch x {
    case 5:
        fmt.Print("5")
        fallthrough
    case 10:
        fmt.Print("10")
    case 15:
        fmt.Print("15")
    }
}
Показать решение

Вывод: 10

Почему:

  1. x = 10, case 5 не подходит
  2. case 10 подходит, выводим 10
  3. Нет fallthrough, switch завершается
03 Что выведет код? (накопительный доступ)
Go 
package main

import "fmt"

func main() {
    level := 1

    switch level {
    case 1:
        fmt.Println("Доступ к базе")
        fallthrough
    case 2:
        fmt.Println("Доступ к API")
        fallthrough
    case 3:
        fmt.Println("Полный доступ")
    }
}
Показать решение

Вывод:

Доступ к базе
Доступ к API
Полный доступ

Почему: Это пример "накопительного" поведения — уровень 1 получает все три уровня доступа благодаря цепочке fallthrough.

Резюме

Key Ключевые идеи
  • В Go switch автоматически выходит после case — break не нужен
  • fallthrough заставляет выполнить следующий блок без проверки условия
  • fallthrough должен быть последней инструкцией в блоке
  • Используется редко — чаще можно объединить case через запятую

Правило: Избегай fallthrough, если можно объединить case через запятую
