fallthrough

Редкий инструмент для проваливания в следующий case.

ПОВЕДЕНИЕ SWITCH ПО УМОЛЧАНИЮ

В Go switch автоматически выходит после выполнения первого подходящего case:

Go
package main

import "fmt"

func main() {
    n := 1

    switch n {
    case 1:
        fmt.Println("один")  // выполнится
    case 2:
        fmt.Println("два")   // НЕ выполнится
    case 3:
        fmt.Println("три")   // НЕ выполнится
    }
}

Вывод: один

Это отличается от C/C++/Java, где нужно явно писать break, чтобы остановить выполнение.

КЛЮЧЕВОЕ СЛОВО FALLTHROUGH

Иногда нужно, чтобы выполнился и следующий case. Для этого используется fallthrough:

Go
package main

import "fmt"

func main() {
    n := 1

    switch n {
    case 1:
        fmt.Println("один")
        fallthrough  // <- переходим к следующему case
    case 2:
        fmt.Println("два")
        fallthrough  // <- и к следующему
    case 3:
        fmt.Println("три")
    }
}

Вывод:

один
два
три

Как это работает:

  1. n = 1 -> совпадает с case 1
  2. Выводим "один"
  3. fallthrough -> переходим к case 2 без проверки условия
  4. Выводим "два"
  5. fallthrough -> переходим к case 3
  6. Выводим "три"
  7. Нет fallthrough -> выход из switch
!

fallthrough переходит к следующему case безусловно — условие следующего case не проверяется!

ЛОВУШКА: БЕЗУСЛОВНЫЙ ПЕРЕХОД

Go
package main

import "fmt"

func main() {
    n := 1

    switch n {
    case 1:
        fmt.Println("один")
        fallthrough
    case 2:
        fmt.Println("два")  // выполнится, хотя n != 2!
    case 3:
        fmt.Println("три")  // НЕ выполнится
    }
}

Вывод:

один
два

case 2 выполнился, хотя n = 1! Условие case 2 не проверялось — fallthrough просто передал управление.

ПРАВИЛА FALLTHROUGH

  1. fallthrough должен быть последним оператором в case
  2. Нельзя использовать в последнем case (или default)
  3. Условие следующего case не проверяется
Go
// Ошибка компиляции: fallthrough не последний оператор
case 1:
    fallthrough
    fmt.Println("один")  // ОШИБКА!

// Ошибка компиляции: fallthrough в последнем case
case 3:
    fmt.Println("три")
    fallthrough  // ОШИБКА! Некуда проваливаться

КОГДА ИСПОЛЬЗОВАТЬ FALLTHROUGH

Почти никогда.

В большинстве случаев лучше использовать объединение case через запятую:

Go
// ПЛОХО: использование fallthrough
switch day {
case "Понедельник":
    fallthrough
case "Вторник":
    fallthrough
case "Среда":
    fallthrough
case "Четверг":
    fallthrough
case "Пятница":
    fmt.Println("Будний день")
}

// ХОРОШО: объединение case
switch day {
case "Понедельник", "Вторник", "Среда", "Четверг", "Пятница":
    fmt.Println("Будний день")
}

fallthrough может быть полезен в очень редких случаях, когда нужно выполнить код текущего case И перейти к следующему. На практике такие ситуации встречаются крайне редко.

ПРОВЕРЬ СЕБЯ

Вопросы

01Что делает fallthrough?
Показать ответ

fallthrough передаёт управление следующему case без проверки его условия. Выполнение продолжается с первого оператора следующего case.

02Проверяется ли условие следующего case при fallthrough?
Показать ответ

Нет. fallthrough безусловно переходит к следующему case, условие не проверяется.

03Когда стоит использовать fallthrough?
Показать ответ

Почти никогда. В большинстве случаев лучше объединить значения через запятую: case "a", "b", "c":

Задачи

01Что выведет?
Go
package main

import "fmt"

func main() {
    x := 2

    switch x {
    case 1:
        fmt.Println("A")
        fallthrough
    case 2:
        fmt.Println("B")
        fallthrough
    case 3:
        fmt.Println("C")
    case 4:
        fmt.Println("D")
    }
}
Показать решение

Вывод:

B
C

Почему:

  1. x = 2 -> совпадает с case 2
  2. Выводим "B"
  3. fallthrough -> переходим к case 3
  4. Выводим "C"
  5. Нет fallthrough -> выход

case 1 пропущен (не совпал), case 4 пропущен (нет fallthrough после case 3).

02Что выведет?
Go
package main

import "fmt"

func main() {
    x := 1

    switch x {
    case 1:
        fmt.Println("один")
        fallthrough
    case 100:
        fmt.Println("сто")
    }
}
Показать решение

Вывод:

один
сто

Почему: fallthrough переходит к case 100 без проверки условия. Хотя x = 1 != 100, код всё равно выполняется.

03Перепиши без fallthrough
Go
switch month {
case 1:
    fallthrough
case 2:
    fallthrough
case 12:
    fmt.Println("Зима")
}
Показать решение
Go
switch month {
case 1, 2, 12:
    fmt.Println("Зима")
}

Объединение через запятую — чище и понятнее.
