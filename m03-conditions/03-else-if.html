Вложенные условия и else if

Проверяем больше двух взаимоисключающих вариантов.

КОГДА ОДНОГО УСЛОВИЯ МАЛО

Иногда нужно проверить больше двух вариантов. Например, определить знак числа:

  • Положительное (больше нуля)
  • Отрицательное (меньше нуля)
  • Ноль

Три разных случая — одним if-else не обойтись.

НЕСКОЛЬКО IF ПОДРЯД (ПЛОХОЙ СПОСОБ)

Первое, что приходит в голову — написать три отдельных if:

Go
package main

import "fmt"

func main() {
    var n int
    fmt.Scan(&n)

    if n > 0 {
        fmt.Println("Положительное")
    }
    if n < 0 {
        fmt.Println("Отрицательное")
    }
    if n == 0 {
        fmt.Println("Ноль")
    }
}

Это работает, но есть две проблемы:

  1. Лишние проверки. Если число положительное, зачем проверять n < 0 и n == 0? Мы уже знаем ответ.
  2. Неочевидная структура. По коду непонятно, что это три взаимоисключающих случая. Другой программист может подумать, что несколько условий могут выполниться одновременно.

ВЛОЖЕННЫЕ УСЛОВИЯ

Правильный способ — использовать вложенные if-else:

Go
package main

import "fmt"

func main() {
    var n int
    fmt.Scan(&n)

    if n > 0 {
        fmt.Println("Положительное")
    } else {
        if n < 0 {
            fmt.Println("Отрицательное")
        } else {
            fmt.Println("Ноль")
        }
    }
}

Как это работает:

┌─────────┐ │ n > 0? │ └────┬────┘ ┌──────┴──────┐ ДА НЕТ │ │ ▼ ▼ "Положительное" ┌─────────┐ │ n < 0? │ └────┬────┘ ┌──────┴──────┐ ДА НЕТ │ │ ▼ ▼ "Отрицательное" "Ноль"

Преимущества:

  • Нет лишних проверок — как только нашли подходящий вариант, остальные не проверяются
  • Код явно показывает, что это взаимоисключающие случаи
  • Последний else автоматически обрабатывает оставшийся случай

ELSE IF — КОРОТКАЯ ЗАПИСЬ

Вложенные условия быстро становятся нечитаемыми. Представь пять уровней вложенности!

Go предлагает удобную запись else if:

Go
package main

import "fmt"

func main() {
    var n int
    fmt.Scan(&n)

    if n > 0 {
        fmt.Println("Положительное")
    } else if n < 0 {
        fmt.Println("Отрицательное")
    } else {
        fmt.Println("Ноль")
    }
}

Это тот же код, только записанный компактнее. Логика идентична вложенным условиям.

✓

else if — это не новый оператор, а просто else, после которого сразу идёт if.

ПРИМЕРЫ

Пример: оценка по баллам

Переведём баллы (0-100) в пятибалльную оценку:

Go
package main

import "fmt"

func main() {
    var grade int
    fmt.Scan(&grade)

    if grade >= 90 {
        fmt.Println(5)
    } else if grade >= 80 {
        fmt.Println(4)
    } else if grade >= 70 {
        fmt.Println(3)
    } else if grade >= 60 {
        fmt.Println(2)
    } else {
        fmt.Println(1)
    }
}

При вводе 75:

  1. 75 >= 90? Нет → переходим к следующему
  2. 75 >= 80? Нет → переходим к следующему
  3. 75 >= 70? Да → выводим 3, остальные условия не проверяются

Ловушка: неправильный порядок условий

Go
// ❌ НЕПРАВИЛЬНО: порядок условий важен!
if grade >= 60 {
    fmt.Println(2)  // 75 >= 60 — true, выведет 2!
} else if grade >= 70 {
    fmt.Println(3)  // сюда никогда не попадём при grade >= 60
} else if grade >= 80 {
    fmt.Println(4)
} else if grade >= 90 {
    fmt.Println(5)
}

Правило: в цепочке else if располагай условия от более строгих к менее строгим: сначала >= 90, потом >= 80, потом >= 70...

Ловушка: лишнее условие

Начинающие часто добавляют лишнюю проверку в последний else:

Go
// ❌ Лишнее условие
if n > 0 {
    fmt.Println("Положительное")
} else if n < 0 {
    fmt.Println("Отрицательное")
} else if n == 0 {  // <- Лишняя проверка!
    fmt.Println("Ноль")
}

Если мы дошли до последнего else, значит оба предыдущих условия (n > 0 и n < 0) были ложными. Единственный оставшийся вариант — n == 0. Проверять это не нужно!

Go
// ✅ Правильно: последний else без условия
if n > 0 {
    fmt.Println("Положительное")
} else if n < 0 {
    fmt.Println("Отрицательное")
} else {
    fmt.Println("Ноль")  // Автоматически n == 0
}

Пример: максимум из трёх чисел

Go
package main

import "fmt"

func main() {
    var a, b, c int
    fmt.Scan(&a, &b, &c)

    if a > b {
        if a > c {
            fmt.Println(a)
        } else {
            fmt.Println(c)
        }
    } else {
        if b > c {
            fmt.Println(b)
        } else {
            fmt.Println(c)
        }
    }
}

Логика:

  1. Сравниваем a и b
  2. Победителя сравниваем с c
  3. Выводим максимальное

ПРОВЕРЬ СЕБЯ

Вопросы

01 Чем else if отличается от нескольких if подряд?
Показать ответ

else if создаёт взаимоисключающие ветки — как только одно условие выполнилось, остальные не проверяются.

Несколько if подряд — независимые проверки, каждое условие проверяется всегда.

02 Нужно ли писать условие в последнем else?
Показать ответ

Нет. Последний else обрабатывает все оставшиеся случаи. Если мы до него дошли, значит все предыдущие условия были ложными — дополнительная проверка избыточна.

03 Почему важен порядок условий в цепочке else if?
Показать ответ

Потому что проверки идут сверху вниз, и как только одно условие истинно — остальные пропускаются.

Если поставить менее строгое условие первым (например, >= 60 перед >= 90), более строгие условия никогда не выполнятся.

04 Сколько условий может быть в цепочке if-else if-else?
Показать ответ

Любое количество. Можно использовать сколько угодно else if между начальным if и финальным else.

Задачи

01 Что выведет при вводе 5, 3, 7?
Go
package main

import "fmt"

func main() {
    var a, b, c int
    fmt.Scan(&a, &b, &c)

    if a > b {
        if a > c {
            fmt.Println(a)
        } else {
            fmt.Println(c)
        }
    } else {
        if b > c {
            fmt.Println(b)
        } else {
            fmt.Println(c)
        }
    }
}
Показать решение

Вывод: 7

Почему:

  1. a = 5, b = 3, c = 7
  2. a > b? → 5 > 3? → true
  3. Входим в первый блок, проверяем a > c? → 5 > 7? → false
  4. Входим в else, выводим c → 7
02 Принадлежность точки

Точка x принадлежит отрезку [-3, 1] или [5, 9] (включая границы). Напиши условие для проверки.

Показать решение
Go
if (x >= -3 && x <= 1) || (x >= 5 && x <= 9) {
    fmt.Println("YES")
} else {
    fmt.Println("NO")
}

Почему:

  • Для каждого отрезка нужно && (оба условия)
  • Между отрезками нужно || (любой из них)
  • Скобки для явности
03 Весовая категория

По весу боксёра определи категорию:

  • до 60 кг — "Легкий вес"
  • от 60 до 64 кг (не включая 64) — "Первый полусредний вес"
  • от 64 до 69 кг (не включая 69) — "Полусредний вес"
Показать решение
Go
if weight < 60 {
    fmt.Println("Легкий вес")
} else if weight < 64 {
    fmt.Println("Первый полусредний вес")
} else if weight < 69 {
    fmt.Println("Полусредний вес")
}

Почему:

  • Порядок от строгих к менее строгим
  • Если weight < 60 ложно, значит weight >= 60 — проверять это явно не нужно
  • Аналогично для следующих условий
04 Количество равных

Три числа. Выведи:

  • 3 если все равны
  • 2 если два равны
  • 0 если все разные
Показать решение
Go
var a, b, c int
fmt.Scan(&a, &b, &c)

if a == b && b == c {
    fmt.Println(3)
} else if a == b || b == c || a == c {
    fmt.Println(2)
} else {
    fmt.Println(0)
}

Почему:

  • Сначала проверяем самое строгое условие (все три равны)
  • Потом менее строгое (хотя бы два равны)
  • Если ни одно не подошло — все разные

РЕЗЮМЕ

Key Ключевые идеи
  • else if позволяет проверять несколько взаимоисключающих условий
  • Как только одно условие истинно — остальные не проверяются
  • Порядок условий важен: от строгих к менее строгим
  • Последний else ловит все оставшиеся случаи — условие не нужно

Главная ловушка: неправильный порядок условий (менее строгое первым)

Правило: Располагай условия от строгих к менее строгим
