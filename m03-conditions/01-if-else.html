Условный оператор if-else

Учимся создавать развилки в программе — выполнять разный код в зависимости от условий.

IF-ELSE НА САМОМ ДЕЛЕ

До сих пор все наши программы выполнялись последовательно — команда за командой, сверху вниз. Каждая строка кода выполнялась всегда, независимо от входных данных.

Но в реальных программах нужна логика принятия решений. Представь программу покупки хлеба:

Выйти из дома
Дойти до магазина
Купить хлеб ← А если хлеба нет?
Дойти до дома

Команда "Купить хлеб" не может выполниться, если хлеба в магазине нет. Нужен способ проверить условие перед выполнением действия.

Как работает условный оператор

Условный оператор if создаёт развилку в программе:

┌─────────────┐ │ Условие? │ └──────┬──────┘ │ ┌──────┴──────┐ │ │ ДА НЕТ │ │ ▼ ▼ ┌─────────┐ ┌─────────┐ │ Блок if │ │ (ничего │ │ │ │ или │ └─────────┘ │ else) │ └─────────┘

Условие — это выражение, результат которого логический тип (bool). Значение true означает "условие выполнено", false — "условие не выполнено".

ОПЕРАТОР IF

В Go проверка условия выполняется с помощью ключевого слова if ("если"):

Go 
package main

import "fmt"

func main() {
    var x int
    fmt.Scan(&x)

    if x > 10 {
        fmt.Println(x)
    }
}

Как это работает:

  1. Читаем число x с клавиатуры
  2. Проверяем условие x > 10
  3. Если условие истинно (true) — выполняем код в фигурных скобках
  4. Если условие ложно (false) — пропускаем блок

Общий вид:

Go 
if условие {
    // код выполняется только если условие = true
}
!

В Go фигурные скобки {} обязательны, даже если внутри одна строка.

ОПЕРАТОРЫ СРАВНЕНИЯ

Для построения условий используются операторы сравнения. Они всегда возвращают bool — либо true, либо false.

Оператор Описание Пример
> больше x > 5
< меньше x < 5
>= больше или равно x >= 5
<= меньше или равно x <= 5
== равно x == 5
!= не равно x != 5
Go 
package main

import "fmt"

func main() {
    a := 2
    b := 3
    c := 2

    fmt.Println(a > b)   // false
    fmt.Println(a < b)   // true
    fmt.Println(a >= b)  // false
    fmt.Println(a >= c)  // true
    fmt.Println(a == b)  // false
    fmt.Println(a == c)  // true
    fmt.Println(a != b)  // true
}

Ловушка: = vs ==

Go 
// ❌ ОШИБКА: присваивание вместо сравнения
if x = 5 {  // Не скомпилируется!
    fmt.Println("x равно 5")
}

// ✅ Правильно: сравнение
if x == 5 {
    fmt.Println("x равно 5")
}

Запомни:

  • = — оператор присваивания (записать значение в переменную)
  • == — оператор сравнения (проверить равенство)
✓

Путаница = и == — одна из самых частых ошибок начинающих. Go защищает от неё: присваивание внутри if без объявления переменной вызовет ошибку компиляции.

ОПЕРАТОР ELSE

Часто нужно выполнить альтернативное действие, когда условие ложно. Для этого используется else ("иначе"):

Go 
package main

import "fmt"

func main() {
    var x int
    fmt.Scan(&x)

    if x > 0 {
        fmt.Println("Положительное")
    } else {
        fmt.Println("Отрицательное или ноль")
    }
}

Конструкция if-else гарантирует, что ровно один из двух блоков выполнится:

  • Если условие true — выполняется блок if
  • Если условие false — выполняется блок else
┌─────────────┐ │ x > 0? │ └──────┬──────┘ │ ┌──────┴──────┐ │ │ ДА НЕТ │ │ ▼ ▼ "Положительное" "Отрицательное или ноль"
!

else не может существовать без if. Он должен идти сразу после закрывающей скобки if.

ОБЛАСТЬ ВИДИМОСТИ В БЛОКАХ

Переменная, объявленная внутри блока {}, существует только внутри этого блока:

Go 
package main

import "fmt"

func main() {
    var x int
    fmt.Scan(&x)

    if x > 0 {
        a := 5  // a существует только здесь
        fmt.Println(a + 2)  // 7
    } else {
        b := 10  // b существует только здесь
        fmt.Println(b + 1)  // 11
    }

    // fmt.Println(a)  // ❌ Ошибка: a не существует
    // fmt.Println(b)  // ❌ Ошибка: b не существует
}

Почему так устроено: каждый блок {} создаёт свою область видимости. Переменные "рождаются" при входе в блок и "умирают" при выходе из него. Это помогает избежать конфликтов имён и делает код предсказуемым.

КОРОТКОЕ ОБЪЯВЛЕНИЕ В IF

Go позволяет объявить переменную прямо в условии if. Такая переменная существует только внутри if и связанного else:

Go 
package main

import "fmt"

func main() {
    if age := 25; age >= 18 {
        fmt.Println("Совершеннолетний")
    } else {
        fmt.Println("Несовершеннолетний")
    }

    // fmt.Println(age)  // ❌ Ошибка: age не существует
}

Синтаксис: if объявление; условие { }

Это часто используется для проверки результатов операций, которые нужны только внутри условия.

Сравнение строк

Операторы сравнения работают и со строками:

Go 
package main

import "fmt"

func main() {
    var password1, password2 string
    fmt.Scan(&password1)
    fmt.Scan(&password2)

    if password1 == password2 {
        fmt.Println("Пароль принят")
    } else {
        fmt.Println("Пароль не принят")
    }
}
✓

При сравнении строк важен регистр: "Go" и "go" — разные строки.

ПРОВЕРЬ СЕБЯ

Вопросы

01 Чем отличается = от == в Go?
Показать ответ

= — оператор присваивания, записывает значение в переменную: x = 5

== — оператор сравнения, проверяет равенство и возвращает bool: x == 5

02 Можно ли использовать else без if?
Показать ответ

Нет. else всегда должен следовать сразу после закрывающей скобки блока if. Отдельно else использовать нельзя.

03 Переменная объявлена внутри блока if. Можно ли использовать её после if?
Показать ответ

Нет. Переменная, объявленная внутри блока {}, существует только внутри этого блока. После выхода из блока она перестаёт существовать.

04 Что произойдёт при выполнении if x = 5 { ... }?
Показать ответ

Ошибка компиляции. В Go нельзя использовать присваивание (=) как условие. Для сравнения нужен оператор ==.

Задачи

01 Что выведет?
Go 
package main

import "fmt"

func main() {
    var x int
    x = 10

    fmt.Print(1)

    if x != 5 {
        fmt.Print(2)
    }

    fmt.Print(3)
}
Показать решение

Вывод: 123

Почему:

  1. x = 10
  2. Выводим 1
  3. Проверяем x != 5 → 10 != 5 → true
  4. Условие истинно, выводим 2
  5. Выводим 3
02 Что выведет при вводе 5?
Go 
package main

import "fmt"

func main() {
    var x int
    fmt.Scan(&x)

    fmt.Print(1)

    if x != 5 {
        fmt.Print(2)
    }

    fmt.Print(3)
}
Показать решение

Вывод: 13

Почему:

  1. Вводим x = 5
  2. Выводим 1
  3. Проверяем x != 5 → 5 != 5 → false
  4. Условие ложно, блок if пропускается
  5. Выводим 3
03 Что выведет при вводе 917?
Go 
package main

import "fmt"

func main() {
    var x int
    fmt.Scan(&x)

    fmt.Print(1)

    if x%10 <= 7 {
        fmt.Print(2)
    } else {
        fmt.Print(3)
    }

    fmt.Print(4)
}
Показать решение

Вывод: 124

Почему:

  1. Вводим x = 917
  2. Выводим 1
  3. Вычисляем x % 10 → 917 % 10 → 7
  4. Проверяем 7 <= 7 → true
  5. Выводим 2 (блок else пропускается)
  6. Выводим 4
04 Чему равны s и a после выполнения?
Go 
s := 1
a := 5

if s > 1 {
    a = a + s
} else {
    s = s + a*2
}
Показать решение

s = 11, a = 5

Почему:

  1. s = 1, a = 5
  2. Проверяем s > 1 → 1 > 1 → false
  3. Выполняется блок else: s = s + a*2 → s = 1 + 5*2 → s = 11
  4. Переменная a не изменилась, осталась равна 5

РЕЗЮМЕ

Key Ключевые идеи
  • if проверяет условие и выполняет блок кода только если условие истинно
  • else задаёт альтернативный блок, который выполняется при ложном условии
  • Операторы сравнения (>, <, >=, <=, ==, !=) возвращают bool
  • Переменные внутри {} существуют только в этом блоке

Главная ловушка: путаница = (присваивание) и == (сравнение)

Правило: Если нужно проверить равенство — используй два знака равно ==
