Возвращаемые значения

Функции могут не только выполнять действия, но и возвращать результат для дальнейшего использования в коде.

Ограничение функций без возврата

Функция printSum выводит результат, но не отдаёт его:

Go 
func printSum(a, b int) {
    fmt.Println(a + b)
}

func main() {
    printSum(3, 5)  // Выводит 8, но мы не можем использовать это значение
}

Что если нужно сохранить результат в переменную? Или использовать его в вычислениях? Для этого функция должна возвращать значение.

Возврат значения на самом деле

Что такое return

Оператор return делает две вещи:

  1. Завершает выполнение функции
  2. Передаёт значение обратно в место вызова
func add(a, b int) int { | +-- тип возвращаемого значения return a + b | +-- значение, которое вернётся }

Как работает возврат

Go 
func add(a, b int) int {
    return a + b
}

func main() {
    result := add(3, 5)  // result = 8
    fmt.Println(result)
}

Выполнение:

  1. Вызывается add(3, 5)
  2. Внутри функции вычисляется 3 + 5 = 8
  3. return 8 завершает функцию и возвращает 8
  4. Значение 8 присваивается переменной result

Синтаксис функции с возвратом

Go 
func имяФункции(параметры) типВозврата {
    // тело
    return значение
}

Тип возвращаемого значения указывается после круглых скобок с параметрами.

Функция без параметров, но с возвратом

Go 
package main

import "fmt"

func main() {
    msg := getMessage()
    fmt.Println(msg)
}

func getMessage() string {
    return "Привет, Go!"
}
Привет, Go!

Функция с параметрами и возвратом

Go 
package main

import "fmt"

func main() {
    result := multiply(4, 5)
    fmt.Println("Результат:", result)
}

func multiply(a, b int) int {
    return a * b
}
Результат: 20

Типы возвращаемых значений

Функция может возвращать любой тип:

Go 
func getNumber() int {
    return 42
}

func getName() string {
    return "Алексей"
}

func isEven(n int) bool {
    return n%2 == 0
}

func getPrice() float64 {
    return 99.99
}
!

Тип значения после return должен совпадать с типом, указанным в объявлении функции.

Совпадение типов

Три места, где тип должен совпадать

Go 
func getMessage() string {  // 1. Тип в объявлении: string
    return "Привет"         // 2. Тип возвращаемого значения: string
}

func main() {
    msg := getMessage()     // 3. Тип переменной: string (выводится автоматически)
    fmt.Println(msg)
}

Ловушка: несовпадение типов

Go 
func getNumber() int {
    return "42"  // Ошибка! string != int
}
Go 
func getNumber() int {
    return 42
}

func main() {
    var result string = getNumber()  // Ошибка! int != string
}

return завершает функцию

Код после return никогда не выполнится:

Go 
func getValue() int {
    return 10
    fmt.Println("Это не выполнится")  // Недостижимый код
}

Go выдаст предупреждение о недостижимом коде.

Несколько return в функции

Функция может иметь несколько return в разных ветках:

Go 
package main

import "fmt"

func main() {
    fmt.Println(checkSign(5))
    fmt.Println(checkSign(-3))
    fmt.Println(checkSign(0))
}

func checkSign(n int) string {
    if n > 0 {
        return "положительное"
    }
    if n < 0 {
        return "отрицательное"
    }
    return "ноль"
}
положительное
отрицательное
ноль
!

Если функция объявлена с типом возврата, каждый путь выполнения должен заканчиваться return. Иначе — ошибка компиляции.

Использование возвращаемого значения

Сохранить в переменную

Go 
result := add(3, 5)
fmt.Println(result)

Использовать напрямую

Go 
fmt.Println(add(3, 5))  // Сразу выводим результат

Использовать в выражении

Go 
total := add(3, 5) + add(10, 20)
fmt.Println(total)  // 8 + 30 = 38

Игнорировать результат

Можно вызвать функцию и не использовать её результат:

Go 
add(3, 5)  // Результат вычисляется, но теряется

Это законно, но обычно бессмысленно для чистых функций.

return без значения

В функциях, которые ничего не возвращают, return можно использовать для досрочного выхода:

Go 
package main

import "fmt"

func main() {
    printPositive(-5)
    printPositive(10)
}

func printPositive(n int) {
    if n <= 0 {
        return  // Досрочный выход, ничего не возвращаем
    }
    fmt.Println("Число:", n)
}
Число: 10

Для n = -5 функция завершается на return, не доходя до fmt.Println.

Вывод vs Возврат

Важно не путать:

Функция выводит Функция возвращает
fmt.Println(x) внутри return x
Результат виден в консоли Результат можно использовать в коде
Нельзя сохранить в переменную Можно сохранить, передать, обработать
Go 
// Выводит — результат нельзя использовать
func printSum(a, b int) {
    fmt.Println(a + b)
}

// Возвращает — результат можно использовать
func getSum(a, b int) int {
    return a + b
}

func main() {
    printSum(3, 5)          // Просто выводит 8
    result := getSum(3, 5)  // Сохраняем 8 в переменную
    fmt.Println(result * 2) // Используем: 16
}

Обязательность return

Если функция объявлена с типом возврата, return обязателен:

Go 
func getMessage() string {
    message := "Привет"
    // Ошибка компиляции: missing return
}

Исправление:

Go 
func getMessage() string {
    message := "Привет"
    return message
}

Вопросы

01 Что делает оператор return?
Показать ответ

Оператор return выполняет две функции:

  1. Завершает выполнение функции
  2. Передаёт значение обратно в место вызова
02 Где указывается тип возвращаемого значения?
Показать ответ

Тип возвращаемого значения указывается после круглых скобок с параметрами:

func add(a, b int) int {
                    ^^^
                    здесь
03 Чем отличается функция, которая выводит результат, от функции, которая его возвращает?
Показать ответ

Функция с выводом (fmt.Println) показывает результат в консоли, но его нельзя использовать в коде. Функция с возвратом (return) передаёт значение обратно — его можно сохранить в переменную и использовать дальше.

04 Выполнится ли код после return?
Показать ответ

Нет. return немедленно завершает функцию. Код после return называется "недостижимым" и никогда не выполнится.

Задачи

01 Что выведет код? 
package main

import "fmt"

func main() {
    x := double(5)
    fmt.Println(x)
}

func double(n int) int {
    return n * 2
}
Показать решение

Вывод: 10

Разбор:

  1. Вызывается double(5)
  2. Внутри: 5 * 2 = 10
  3. return 10 возвращает 10
  4. x = 10, выводится 10
02 Что выведет код? 
package main

import "fmt"

func main() {
    result := checkSign(-7)
    fmt.Println(result)
}

func checkSign(n int) int {
    if n > 0 {
        return 1
    }
    if n < 0 {
        return -1
    }
    return 0
}
Показать решение

Вывод: -1

Разбор:

  1. Вызывается checkSign(-7)
  2. -7 > 0 — ложно, пропускаем
  3. -7 < 0 — истинно, return -1
  4. Результат: -1
03 Найди ошибку 
func getGreeting() string {
    greeting := "Привет!"
}
Показать решение

Ошибка: Отсутствует return. Функция объявлена как возвращающая string, но не содержит return.

Исправление:

func getGreeting() string {
    greeting := "Привет!"
    return greeting
}

  1. a = getValue() = 5
  2. b = getValue() = 5

Вывод: 10

  1. add(1, 2) = 3
  2. add(3, 4) = 7
  3. add(3, 7) = 10
06 Что выведет код? 
package main

import "fmt"

func main() {
    result := process(10)
    fmt.Println(result)
}

func process(n int) int {
    if n > 5 {
        return n * 2
    }
    return n + 100
    fmt.Println("Конец функции")
}
Показать решение

Вывод: 20

Разбор:

  1. Вызывается process(10)
  2. 10 > 5 — истинно
  3. return 10 * 2 = 20
  4. Строка fmt.Println("Конец функции") никогда не выполнится — она после return
Key Ключевые идеи
  • return завершает функцию и возвращает значение
  • Тип возврата указывается после параметров: func name() int
  • Тип значения после return должен совпадать с объявленным типом
  • Код после return не выполняется

Правило: Если функция объявлена с типом возврата, каждый путь выполнения должен заканчиваться return.
