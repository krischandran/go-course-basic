Множественные возвраты

Go позволяет функциям возвращать несколько значений одновременно. Это мощная особенность языка, которая особенно полезна для обработки ошибок.

Зачем возвращать несколько значений

В большинстве языков функция возвращает одно значение. Но иногда нужно вернуть несколько:

  • Результат операции и признак успеха
  • Частное и остаток от деления
  • Минимум и максимум из набора чисел

В Go функция может возвращать несколько значений одновременно.

Множественный возврат на самом деле

Синтаксис

Типы возвращаемых значений перечисляются в круглых скобках:

Go
func divide(a, b int) (int, int) {
    quotient := a / b
    remainder := a % b
    return quotient, remainder
}
func divide(a, b int) (int, int) { | | | +-- тип второго возвращаемого значения +-- тип первого возвращаемого значения return quotient, remainder | | | +-- второе значение +-- первое значение }

Получение нескольких значений

При вызове указываем несколько переменных через запятую:

Go
package main

import "fmt"

func main() {
    q, r := divide(17, 5)
    fmt.Println("Частное:", q)
    fmt.Println("Остаток:", r)
}

func divide(a, b int) (int, int) {
    return a / b, a % b
}
Частное: 3
Остаток: 2

Примеры использования

Минимум и максимум

Go
package main

import "fmt"

func main() {
    min, max := minMax(3, 7)
    fmt.Println("Минимум:", min)
    fmt.Println("Максимум:", max)
}

func minMax(a, b int) (int, int) {
    if a < b {
        return a, b
    }
    return b, a
}
Минимум: 3
Максимум: 7

Разные типы

Возвращаемые значения могут быть разных типов:

Go
package main

import "fmt"

func main() {
    name, age := getUserInfo()
    fmt.Println("Имя:", name)
    fmt.Println("Возраст:", age)
}

func getUserInfo() (string, int) {
    return "Алексей", 25
}
Имя: Алексей
Возраст: 25

Игнорирование значений

Если нужно только одно из возвращаемых значений, ненужное можно игнорировать с помощью _ (пустой идентификатор):

Go
package main

import "fmt"

func main() {
    q, _ := divide(17, 5)  // Нужно только частное
    fmt.Println("Частное:", q)

    _, r := divide(17, 5)  // Нужен только остаток
    fmt.Println("Остаток:", r)
}

func divide(a, b int) (int, int) {
    return a / b, a % b
}
Частное: 3
Остаток: 2

_ говорит Go: "это значение мне не нужно, не создавай для него переменную".

!

В Go нельзя объявить переменную и не использовать её — будет ошибка компиляции. _ решает эту проблему.

Порядок значений

Порядок переменных при получении должен соответствовать порядку в return:

Go
func getCoordinates() (int, int) {
    return 10, 20  // x, y
}

func main() {
    x, y := getCoordinates()  // x=10, y=20 — правильно
    y, x := getCoordinates()  // y=10, x=20 — перепутано!
}

Ловушка: перепутанный порядок

Go
package main

import "fmt"

func main() {
    remainder, quotient := divide(17, 5)  // Неправильный порядок!
    fmt.Println("Частное:", quotient)     // Выведет: Частное: 2
    fmt.Println("Остаток:", remainder)    // Выведет: Остаток: 3
}

func divide(a, b int) (int, int) {
    return a / b, a % b  // частное, остаток
}

Компилятор не заметит ошибку — типы совпадают. Всегда проверяйте порядок.

Именованные возвращаемые значения

Go позволяет давать имена возвращаемым значениям:

Go
func divide(a, b int) (quotient int, remainder int) {
    quotient = a / b
    remainder = a % b
    return  // "голый" return возвращает именованные значения
}

При этом:

  • Именованные значения автоматически инициализируются нулевыми значениями
  • return без аргументов возвращает текущие значения именованных переменных
✓

Именованные возвраты полезны для документирования — сразу видно, что функция возвращает. Но "голый" return может ухудшить читаемость в длинных функциях.

Количество переменных при получении

Количество переменных слева от := должно точно соответствовать количеству возвращаемых значений:

Go
func getTwo() (int, int) {
    return 1, 2
}

func main() {
    a := getTwo()        // Ошибка! Ожидается 2 переменные
    a, b, c := getTwo()  // Ошибка! Ожидается 2 переменные
    a, b := getTwo()     // OK
}

Исключение — игнорирование с _:

Go
a, _ := getTwo()  // OK: берём только первое
_, b := getTwo()  // OK: берём только второе

Проверь себя

Вопросы

01Как объявить функцию, возвращающую два значения типа int?
Показать ответ

func имяФункции() (int, int) { return значение1, значение2 }
Типы указываются в круглых скобках через запятую.

02Как игнорировать одно из возвращаемых значений?
Показать ответ

Использовать пустой идентификатор _:
value, _ := functionReturningTwo()

03Что произойдёт, если перепутать порядок переменных при получении?
Показать ответ

Если типы одинаковые, компилятор не заметит ошибку. Программа скомпилируется, но логика будет неверной — значения присвоятся в неправильном порядке.

04Что такое "голый" return?
Показать ответ

"Голый" return — это return без аргументов в функции с именованными возвращаемыми значениями. Он возвращает текущие значения именованных переменных.

Задачи

01Что выведет код?
package main

import "fmt"

func main() {
    a, b := swap(5, 10)
    fmt.Println(a, b)
}

func swap(x, y int) (int, int) {
    return y, x
}
Показать решение

Вывод: 10 5

swap(5, 10) возвращает y, x = 10, 5

02Что выведет код?
package main

import "fmt"

func main() {
    _, second := getTwo()
    fmt.Println(second)
}

func getTwo() (int, int) {
    return 100, 200
}
Показать решение

Вывод: 200

Первое значение (100) игнорируется с _.

03Найди ошибку
func getData() (int, string) {
    return "hello", 42
}
Показать решение

Ошибка: Порядок значений не соответствует типам. Объявлено (int, string), возвращается ("hello", 42) — строка и число в обратном порядке.

Исправление: return 42, "hello"

04Что выведет код?
package main

import "fmt"

func main() {
    sum, product := calculate(3, 4)
    fmt.Println(sum + product)
}

func calculate(a, b int) (int, int) {
    return a + b, a * b
}
Показать решение

Вывод: 19

sum = 3+4 = 7, product = 3*4 = 12, 7 + 12 = 19

05Что выведет код?
package main

import "fmt"

func main() {
    name, ok := lookup(5)
    if ok {
        fmt.Println(name)
    } else {
        fmt.Println("не найдено")
    }
}

func lookup(id int) (string, bool) {
    if id == 1 {
        return "Алексей", true
    }
    return "", false
}
Показать решение

Вывод: не найдено

lookup(5): id не равен 1, возвращается "", false.

06Что выведет код?
package main

import "fmt"

func main() {
    a, b := getValues()
    c, d := getValues()
    fmt.Println(a + c, b + d)
}

func getValues() (int, int) {
    return 10, 20
}
Показать решение

Вывод: 20 40

a=10, b=20, c=10, d=20. 10+10=20, 20+20=40.

KeyКлючевые идеи
  • Go позволяет возвращать несколько значений: func f() (int, string)
  • Получение: a, b := f()
  • Игнорирование: a, _ := f()
  • Порядок переменных должен соответствовать порядку в return

Правило: Количество переменных слева от := должно равняться количеству возвращаемых значений (или использовать _ для игнорирования).
