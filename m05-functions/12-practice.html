Модуль 5: Функции

Практика: функции

Закрепим всё, что изучили в модуле. Каждая задача проверяет конкретную тему.

Задачи

Задача 1: Объявление функций

Что выведет код?

func main() { greet() greet() } func greet() { fmt.Println("Привет!") }

Вывод:

Привет! Привет!

Тема: Объявление и вызов функций. Функция greet объявлена без параметров и возвращаемых значений. Вызывается дважды - выполняется дважды.

Задача 2: Параметры и аргументы

Что выведет код?

func main() { x := 10 double(x) fmt.Println(x) } func double(n int) { n = n * 2 }

Вывод: 10

Тема: Передача по значению. Параметр n - копия x. Изменение n не влияет на x в main.

Задача 3: Возвращаемые значения

Что выведет код?

func main() { result := add(3, multiply(2, 4)) fmt.Println(result) } func add(a, b int) int { return a + b } func multiply(a, b int) int { return a * b }

Вывод: 11

Тема: Возвращаемые значения, вложенные вызовы.

  1. multiply(2, 4) возвращает 8
  2. add(3, 8) возвращает 11
Задача 4: Множественные возвраты

Что выведет код?

func main() { a, b := swap(5, 10) c, _ := swap(a, b) fmt.Println(c) } func swap(x, y int) (int, int) { return y, x }

Вывод: 5

Тема: Множественные возвраты, игнорирование значений.

  1. swap(5, 10) возвращает a=10, b=5
  2. swap(10, 5) возвращает c=5, _=10
  3. Выводится c = 5
Задача 5: Область видимости

Что выведет код?

var x = 100 func main() { x := 10 fmt.Println(x) printX() } func printX() { fmt.Println(x) }

Вывод:

10 100

Тема: Локальные vs глобальные переменные.

  1. В main создаётся локальная x = 10 (затеняет глобальную)
  2. Выводится локальная x = 10
  3. В printX нет локальной x, используется глобальная x = 100
Задача 6: Shadowing

Что выведет код?

func main() { n := 5 for i := 0; i < 3; i++ { n := n * 2 fmt.Print(n, " ") } fmt.Println(n) }

Вывод: 10 10 10 5

Тема: Shadowing в цикле.

  1. Внешняя n = 5
  2. На каждой итерации создаётся НОВАЯ локальная n := 5 * 2 = 10
  3. Выводится 10 трижды
  4. После цикла внешняя n = 5
Задача 7: defer

Что выведет код?

func main() { fmt.Print(count()) } func count() int { defer fmt.Print("A") defer fmt.Print("B") defer fmt.Print("C") return 42 }

Вывод: CBA42

Тема: defer и LIFO.

  1. return готовит значение 42
  2. defer выполняются в обратном порядке: C, B, A
  3. Функция возвращает 42
  4. main печатает 42
Задача 8: Аргументы defer

Что выведет код?

func main() { x := 1 defer fmt.Println(x) x = 2 defer fmt.Println(x) x = 3 fmt.Println(x) }

Вывод:

3 2 1

Тема: Аргументы defer вычисляются сразу.

  1. defer с x=1 запоминает 1
  2. defer с x=2 запоминает 2
  3. Выводится x=3
  4. defer в обратном порядке: 2, 1
Задача 9: error и nil

Что выведет код?

func main() { a, err := process(10) if err != nil { fmt.Println("Ошибка A") } b, err := process(-5) if err != nil { fmt.Println("Ошибка B") } fmt.Println(a, b) } func process(n int) (int, error) { if n < 0 { return 0, fmt.Errorf("отрицательное число") } return n * 2, nil }

Вывод:

Ошибка B 20 0

Тема: Паттерн if err != nil.

  1. process(10) возвращает a=20, err=nil - без ошибки
  2. process(-5) возвращает b=0, err!=nil - выводится "Ошибка B"
  3. Выводится 20 0
Задача 10: Цепочка ошибок

Что выведет код?

func main() { err := level1() fmt.Println(err) } func level1() error { err := level2() if err != nil { return fmt.Errorf("level1: %w", err) } return nil } func level2() error { return fmt.Errorf("база данных недоступна") }

Вывод: level1: база данных недоступна

Тема: fmt.Errorf и оборачивание ошибок.

  1. level2() возвращает ошибку "база данных недоступна"
  2. level1() оборачивает её с контекстом "level1: "
  3. Итоговое сообщение показывает всю цепочку
Задача 11: panic и defer

Что выведет код?

func main() { defer fmt.Println("main defer") dangerous() fmt.Println("after dangerous") } func dangerous() { defer fmt.Println("dangerous defer") panic("boom") }

Вывод:

dangerous defer main defer panic: boom ...

Тема: defer выполняется при panic.

  1. panic в dangerous
  2. Выполняется defer в dangerous: "dangerous defer"
  3. Возврат в main, выполняется defer: "main defer"
  4. Программа завершается с panic
  5. "after dangerous" никогда не выводится
Задача 12: Комплексная

Что выведет код?

func main() { result, err := calculate(10, 2) if err != nil { fmt.Println("Ошибка:", err) return } sum, product := process(result) fmt.Printf("Сумма: %d, Произведение: %d\n", sum, product) } func calculate(a, b int) (int, error) { if b == 0 { return 0, fmt.Errorf("деление на ноль") } return a / b, nil } func process(n int) (int, int) { return n + 10, n * 10 }

Вывод: Сумма: 15, Произведение: 50

Тема: Комбинация всех концепций.

  1. calculate(10, 2) возвращает 5, nil
  2. Ошибки нет, продолжаем
  3. process(5) возвращает 15, 50
  4. Выводим результат

Задачи на написание кода

Задача 13: Напиши функцию max3

Напиши функцию max3(a, b, c int) int, которая возвращает максимум из трёх чисел.

func main() { fmt.Println(max3(1, 5, 3)) // 5 fmt.Println(max3(10, 2, 7)) // 10 fmt.Println(max3(4, 4, 4)) // 4 } func max3(a, b, c int) int { max := a if b > max { max = b } if c > max { max = c } return max }
Задача 14: Напиши функцию с ошибкой

Напиши функцию safeDivide(a, b float64) (float64, error), которая делит a на b. При b == 0 возвращает ошибку.

func main() { result, err := safeDivide(10, 2) if err != nil { fmt.Println("Ошибка:", err) return } fmt.Println("Результат:", result) result2, err2 := safeDivide(10, 0) if err2 != nil { fmt.Println("Ошибка:", err2) return } fmt.Println("Результат:", result2) } func safeDivide(a, b float64) (float64, error) { if b == 0 { return 0, fmt.Errorf("деление на ноль") } return a / b, nil }
Результат: 5 Ошибка: деление на ноль
Задача 15: Функция с defer

Напиши функцию withLogging(name string), которая выводит "Начало: name" при входе и "Конец: name" при выходе (используя defer).

func main() { withLogging("main") } func withLogging(name string) { fmt.Println("Начало:", name) defer fmt.Println("Конец:", name) fmt.Println("Выполняем работу...") }
Начало: main Выполняем работу... Конец: main
