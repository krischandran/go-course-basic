Scan и ввод данных

Программы, которые всегда выводят одно и то же -- скучные. Полезная программа взаимодействует с пользователем: спрашивает имя, запрашивает число, ждет команду. Для этого в Go есть функции ввода.

Зачем программе ввод?

Любая полезная программа работает по схеме:

Ввод данных (от пользователя)
Обработка (логика)
Вывод результата (на экран)

Без ввода программа статична -- всегда делает одно и то же:

Go 
// Скучная программа — всегда здоровается только с Павлом
name := "Павел"
fmt.Println("Привет,", name)

С вводом программа интерактивна:

Go 
// Интересная программа — здоровается с кем угодно
var name string
fmt.Scan(&name)
fmt.Println("Привет,", name)

fmt.Scan -- чтение данных

fmt.Scan читает данные, которые вводит пользователь, и записывает их в переменную:

Go 
package main

import "fmt"

func main() {
    fmt.Print("Введите имя: ")

    var name string
    fmt.Scan(&name)

    fmt.Println("Привет,", name)
}

Как это работает:

  1. Print выводит приглашение (без перехода на новую строку)
  2. Программа останавливается и ждет ввода
  3. Пользователь вводит текст и нажимает Enter
  4. Scan записывает введенное в переменную name
  5. Программа продолжает работу

Зачем нужен & (амперсанд)?

!
Important

& -- обязательный символ перед переменной в Scan. Без него код не скомпилируется.

Почему так?

Scan должен записать значение в переменную. Но если передать просто name, функция получит копию переменной -- пустую строку. Изменив копию, оригинал останется пустым.

&name передает не копию, а адрес переменной -- место в памяти, где она хранится. Зная адрес, Scan может записать данные напрямую в оригинальную переменную.

fmt.Scan(name) -- передаем копию (пустую строку). Scan изменит копию, оригинал пустой
fmt.Scan(&name) -- передаем адрес переменной. Scan запишет данные по этому адресу
*
Tip

Думай о & как об адресе дома. Если сказать курьеру "дом" -- он ничего не доставит. А если дать адрес -- посылка придет куда нужно.

Чтение нескольких значений

Scan умеет читать несколько значений за раз. Пользователь вводит их через пробел:

Go 
var firstName, lastName string
fmt.Scan(&firstName, &lastName)

fmt.Printf("Привет, %s %s!\n", firstName, lastName)

Ввод: Павел Тарасов

Вывод: Привет, Павел Тарасов!

Как это работает?

Scan разделяет ввод по пробелам. Первое слово -- в первую переменную, второе -- во вторую.

!
Important

Количество переменных должно соответствовать количеству введенных слов. Лишние слова будут проигнорированы.

Чтение чисел

Scan умеет читать не только строки. Тип переменной определяет, что ожидается:

Go 
// Целое число
var age int
fmt.Scan(&age)

// Дробное число
var price float64
fmt.Scan(&price)

Пример:

Go 
fmt.Print("Сколько вам лет? ")
var age int
fmt.Scan(&age)

fmt.Printf("Через 10 лет вам будет %d\n", age+10)
*
Tip

Для дробных чисел вводи точку, не запятую: 3.14, не 3,14.

Scan vs Scanln

В пакете fmt есть две похожие функции:

Функция Разделитель Когда использовать
Scan Пробел или Enter Чтение отдельных слов/чисел
Scanln Только Enter Чтение до конца строки
Go 
// Scan — пробел разделяет значения
fmt.Scan(&a, &b)  // ввод "5 10" → a=5, b=10

// Scanln — читает до Enter
fmt.Scanln(&a, &b)  // то же самое, но строго одна строка

На практике Scan используется чаще.

Проблема: строка с пробелами

Scan разделяет ввод по пробелам. Но что если нужно прочитать всю строку целиком?

Go 
var fullName string
fmt.Scan(&fullName)
// Ввод: "Павел Тарасов"
// fullName = "Павел"  — только первое слово!

Для чтения строки с пробелами нужен другой инструмент -- bufio.Scanner.

bufio.Scanner -- чтение строк целиком

bufio.Scanner читает ввод построчно, сохраняя пробелы:

Go 
package main

import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    fmt.Print("Введите полное имя: ")

    scanner := bufio.NewScanner(os.Stdin)
    scanner.Scan()
    fullName := scanner.Text()

    fmt.Println("Привет,", fullName)
}

Ввод: Павел Тарасов

Вывод: Привет, Павел Тарасов

Как это работает?

Go 
scanner := bufio.NewScanner(os.Stdin)  // создаем сканер для консольного ввода
scanner.Scan()                          // ждем ввода и читаем строку
fullName := scanner.Text()              // получаем прочитанный текст

Три новых элемента:

Элемент Что это Зачем
bufio Пакет Буферизованный ввод-вывод
os.Stdin Источник Стандартный ввод (консоль)
scanner.Text() Метод Возвращает прочитанную строку
*
Tip

Когда что использовать: fmt.Scan -- для отдельных слов и чисел. bufio.Scanner -- для строк с пробелами.

Возвращаемые значения Scan

Scan возвращает два значения:

Go 
n, err := fmt.Scan(&name)
Значение Тип Что содержит
n int Количество успешно прочитанных значений
err error Ошибка (или nil, если все хорошо)

Когда может быть ошибка?

  • Пользователь ввел текст вместо числа
  • Ввод закончился раньше, чем ожидалось

В учебных программах ошибки часто игнорируют:

Go 
fmt.Scan(&name)  // без проверки — для простоты

Но в реальном коде ошибки всегда нужно обрабатывать. Об этом -- в модуле про ошибки.

Пустой идентификатор _

Если не нужны возвращаемые значения, но хочется явно показать это:

Go 
_, _ = fmt.Scan(&name)  // явно игнорируем n и err

_ -- это пустой идентификатор (blank identifier). Он говорит: "это значение мне не нужно".

Go не позволяет объявлять неиспользуемые переменные -- будет ошибка компиляции. _ решает эту проблему, когда значение есть, но не нужно.

Проверь себя

Вопросы

01 Зачем перед переменной в fmt.Scan нужен символ &?
Показать ответ

& передает адрес переменной, а не ее копию. Это позволяет Scan записать введенное значение напрямую в оригинальную переменную. Без & функция получит копию и изменит только ее -- оригинал останется пустым.

02 Чем отличается fmt.Scan от bufio.Scanner?
Показать ответ

fmt.Scan разделяет ввод по пробелам -- каждое слово идет в отдельную переменную.

bufio.Scanner читает строку целиком, включая пробелы между словами.

03 Что вернет fmt.Scan при успешном чтении трех значений?
Показать ответ

Вернет 3 (количество успешно прочитанных значений) и nil (отсутствие ошибки).

04 Для чего используется пустой идентификатор _?
Показать ответ

Для явного игнорирования значений, которые не нужны. Go требует использовать все объявленные переменные -- _ позволяет "принять" значение и сразу отбросить его.

Задачи

01 Что выведет?
Go 
var name string
fmt.Scan(&name)
fmt.Println("Привет,", name)

Ввод: Мария

Показать решение

Вывод: Привет, Мария

Почему: Scan читает строку "Мария" и записывает в переменную name. Println выводит "Привет," и значение переменной через пробел.

02 Что выведет?
Go 
var a, b string
fmt.Scan(&a, &b)
fmt.Println(b, a)

Ввод: Hello World

Показать решение

Вывод: World Hello

Почему: Scan разделяет ввод по пробелу: a = "Hello", b = "World". Вывод в обратном порядке: сначала b, потом a.

03 Что выведет?
Go 
var x int
fmt.Scan(&x)
fmt.Printf("Результат: %d\n", x*2)

Ввод: 15

Показать решение

Вывод: Результат: 30

Почему: Scan читает "15" и преобразует в число 15 (тип int). Умножаем на 2, получаем 30.

04 Что выведет?
Go 
var name string
fmt.Scan(&name)
fmt.Println(name)

Ввод: Анна Каренина

Показать решение

Вывод: Анна

Почему: Scan читает до первого пробела. Получаем только "Анна", "Каренина" игнорируется.

05 Найди ошибку
Go 
var age int
fmt.Scan(age)
fmt.Println("Возраст:", age)
Показать решение

Ошибка: Отсутствует & перед age.

Правильно: fmt.Scan(&age)

Без & код не скомпилируется. Scan требует адрес переменной, чтобы записать в нее значение.

Резюме

Key Ввод данных в Go
  • fmt.Scan -- чтение данных из консоли. Разделяет ввод по пробелам
  • & (амперсанд) -- передает адрес переменной, чтобы Scan мог записать в нее значение
  • Несколько значений: fmt.Scan(&a, &b) -- пользователь вводит через пробел
  • Типы: переменная int -- читаем число, string -- читаем текст
  • bufio.Scanner -- для чтения строк с пробелами целиком
  • Пустой идентификатор _ -- для игнорирования ненужных возвращаемых значений
*
Tip

Для простых случаев (слово, число) используй fmt.Scan. Для строк с пробелами -- bufio.Scanner.
