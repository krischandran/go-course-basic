Операторы break и continue

Иногда нужно прервать цикл досрочно или пропустить текущую итерацию. Для этого в Go есть операторы break и continue.

  • break — полностью выходит из цикла
  • continue — пропускает остаток текущей итерации и переходит к следующей

Оба оператора работают только внутри циклов.

Оператор break

break немедленно завершает цикл и передаёт управление первой строке после цикла.

Go 
for {
    // ...
    if условиеВыхода {
        break  // выходим из цикла
    }
    // ...
}
// сюда попадаем после break

Пример: игра "Угадай число"

Go 
package main

import "fmt"

func main() {
    secretNumber := 7

    for {
        var guess int
        fmt.Print("Введите число: ")
        fmt.Scan(&guess)

        if guess == secretNumber {
            fmt.Println("Вы угадали!")
            break  // выходим из цикла
        }

        fmt.Println("Попробуйте ещё")
    }

    fmt.Println("Игра окончена")
}
✓
Tip

break делает код понятнее — сразу видно точку выхода из цикла.

Оператор continue

continue пропускает остаток текущей итерации и переходит к следующей.

Go 
for i := 0; i < 10; i++ {
    if условиеПропуска {
        continue  // пропускаем эту итерацию
    }
    // этот код не выполнится, если был continue
}

Пример: вывод только чётных чисел

Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 10; i++ {
        if i%2 != 0 {
            continue  // нечётное — пропускаем
        }
        fmt.Println(i)
    }
}

Вывод: 2 4 6 8 10

✓
Tip

continue полезен, когда нужно исключить некоторые случаи в начале итерации.

Когда использовать

Используйте break, когда:

  • Дальнейшие итерации не имеют смысла (элемент найден)
  • Нужен досрочный выход по условию
  • В бесконечном цикле для выхода

Используйте continue, когда:

  • Нужно пропустить определённые значения
  • Хотите избежать глубокой вложенности
  • Исключаете "плохие" данные в начале итерации

break во вложенных циклах

!
Important

break прерывает только ближайший цикл, в котором находится.

Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 3; i++ {
        fmt.Println("Внешний цикл: i =", i)

        for j := 1; j <= 5; j++ {
            if j == 3 {
                fmt.Println("  break при j =", j)
                break  // выход только из внутреннего цикла
            }
            fmt.Println("  Внутренний цикл: j =", j)
        }
    }
}

break выходит только из внутреннего цикла. Внешний продолжает работать.

Проверь себя

Вопросы

01 Чем отличается break от continue?
Показать ответ
  • break — полностью выходит из цикла, переходит к коду после цикла
  • continue — пропускает остаток текущей итерации и переходит к следующей итерации
02 Что произойдёт с внешним циклом, если break вызван во вложенном?
Показать ответ

break прервёт только ближайший (внутренний) цикл. Внешний цикл продолжит работу и перейдёт к своей следующей итерации.

03 Когда continue полезнее, чем обычный if?
Показать ответ

continue полезнее, когда нужно исключить некоторые случаи в начале итерации. Это уменьшает вложенность кода.

04 Можно ли использовать break вне цикла?
Показать ответ

Нет, break работает только внутри циклов (for) и оператора switch. Использование break вне этих конструкций вызовет ошибку компиляции.

Задачи

01 Что выведет код?
Go 
package main

import "fmt"

func main() {
    sum := 0
    for i := 2; i < 20; i++ {
        if i%4 == 0 {
            break
        }
        sum = sum + i
    }
    fmt.Println(sum)
}
Показать решение

Вывод: 5

Разбор:

  1. i = 2: 2 % 4 = 2, sum = 2
  2. i = 3: 3 % 4 = 3, sum = 5
  3. i = 4: 4 % 4 = 0, break
02 Что выведет код?
Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 5; i++ {
        if i == 3 {
            continue
        }
        fmt.Print(i)
    }
}
Показать решение

Вывод: 1245

При i = 3 выполняется continue, вывод пропускается.

03 Что выведет код?
Go 
package main

import "fmt"

func main() {
    for i := 0; i < 3; i++ {
        for j := 0; j < 3; j++ {
            if j == 1 {
                break
            }
            fmt.Print(i, j, " ")
        }
    }
}
Показать решение

Вывод: 00 10 20

break прерывает только внутренний цикл при j = 1.

04 Найди ошибку
Go 
for i := 1; i <= 10; i++ {
    if i%2 == 0 {
        continue
        fmt.Println(i)  // чётные числа
    }
}
Показать решение

Ошибка: fmt.Println(i) стоит после continue и никогда не выполнится.

Исправление:

Go
for i := 1; i <= 10; i++ {
    if i%2 != 0 {
        continue  // пропускаем нечётные
    }
    fmt.Println(i)  // выводим чётные
}
05 Что выведет код?
Go 
package main

import "fmt"

func main() {
    count := 0
    for i := 1; i <= 20; i++ {
        if i%3 != 0 {
            continue
        }
        count++
        if count == 3 {
            break
        }
    }
    fmt.Println(count)
}
Показать решение

Вывод: 3

Считаем числа, кратные 3: 3, 6, 9. При count = 3 выходим.

06 Что выведет код?
Go 
package main

import "fmt"

func main() {
    i := 0
    s := 0
    for i+1 < 10 {
        i = i + 2
        s = s + i
        if s > 16 {
            break
        }
    }
    i = i + 1
    fmt.Println(i)
}
Показать решение

Вывод: 9

i: 2, 4, 6, 8 (s = 2+4+6+8 = 20 > 16, break). Затем i = 8 + 1 = 9.

Key Ключевые идеи
  • break — выходит из цикла полностью
  • continue — пропускает остаток итерации
  • break во вложенных циклах прерывает только ближайший
  • continue уменьшает вложенность кода

Правило: break — для выхода, continue — для пропуска. Оба работают только в циклах.
