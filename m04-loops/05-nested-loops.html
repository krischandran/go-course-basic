Вложенные циклы

Цикл может содержать внутри себя другой цикл. Такая конструкция называется вложенным циклом.

Представь обычные часы: часовая стрелка делает 12 шагов за полдня, минутная - 60 шагов за час, секундная - 60 шагов за минуту. Каждый раз, когда секундная стрелка делает полный оборот, минутная сдвигается на 1. Это и есть вложенные циклы: внутренний цикл полностью выполняется на каждой итерации внешнего.

Базовый пример

Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 4; j++ {
            fmt.Print("*")
        }
        fmt.Println()  // переход на новую строку
    }
}

Вывод:

Output
****
****
****

Как это работает:

  1. Внешний цикл (i): 3 итерации
  2. На каждой итерации внешнего - внутренний цикл (j): 4 итерации
  3. Всего выводов *: 3 * 4 = 12
!
Important

Внутренний цикл полностью выполняется на каждой итерации внешнего.

Подсчёт итераций

Общее количество итераций внутреннего цикла:

Всего = итерации_внешнего * итерации_внутреннего
Go 
for i := 0; i < 12; i++ {      // 12 итераций
    for j := 0; j < 60; j++ {  // 60 итераций
        // ...
    }
}
// Всего: 12 * 60 = 720 выполнений внутреннего кода

Таблица умножения

Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 5; i++ {
        for j := 1; j <= 5; j++ {
            fmt.Printf("%4d", i*j)  // ширина 4 символа
        }
        fmt.Println()
    }
}

Важное правило

!
Important

Переменные вложенных циклов должны иметь разные имена.

Go 
// Правильно - разные имена
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        fmt.Print(i, j, " ")
    }
}

// Неправильно - одинаковые имена
for i := 0; i < 3; i++ {
    for i := 0; i < 3; i++ {  // затеняет внешний i!
        // ...
    }
}

Во втором случае внутренний i затеняет внешний. Программа скомпилируется, но работать будет неправильно.

break во вложенных циклах

break прерывает только ближайший цикл:

Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 5; j++ {
            if j == 3 {
                break  // выходим только из внутреннего
            }
            fmt.Println(i, j)
        }
    }
}

Внутренний цикл прерывается при j == 3, но внешний продолжает работать.

Проверь себя

Вопросы

01 Сколько раз выполнится внутренний код?
Go
for i := 0; i < 5; i++ {
    for j := 0; j < 3; j++ {
        // внутренний код
    }
}
Показать ответ

15 раз (5 * 3 = 15)

02 Почему переменные должны иметь разные имена?
Показать ответ

Если использовать одинаковые имена, внутренняя переменная затенит внешнюю. Это приведёт к тому, что внутри внутреннего цикла нельзя будет обратиться к переменной внешнего цикла.

03 Что произойдёт с внешним циклом при break во внутреннем?
Показать ответ

Внешний цикл продолжит работу. break прерывает только ближайший (внутренний) цикл и переходит к следующей итерации внешнего.

04 Что выведет код?
Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 3; j++ {
            if i == j {
                continue
            }
            fmt.Print(i, j, " ")
        }
    }
}
Показать решение

Вывод: 12 13 21 23 31 32

Пропускаются пары где i == j: (1,1), (2,2), (3,3)

05 Что выведет код?
Go 
package main

import "fmt"

func main() {
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 3; j++ {
            if j == 2 {
                break
            }
            fmt.Print(i, j, " ")
        }
    }
}
Показать решение

Вывод: 11 21 31

На каждой итерации внешнего цикла внутренний прерывается при j == 2.

Key Ключевые идеи
  • Вложенный цикл - цикл внутри цикла
  • Внутренний полностью выполняется на каждой итерации внешнего
  • Всего итераций = произведение итераций всех циклов
  • Переменные должны иметь разные имена
  • break прерывает только ближайший цикл

Правило: Внешний цикл - i, вложенный - j, ещё глубже - k.
