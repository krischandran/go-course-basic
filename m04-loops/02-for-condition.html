Цикл for с условием

Когда количество итераций заранее неизвестно, используется for только с условием — аналог while из других языков.

В предыдущем уроке мы изучили классический for с тремя частями: init; condition; post. Но что если количество повторений заранее неизвестно?

Представь игру "Угадай число". Программа загадала число, пользователь вводит варианты до тех пор, пока не угадает. Сколько попыток потребуется? Одна? Десять? Сто? Это зависит от удачи игрока.

В таких случаях используется for с условием — упрощённая форма цикла, где указывается только условие продолжения.

Синтаксис

Go 
for условие {
    // тело цикла
}

Цикл выполняется, пока условие истинно. Как только условие становится ложным — цикл завершается.

✓
Tip

В других языках (Python, C, Java) для этого есть ключевое слово while. В Go его нет — используется for только с условием.

Форма Синтаксис Когда использовать
Классическая for i := 0; i < n; i++ Известно количество итераций
С условием for условие Количество итераций неизвестно

Пример: игра "Угадай число"

Проблема с классическим for

Попробуем ограничить игру 5 попытками:

Go 
package main

import "fmt"

func main() {
    secretNumber := 7

    for i := 0; i < 5; i++ {
        var guess int
        fmt.Print("Введите число: ")
        fmt.Scan(&guess)

        if guess == secretNumber {
            fmt.Println("Вы угадали!")
        } else {
            fmt.Println("Попробуйте ещё")
        }
    }
}

Проблема: даже если игрок угадал на 2-й попытке, программа продолжит запрашивать числа ещё 3 раза.

Решение с for и условием

Используем переменную-флаг для отслеживания состояния:

Go 
package main

import "fmt"

func main() {
    secretNumber := 7
    isGuessed := false  // флаг: угадано или нет

    for !isGuessed {
        var guess int
        fmt.Print("Введите число: ")
        fmt.Scan(&guess)

        if guess == secretNumber {
            isGuessed = true
            fmt.Println("Вы угадали!")
        } else {
            fmt.Println("Попробуйте ещё")
        }
    }
}

Как это работает:

  1. isGuessed начинается как false
  2. Условие !isGuessed (не угадано) истинно — входим в цикл
  3. Если угадали — устанавливаем isGuessed = true
  4. На следующей проверке !isGuessed станет ложным — выходим

Переменные-флаги

Флаг — это булева переменная (bool), которая отвечает на вопрос "да/нет".

Префикс Пример Значение
is... isGuessed, isValid Является чем-то
has... hasError, hasData Имеет что-то
can... canContinue, canProcess Может делать что-то
!
Important

Флаги делают код понятнее. Вместо for x == 0 лучше писать for !isFinished.

Ещё примеры

Накопление суммы

Читаем числа с клавиатуры, пока сумма не превысит 100:

Go 
package main

import "fmt"

func main() {
    sum := 0

    for sum <= 100 {
        var n int
        fmt.Print("Введите число: ")
        fmt.Scan(&n)
        sum = sum + n
        fmt.Println("Текущая сумма:", sum)
    }

    fmt.Println("Сумма превысила 100!")
}

Вычисление степени двойки

Задача: найти 2^n, где n вводится с клавиатуры:

Go 
package main

import "fmt"

func main() {
    var n int
    fmt.Scan(&n)

    counter := 0
    result := 1

    for counter < n {
        result = result * 2
        counter++
    }

    fmt.Println(result)
}

Ловушка: бесконечный цикл

Если условие всегда истинно, цикл никогда не завершится:

Go 
x := 5
for x > 0 {
    fmt.Println(x)
    // Забыли уменьшать x!
}
// Бесконечный вывод "5"
!
Important

Всегда убедитесь, что условие когда-нибудь станет ложным. Иначе получится бесконечный цикл.

Проверь себя

Вопросы

01 Чем отличается for условие от классического for?
Показать ответ

Классический for используется, когда количество итераций известно заранее.

for с условием используется, когда количество итераций зависит от внешних факторов и заранее неизвестно.

02 Что такое переменная-флаг?
Показать ответ

Флаг — это булева переменная (bool), которая отвечает на вопрос "да/нет".

Примеры: isValid, hasError, canContinue, isGuessed

03 Что произойдёт, если условие цикла всегда истинно?
Показать ответ

Получится бесконечный цикл — программа будет выполняться вечно и "зависнет". Придётся принудительно остановить программу.

04 Как в Go написать аналог while из других языков?
Показать ответ

В Go нет ключевого слова while. Вместо него используется for только с условием:

Go
for условие {
    // тело
}

Задачи

01 Что выведет код?
Go 
package main

import "fmt"

func main() {
    x := 2
    for x < 20 {
        fmt.Print(x, " ")
        x = x + 3
    }
}
Показать решение

Вывод: 2 5 8 11 14 17

Разбор: x начинается с 2 и увеличивается на 3. При x = 20 условие ложно.

02 Что выведет код?
Go 
package main

import "fmt"

func main() {
    n := 5
    result := 1

    for n > 1 {
        result = result * n
        n--
    }

    fmt.Println(result)
}
Показать решение

Вывод: 120

Разбор: Это вычисление факториала 5! = 5 * 4 * 3 * 2 = 120

03 Сколько итераций?
Go 
i := 10
for i > 0 {
    i = i - 2
}
Показать решение

Ответ: 5 итераций

i: 10 -> 8 -> 6 -> 4 -> 2 -> 0 (выход при i = 0)

04 Найди ошибку
Go 
package main

import "fmt"

func main() {
    isRunning := true
    counter := 0

    for isRunning {
        fmt.Println(counter)
        counter++
    }
}
Показать решение

Ошибка: Переменная isRunning никогда не меняется на false, поэтому цикл бесконечный.

Исправление: Добавить условие выхода:

Go
for isRunning {
    fmt.Println(counter)
    counter++
    if counter >= 10 {
        isRunning = false
    }
}
05 Что выведет код?
Go 
package main

import "fmt"

func main() {
    sum := 0
    num := 1

    for sum < 15 {
        sum = sum + num
        num++
    }

    fmt.Println(sum, num)
}
Показать решение

Вывод: 15 6

Разбор: sum: 0+1=1, 1+2=3, 3+3=6, 6+4=10, 10+5=15 (выход). num = 6.

Key Ключевые идеи
  • for условие { } — цикл с неизвестным числом итераций
  • Работает пока условие истинно
  • Заменяет while из других языков
  • Переменные-флаги (isValid, hasError) делают код понятнее

Правило: Если знаешь сколько раз — используй классический for. Если зависит от условия — используй for условие.
