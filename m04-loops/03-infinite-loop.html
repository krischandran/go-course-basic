Бесконечный цикл

Иногда нужен цикл, который работает постоянно — пока его не остановят извне. В Go для этого используется for без условия.

Примеры использования:

  • Сервер, ожидающий запросы от клиентов
  • Игровой цикл, обрабатывающий действия игрока
  • Программа, читающая данные до сигнала остановки

Синтаксис

Go 
for {
    // тело цикла — выполняется бесконечно
}

Это самая короткая форма цикла. Она эквивалентна for true { } — условие всегда истинно.

Форма Синтаксис Когда использовать
Классическая for i := 0; i < n; i++ Известно количество итераций
С условием for условие Итерации зависят от условия
Бесконечная for Цикл работает до явной остановки

Когда бесконечный цикл полезен

Ожидание ввода

Программа-калькулятор, которая работает до ввода "exit":

Go 
package main

import "fmt"

func main() {
    for {
        var input string
        fmt.Print("Введите команду: ")
        fmt.Scan(&input)

        if input == "exit" {
            fmt.Println("Выход из программы")
            return  // выход из main = завершение программы
        }

        fmt.Println("Вы ввели:", input)
    }
}

Игровой цикл

Go 
package main

import "fmt"

func main() {
    isGameOver := false

    for !isGameOver {
        fmt.Println("Игровой цикл...")

        // Здесь была бы логика игры
        // Когда игра заканчивается:
        // isGameOver = true
    }
}

Опасность бесконечных циклов

Бесконечный цикл без способа выхода — это баг:

Go 
// Плохо! Программа зависнет
for {
    fmt.Println("Зациклились!")
}
!

Бесконечный цикл без способа выхода — это обычно ошибка. В следующем уроке мы изучим break для выхода из цикла.

Способы выхода из бесконечного цикла:

  1. Оператор break (изучим в следующем уроке)
  2. Оператор return (выход из функции)
  3. Внешнее прерывание (Ctrl+C)

Преобразование форм

Бесконечный цикл можно записать разными способами:

Go 
// Вариант 1: каноническая форма
for {
    fmt.Println("A")
}

// Вариант 2: явное true
for true {
    fmt.Println("A")
}

// Вариант 3: условие, которое всегда истинно
for 1 == 1 {
    fmt.Println("A")
}

Все три варианта работают одинаково, но первый — идиоматичный для Go.

i

Используйте for { } — это стандартный способ написания бесконечного цикла в Go.

Проверь себя

Вопросы

01 Как записать бесконечный цикл в Go?
Показать ответ
Go
for {
    // тело цикла
}

Это сокращённая форма цикла без условия. Цикл выполняется бесконечно, пока не будет остановлен оператором break, return или внешним прерыванием.

02 Почему for { } эквивалентен for true { }?
Показать ответ

Когда условие в цикле отсутствует, Go считает его всегда истинным. Поэтому for { } работает так же, как for true { } — цикл никогда не завершится сам по себе.

03 Когда бесконечный цикл — это не ошибка?
Показать ответ

Примеры корректного использования:

  • Серверы, ожидающие клиентские запросы
  • Игровые циклы, обрабатывающие события
  • Программы мониторинга, работающие постоянно
  • Интерактивные оболочки (REPL)

Во всех случаях есть механизм выхода: break, return или внешний сигнал.

04 Какие способы выхода из бесконечного цикла?
Показать ответ
  1. break — выходит из текущего цикла
  2. return — выходит из функции (и, следовательно, из цикла)
  3. Внешнее прерывание — Ctrl+C или завершение процесса

Задачи

01 Что произойдёт?
Go 
package main

import "fmt"

func main() {
    i := 0
    for {
        fmt.Println(i)
        i++
    }
}
Показать решение

Результат: Программа будет бесконечно выводить числа: 0, 1, 2, 3, ... и так далее.

Цикл бесконечный, и нет никакого условия выхода. Программа не остановится сама — её нужно прервать вручную (Ctrl+C).

02 Найди проблему
Go 
package main

import "fmt"

func main() {
    counter := 0

    for {
        counter++
    }

    fmt.Println("Счётчик:", counter)
}
Показать решение

Проблема: Строка fmt.Println("Счётчик:", counter) никогда не выполнится.

Цикл for { } бесконечный и не содержит способа выхода. Программа будет вечно увеличивать counter и никогда не дойдёт до вывода.

Исправление: Добавить условие выхода:

Go
for {
    counter++
    if counter >= 100 {
        break
    }
}
fmt.Println("Счётчик:", counter)
03 Что выведет код?
Go 
package main

import "fmt"

func main() {
    for {
        fmt.Println("Начало")
        return
        fmt.Println("Конец")
    }
}
Показать решение

Вывод: Начало

Разбор:

  1. Входим в бесконечный цикл
  2. Выводим "Начало"
  3. return завершает функцию main
  4. Программа заканчивается
  5. "Конец" никогда не выведется

Строка после return — это недостижимый код.

04 Перепишите с for { }
Go 
isRunning := true
for isRunning {
    fmt.Println("Работаем")
    // ... какое-то условие ...
    // isRunning = false
}
Показать решение
Go
for {
    fmt.Println("Работаем")
    // ... какое-то условие ...
    if условиеВыхода {
        break
    }
}

Бесконечный цикл с явным break часто понятнее, чем цикл с флагом.

Key Ключевые идеи
  • for { } — бесконечный цикл в Go
  • Эквивалентен for true { }
  • Полезен для серверов, игр, интерактивных программ
  • Требует явного способа выхода (break, return)

Правило: Бесконечный цикл = обязательный break или return внутри.
