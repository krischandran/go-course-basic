Метки (Labels)

В предыдущем уроке мы узнали, что break прерывает только ближайший цикл. Но что если нужно выйти сразу из нескольких вложенных циклов? В Go для этого есть метки.

Синтаксис меток

Метка - это идентификатор, за которым следует двоеточие:

Go 
outer:  // метка
    for ... {
        for ... {
            if условие {
                break outer  // выход к метке
            }
        }
    }

Метки используются с break и continue:

Go 
break имяМетки     // выход из цикла, помеченного меткой
continue имяМетки  // переход к следующей итерации помеченного цикла

break с меткой

Проблема: поиск в матрице

Ищем число в двумерной структуре. При нахождении нужно выйти из обоих циклов:

Go 
package main

import "fmt"

func main() {
    target := 5

outer:
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 3; j++ {
            if i*j == target {
                fmt.Println("Найдено:", i, "*", j, "=", target)
                break outer
            }
        }
    }
}

Вывод: Найдено: 1 * 5 = 5

✓
Tip

break outer выходит не из внутреннего цикла, а из цикла с меткой outer - то есть из внешнего.

continue с меткой

continue с меткой переходит к следующей итерации помеченного цикла:

Go 
package main

import "fmt"

func main() {
outer:
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 3; j++ {
            if j == 2 {
                continue outer  // переход к следующей итерации внешнего
            }
            fmt.Println(i, j)
        }
    }
}

Вывод:

Output
1 1
2 1
3 1

Разбор:

  • При j == 2 мы не просто пропускаем внутреннюю итерацию
  • Мы переходим к следующей итерации внешнего цикла
  • Поэтому j никогда не достигает 3

Когда использовать метки

Используйте метки, когда:

  • Нужно выйти из нескольких вложенных циклов
  • Нужно перейти к следующей итерации внешнего цикла
  • Альтернатива с флагом слишком громоздкая

Не используйте метки, когда:

  • Можно обойтись простым break или continue
  • Логика становится запутанной
  • Есть альтернатива: вынести код в функцию с return
i
Note

Функции мы изучим в модуле M05. Пока просто знайте, что это альтернатива меткам.

Правила именования меток

Go 
// Общепринятые имена
outer:      // для внешнего цикла
mainLoop:   // для основного цикла
searchLoop: // для цикла поиска
✓
Tip

Используйте понятные имена: outer, searchLoop, mainLoop. Избегайте l1, l2 - это неинформативно.

Проверь себя

Вопросы

01 Что такое метка в Go?
Показать ответ

Метка - это именованная точка в коде, которая используется для указания, к какому циклу относится break или continue. Синтаксис: имяМетки: перед циклом.

02 Чем отличается break от break label?
Показать ответ
  • break - выходит из ближайшего цикла
  • break label - выходит из цикла, помеченного указанной меткой (может быть внешним)
03 Что делает continue label?
Показать ответ

continue label пропускает остаток текущей итерации и переходит к следующей итерации помеченного цикла. Если метка на внешнем цикле - внутренний цикл полностью прерывается, и начинается новая итерация внешнего.

Задачи

01 Что выведет код?
Go 
package main

import "fmt"

func main() {
outer:
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 3; j++ {
            if i == 2 && j == 2 {
                break outer
            }
            fmt.Print(i, j, " ")
        }
    }
}
Показать решение

Вывод: 11 12 13 21

i = 1: все j. i = 2, j = 1 выводим, j = 2 - break outer.

02 Что выведет код?
Go 
package main

import "fmt"

func main() {
outer:
    for i := 1; i <= 3; i++ {
        for j := 1; j <= 3; j++ {
            if j == 2 {
                continue outer
            }
            fmt.Print(i, j, " ")
        }
        fmt.Print("* ")
    }
}
Показать решение

Вывод: 11 21 31

При j == 2 переходим к следующей итерации внешнего, строка fmt.Print("* ") никогда не выполняется.

03 Что выведет код?
Go 
package main

import "fmt"

func main() {
    sum := 0

loop:
    for i := 1; i <= 5; i++ {
        for j := 1; j <= 5; j++ {
            sum = sum + 1
            if sum >= 10 {
                break loop
            }
        }
    }

    fmt.Println(sum)
}
Показать решение

Вывод: 10

Каждая итерация увеличивает sum на 1. При sum = 10 выходим из обоих циклов.

Key Ключевые идеи
  • Метки позволяют выйти из нескольких вложенных циклов
  • break label - выход из помеченного цикла
  • continue label - переход к следующей итерации помеченного цикла
  • Альтернатива - функция с return

Правило: Если метки делают код сложнее - вынесите логику в функцию.
