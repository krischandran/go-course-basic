Модуль 1: Переменные

Константы и iota

Константа - это значение, которое не изменяется во время работы программы. Число пи всегда 3.14159, количество дней в неделе всегда 7, максимальное количество попыток входа - 3. Такие значения логично сделать константами. С константой компилятор сразу остановит сборку с ошибкой при попытке изменения. Константы защищают важные значения от случайного изменения.

Объявление констант

Константы создаются с помощью ключевого слова const:

Go 
package main

import "fmt"

func main() {
    const pi = 3.14159
    fmt.Println(pi)
}
3.14159

А что если попробовать изменить константу? Давай посмотрим:

Go 
package main

func main() {
    const pi = 3.14159
    pi = 3.14 // Ошибка компиляции!
}
cannot assign to pi

Компилятор не позволяет изменить константу - именно для этого они и нужны. Это защита на уровне языка, а не договорённость между программистами.

Тип константы можно указать явно, но обычно это не требуется:

Go 
package main

import "fmt"

func main() {
    const pi float64 = 3.14159 // с явным типом
    const e = 2.71828          // без типа

    fmt.Println(pi, e)
}
3.14159 2.71828

Константа без явного типа называется "нетипизированной" (от англ. untyped - без типа). Это просто число, которое пока не привязано к конкретному типу.

В чём польза? Одну константу можно использовать с переменными разных типов:

Go 
package main

import "fmt"

func main() {
    const maxSize = 100 // нетипизированная константа

    var i32 int32 = maxSize   // OK
    var i64 int64 = maxSize   // OK
    var f64 float64 = maxSize // OK - та же константа!

    fmt.Println(i32, i64, f64)
}
100 100 100

Как это работает? При каждом присваивании Go проверяет: "Можно ли значение 100 представить как int32? как int64? как float64?" Да - значит всё работает. Сама константа при этом не меняется - она остаётся просто числом 100.

Что можно хранить в константах

В константах можно хранить только простые значения:

  • Числа (целые и дробные)
  • Строки
  • Булевы значения (true/false)
Go 
package main

import "fmt"

func main() {
    const maxRetries = 3          // целое число
    const pi = 3.14159            // дробное число
    const appName = "MyApp"       // строка
    const debugMode = true        // булево значение

    fmt.Println(maxRetries, pi, appName, debugMode)
}
3 3.14159 MyApp true

Почему только простые значения? Константы вычисляются на этапе компиляции, до запуска программы. Компилятор должен знать значение заранее. Массивы и другие сложные структуры создаются во время выполнения - их значения компилятор заранее не знает.

Группы констант

Когда констант много, их удобно объединять в группу с помощью круглых скобок:

Go 
package main

import "fmt"

func main() {
    const (
        statusPending  = 0
        statusActive   = 1
        statusComplete = 2
    )

    fmt.Println(statusPending, statusActive, statusComplete)
}
0 1 2

Если константа объявлена без значения, Go повторяет выражение из предыдущей строки:

Go 
package main

import "fmt"

func main() {
    const (
        a = 1
        b = 5.5
        c      // c = 5.5 (повторяет выражение b)
        d      // d = 5.5 (тоже повторяет)
    )

    fmt.Println(a, b, c, d)
}
1 5.5 5.5 5.5

Почему c и d тоже равны 5.5? Go берёт выражение из предыдущей строки (5.5) и подставляет его заново. Получается c = 5.5, d = 5.5.

Это важно понять для iota - там тоже повторяется выражение, а не значение.

iota - автоматическая нумерация

Часто нужны константы с последовательными значениями: 0, 1, 2, 3... Можно прописать вручную, но это неудобно - легко ошибиться или забыть обновить при добавлении новой константы.

В Go есть элегантное решение - iota (от греческой буквы йота, обозначающей наименьшую величину). Это специальный счётчик, который начинается с 0 и увеличивается на 1 для каждой строки в блоке const:

Go 
package main

import "fmt"

func main() {
    const (
        Red   = iota // 0
        Green        // 1
        Blue         // 2
    )

    fmt.Println(Red, Green, Blue)
}
0 1 2

Как это работает?

  1. Red = iota - Go подставляет текущее значение iota (0)
  2. Green - значение не указано, Go повторяет выражение iota, но iota уже стала 1
  3. Blue - снова повторяет iota, которая теперь 2

Ключевой момент: Go не копирует значение 0, а повторяет само выражение iota. Поэтому каждая константа получает новое значение.

Пропуск значений

Иногда первое значение (0) не нужно. Например, хочешь начать нумерацию дней недели с 1, а не с 0. Для этого используй пустой идентификатор _:

Go 
package main

import "fmt"

func main() {
    const (
        _        = iota // 0 - пропускаем
        Monday          // 1
        Tuesday         // 2
        Wednesday       // 3
        Thursday        // 4
        Friday          // 5
        Saturday        // 6
        Sunday          // 7
    )

    fmt.Println("Понедельник:", Monday)
    fmt.Println("Воскресенье:", Sunday)
}
Понедельник: 1
Воскресенье: 7

Нижнее подчёркивание _ - это специальный идентификатор в Go, который означает "это значение мне не нужно, выброси его". iota всё равно увеличивается (становится 1, 2, 3...), но значение 0 никуда не записывается.

Выражения с iota

iota можно использовать в математических выражениях. Например, нужны значения, начинающиеся не с 0, а с 5:

Go 
package main

import "fmt"

func main() {
    const (
        a = iota + 5 // 0 + 5 = 5
        b            // 1 + 5 = 6
        c            // 2 + 5 = 7
        d            // 3 + 5 = 8
    )

    fmt.Println(a, b, c, d)
}
5 6 7 8

Почему получилось 5, 6, 7, 8? Компилятор подставляет iota + 5 для каждой строки. iota увеличивается: 0+5=5, 1+5=6, 2+5=7, 3+5=8.

Ещё пример - умножение:

Go 
package main

import "fmt"

func main() {
    const (
        a = iota * 10 // 0 * 10 = 0
        b             // 1 * 10 = 10
        c             // 2 * 10 = 20
        d             // 3 * 10 = 30
    )

    fmt.Println(a, b, c, d)
}
0 10 20 30

Сброс iota в новом блоке

Важный нюанс: iota сбрасывается в 0 в каждом новом блоке const:

Go 
package main

import "fmt"

func main() {
    const (
        a = iota // 0
        b        // 1
    )

    const (
        c = iota // 0 (сброс!)
        d        // 1
    )

    fmt.Println("Первый блок:", a, b)
    fmt.Println("Второй блок:", c, d)
}
Первый блок: 0 1
Второй блок: 0 1

Почему так? Каждый блок const - это отдельная группа констант. iota "живёт" только внутри своего блока. Это логично: цвета и дни недели - разные перечисления, и их нумерация не должна зависеть друг от друга.

Стиль именования

В Go константы именуются так же, как переменные - в camelCase:

Go 
const maxRetries = 3       // локальная константа (маленькая буква)
const MaxConnections = 100 // экспортируемая константа (большая буква)

Можно встретить стиль с заглавными буквами и подчёркиваниями (как в языке C):

Go 
const MAX_RETRIES = 3 // работает, но не принято в Go

Это работает, но не соответствует стилю Go. Придерживайся camelCase для локальных и PascalCase для экспортируемых констант.

Вопросы

01Чем константа отличается от переменной?
Показать ответ

Константу нельзя изменить после объявления, переменную - можно. Константы вычисляются при компиляции, переменные - во время выполнения.

02Что произойдёт при попытке изменить значение константы?
Показать ответ

Ошибка компиляции. Программа не соберётся.

03С какого числа начинается iota?
Показать ответ

С нуля (0).

04Что произойдёт с iota в новом блоке const?
Показать ответ

iota сбрасывается в 0. Каждый блок const начинает счёт заново.

05Как пропустить первое значение iota?
Показать ответ

Использовать пустой идентификатор: _ = iota.

06Какие типы данных можно хранить в константах?
Показать ответ

Числа (целые и дробные), строки, булевы значения (true/false).

Первое значение (0) пропущено с помощью _. first получает 1, second получает 2.

Go

    a = 5
    b
    c

fmt.Println(a, b, c)
Показать решение

Вывод: 5 5 5

Здесь нет iota! Go повторяет выражение 5 для b и c. Все три константы равны 5.

z находится в новом блоке const, где iota сбрасывается в 0.

Выражение iota * 2 повторяется: 0*2=0, 1*2=2, 2*2=4.

Резюме

Что Как Пример
Объявить константу const name = value const pi = 3.14
Группа констант const ( ... ) const ( a = 1; b = 2 )
Автонумерация iota const ( a = iota; b; c )
Пропуск значения _ = iota const ( _ = iota; first )
Выражение с iota iota + N или iota * N const ( a = iota + 5 )

Ключевые моменты:

  • const создаёт неизменяемое значение
  • В константах можно хранить числа, строки и bool
  • iota - счётчик, начинающийся с 0
  • iota увеличивается на 1 для каждой строки в блоке
  • iota сбрасывается в 0 в каждом новом блоке const
  • Пустой идентификатор _ пропускает ненужное значение
