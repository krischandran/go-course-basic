Строки (string)

Строка (от англ. string - нить, последовательность) - это текст, состоящий из символов. В Go строка записывается в двойных кавычках. Строка - это последовательность байтов: "Hi" состоит из двух байтов: 72 (код 'H') и 105 (код 'i').

Создание строк

Давай создадим первую строку:

Go
package main

import "fmt"

func main() {
    s := "Hello, Go!"
    fmt.Println(s)
}
Output
Hello, Go!

Строку можно объявить несколькими способами:

Go
package main

import "fmt"

func main() {
    // Короткое объявление
    name := "Анна"

    // Полное объявление
    var city string = "Москва"

    // Объявление с нулевым значением
    var empty string

    fmt.Println(name)  // Анна
    fmt.Println(city)  // Москва
    fmt.Println(empty) // (пустая строка)
}
Output
Анна
Москва

Нулевое значение для string - это пустая строка "". Она занимает 0 байт текста.

Конкатенация строк

Часто нужно склеить несколько строк в одну. Для этого используем оператор +:

Go
package main

import "fmt"

func main() {
    firstName := "Иван"
    lastName := "Петров"
    fullName := firstName + " " + lastName
    fmt.Println(fullName)
}
Output
Иван Петров

Такое склеивание называется конкатенацией (от англ. concatenation - связывание).

Порядок имеет значение. Посмотри:

Go
package main

import "fmt"

func main() {
    a := "Hello "
    b := "world!"

    fmt.Println(a + b) // Hello world!
    fmt.Println(b + a) // world!Hello
}
Output
Hello world!
world!Hello

При сложении чисел порядок не важен: 5 + 3 = 3 + 5. Со строками иначе: "Hello " + "world!" и "world!" + "Hello " дают разный результат.

Многострочный текст

Для текста с переносами строк используй обратные кавычки (клавиша с буквой Ё в английской раскладке):

Go
package main

import "fmt"

func main() {
    text := `Первая строка
Вторая строка
Третья строка`
    fmt.Println(text)
}
Output
Первая строка
Вторая строка
Третья строка

Обратные кавычки сохраняют текст как есть - со всеми переносами и пробелами.

Длина строки

Функция len() возвращает длину строки:

Go
package main

import "fmt"

func main() {
    s := "Hello, World!"
    fmt.Println(len(s))
}
Output
13

Пробел и знаки препинания тоже считаются. В строке "Hello, World!" 13 символов: 5 букв + запятая + пробел + 5 букв + восклицательный знак.

Но будь внимателен: len() считает байты, не буквы. Вот неожиданный результат:

Go
package main

import "fmt"

func main() {
    fmt.Println(len("Hi"))     // 2
    fmt.Println(len("Привет")) // 12
}
Output
2
12

Почему "Привет" - это 12, а не 6? Вспомни урок про byte и rune: русская буква не влезает в один байт. В UTF-8 разные символы занимают разное количество байтов:

Символы Байтов на символ Пример
Английские буквы, цифры1"Hi" = 2 байта
Русские буквы2"Привет" = 6 x 2 = 12 байт

Функция len() считает байты, а не буквы. Для английского текста это одно и то же (1 буква = 1 байт). Для русского - нет.

Доступ к символам по индексу

Символы в строке пронумерованы с нуля. Получить символ по номеру можно через квадратные скобки:

Go
package main

import "fmt"

func main() {
    s := "Hi"
    fmt.Println(s[0]) // 72 - код буквы H
    fmt.Println(s[1]) // 105 - код буквы i
}
Output
72
105

Индекс 0 - первый символ, индекс 1 - второй. Последний индекс = длина строки минус 1.

Обрати внимание: s[0] вернул 72, а не "H". Индексация возвращает байт - числовой код символа (вспомни урок про byte). Чтобы получить саму букву, преобразуй в строку:

Go
package main

import "fmt"

func main() {
    s := "Hi"
    fmt.Println(string(s[0])) // H
    fmt.Println(string(s[1])) // i
}
Output
H
i

Каждая буква имеет свой числовой код. Буква 'H' = 72, буква 'i' = 105. Компьютер работает с числами, не с буквами напрямую.

Ещё одна опасность - выход за границы. Если обратиться к индексу за пределами строки, программа упадёт:

Go
package main

import "fmt"

func main() {
    s := "Hi"
    // fmt.Println(s[2]) // Паника! Индекс 2 не существует
    fmt.Println(len(s)) // 2, значит допустимые индексы: 0 и 1
}

В строке "Hi" два байта. Допустимые индексы: 0 и 1. Индекс 2 вызовет панику (аварийное завершение программы).

Строки неизменяемы

Строки в Go нельзя изменить напрямую:

Go
s := "hello"
// s[0] = 'H'  // Ошибка компиляции!

Это ошибка компиляции - программа даже не запустится.

Почему так? Неизменяемость делает строки безопасными для передачи между частями программы. Если ты передал строку в функцию, можешь быть уверен - она не изменится.

Как тогда изменить букву? Через преобразование в другой тип. Как это сделать - в уроке про преобразования текстовых типов.

Сравнение строк

Строки можно сравнивать операторами == и !=:

Go
package main

import "fmt"

func main() {
    a := "HELLO"
    b := "Hello"

    fmt.Println(a == b) // false
    fmt.Println(a != b) // true
}
Output
false
true

Сравнение учитывает регистр: "HELLO" и "Hello" - разные строки. Буква 'H' и буква 'h' имеют разные коды.

Как строка хранится в памяти

Строка - это последовательность байтов. Вот как "Hi" выглядит в памяти:

+-----+-----+
| 72 | 105 | 2 байта
| 'H' | 'i' |
+-----+-----+

Каждый английский символ занимает 1 байт. Строка "Hi" = 2 байта.

А вот строка "Привет" в памяти:

+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+
| 208 | 159 | 209 | 128 | 208 | 184 | 208 | 178 | 208 | 181 | 209 | 130 |
| П | р | и | в | е | т |
+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+-----+
12 байт (каждая русская буква = 2 байта)

Каждая русская буква занимает 2 байта. Поэтому len("Привет") = 12, а s[0] = 208 (первый байт буквы П, не сама буква).

Это следствие кодировки UTF-8. UTF-8 (от Unicode Transformation Format, 8-bit) - способ кодирования символов, где разные символы занимают от 1 до 4 байтов.

Проверь себя

Вопросы

01Почему len("Go") = 2, а len("Го") = 4?
Показать ответ

len() считает байты. Английские буквы = 1 байт, русские = 2 байта. "Go" = 2 x 1 = 2, "Го" = 2 x 2 = 4.

02Что вернёт "Hello"[0] - букву или число?
Показать ответ

Число (код символа). Чтобы получить букву: string("Hello"[0]).

03Можно ли изменить символ в строке через s[0] = 'X'?
Показать ответ

Нет. Строки неизменяемы. Это ошибка компиляции.

04Чем отличаются двойные кавычки от обратных?
Показать ответ

Двойные "..." - обычная строка. Обратные `...` - многострочная строка с сохранением форматирования.

05Что произойдёт при обращении к s[10] для строки длиной 5?
Показать ответ

Паника (программа упадёт). Индекс за пределами строки.

06Что выведет код?
Go
a := "Go"
b := "Go"
fmt.Println(a == b)
Показать ответ

true - строки с одинаковым содержимым равны.
