Преобразование типов

Преобразование типов (type conversion) - это явное изменение типа значения. В отличие от многих языков, Go не выполняет автоматические преобразования между типами - даже между int и int64.

Преобразование между числовыми типами

Go строг: нельзя просто взять и присвоить int в int64. Это разные типы, даже если оба хранят целые числа.

Go
package main

import "fmt"

func main() {
    num := 200      // тип int
    var num64 int64

    // num64 = num  // ошибка! cannot use num (type int) as type int64
    num64 = int64(num)  // OK - явное преобразование

    fmt.Println(num64)
}
Output
200

Синтаксис преобразования прост: тип(значение). Ты как бы говоришь компилятору: "представь это значение как указанный тип".

Преобразование float в int

При преобразовании float в int дробная часть просто отбрасывается. Не округляется, а именно отбрасывается:

Go
package main

import "fmt"

func main() {
    price := 199.99
    rounded := int(price)
    fmt.Println(rounded)
}
Output
199

Важно понимать: 199.99 становится 199, а не 200. Если нужно округление - используй math.Round из пакета math.

Преобразование int в float

Обратное преобразование всегда безопасно - целое число точно представляется в float:

Go
package main

import "fmt"

func main() {
    count := 7
    average := float64(count) / 2
    fmt.Println(average)
}
Output
3.5

Если бы мы написали 7 / 2, результат был бы 3 (целочисленное деление). Преобразование в float64 даёт точный результат.

Переполнение при преобразовании

Что если положить большое число в маленький тип? Представь стакан: если налить больше, чем он вмещает, вода перельётся.

Go
package main

import "fmt"

func main() {
    num64 := int64(200)
    num8 := int8(num64)  // 200 не влезает в int8 (максимум 127)
    fmt.Println(num8)
}
Output
-56

Почему -56? Число "переливается" через край. int8 хранит от -128 до 127. Когда мы пытаемся положить 200:

  • 200 - 128 = 72 (насколько превышаем максимум)
  • Идём с другого края: -128 + 72 = -56

Go не выдаёт ошибку или предупреждение - это сделано для производительности. За переполнением следишь ты сам.

Go
package main

import "fmt"

func main() {
    var x int8 = 127
    x = x + 1  // переполнение в рантайме
    fmt.Println(x)
}
Output
-128

Компилятор не может проверить это на этапе компиляции, потому что значение вычисляется во время работы программы.

Преобразование строк и чисел

Для преобразования между строками и числами в Go есть пакет strconv (от string conversion - преобразование строк).

Строка в число: strconv.Atoi

Функция Atoi (ASCII to integer) преобразует строку в int:

Go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    str := "42"
    num, _ := strconv.Atoi(str)
    fmt.Println(num * 2)
}
Output
84

Функция возвращает два значения: число и ошибку. Мы пока игнорируем ошибку через _, но в реальном коде её нужно проверять - строка может содержать не число.

Число в строку: strconv.Itoa

Функция Itoa (integer to ASCII) преобразует int в строку:

Go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    num := 42
    str := strconv.Itoa(num)
    fmt.Println(str + str)  // конкатенация строк
}
Output
4242

Обрати внимание: 42 + 42 = 84, но "42" + "42" = "4242". Оператор + для строк означает склеивание, не сложение.

Почему Go строг к типам?

Почему Go не преобразует типы автоматически?

В языках вроде JavaScript число 5 + "3" даст "53". Удобно? Иногда. Но это источник багов.

Go выбрал явность: ты всегда знаешь, какой тип получишь. Если видишь int64(num), сразу понятно - тут преобразование. Никаких сюрпризов.

Почему нет предупреждения о переполнении?

Проверка переполнения при каждой операции замедляет программу. Go выбрал производительность. В большинстве программ переполнение не происходит - ты используешь int, который на 64-битной системе хранит огромные числа.

Если работаешь с int8 или int16 для экономии памяти - помни о границах. Но в обычном коде просто используй int.

Go
package main

import "fmt"

func main() {
    x := 7
    fmt.Println(x * x)
}

Результат: 49

02Километры в метры

Показать решение
Go
package main

import "fmt"

func main() {
    meters := 3450
    km := meters / 1000
    fmt.Println(km)
}

Результат: 3

03Раздели яблоки

У тебя 17 яблок и 5 друзей. Сколько яблок получит каждый? Сколько останется?

Показать решение
Go
package main

import "fmt"

func main() {
    apples := 17
    friends := 5
    each := apples / friends
    left := apples % friends
    fmt.Println("Каждому:", each)
    fmt.Println("Осталось:", left)
}

Результат: Каждому: 3, Осталось: 2

04Пересчёт времени

Дано 150 минут. Вычисли, сколько это полных часов и минут.

Показать решение
Go
package main

import "fmt"

func main() {
    minutes := 150
    hours := minutes / 60
    mins := minutes % 60
    fmt.Println(hours, "ч", mins, "мин")
}

Результат: 2 ч 30 мин

06Среднее с дробной частью

Найди среднее арифметическое чисел 5 и 4 с дробной частью (не целочисленное!).

KeyКлючевые моменты
  • Go требует явного преобразования типов - автоматически не преобразует
  • Синтаксис: тип(значение) - например, int64(num), float64(x)
  • При float -> int дробная часть отбрасывается (не округляется!)
  • При преобразовании в меньший тип возможно переполнение без предупреждения
  • strconv.Atoi(str) - строка в int
  • strconv.Itoa(num) - int в строку
  • Используй int по умолчанию - он оптимизирован и достаточно большой
