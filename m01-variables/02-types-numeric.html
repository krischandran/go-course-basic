Числовые типы в Go

Go предоставляет богатый набор числовых типов для разных задач. Понимание их особенностей — не просто теория, а практический навык для собеседований и написания надёжного кода.

Что спрашивают на собеседованиях

На собеседованиях по числовым типам любят давать "ловушки" — код, который ведёт себя неожиданно:

"Что будет, если к MaxInt64 прибавить 1?" — проверка на знание переполнения.

"Почему 0.1 + 0.2 != 0.3?" — проверка на понимание IEEE 754.

"Какой тип выбрать для ID пользователя? Для порта? Для денег?" — практический выбор типов.

Чтобы уверенно отвечать, нужно понимать как числа хранятся в памяти и почему Go ведёт себя именно так.

Как устроены числа

Целые: знаковый бит

У int8 первый бит определяет знак:

int8 = 127 (максимум):
0 1 1 1 1 1 1 1
0 = положительное число
int8 = -128 (минимум):
1 0 0 0 0 0 0 0
1 = отрицательное число

Почему -128...127, а не -128...128?

Всего 256 комбинаций (2⁸). Ноль занимает место среди "положительных":

  • Положительные + ноль: 0...127 = 128 штук
  • Отрицательные: -1...-128 = 128 штук

Float: почему 0.1 + 0.2 ≠ 0.3?

Как 1/3 = 0.333... (бесконечная дробь в десятичной), так 0.1 — бесконечная дробь в двоичной. Компьютер округляет, и накапливается погрешность:

Go 
fmt.Println(0.1 + 0.2 == 0.3)  // false!
fmt.Println(0.1 + 0.2)         // 0.30000000000000004

Вывод: сравнивай float через разницу:

Go 
const epsilon = 1e-9
equal := math.Abs(a - b) < epsilon  // ✓

Какой тип выбрать?

Простое правило: используй int, пока нет причины для другого.

Что храню Тип Почему
Обычные числа int Универсальный, по умолчанию
ID из базы данных int64 Может быть > 2 млрд
Сетевой порт uint16 Стандарт TCP: 0-65535
Деньги int64 В копейках, не в рублях!
Размер файла int64 Файлы бывают > 4 GB
Байты (RGB, данные) byte = uint8, 0-255
✓
Для свитчеров из Python/JS

Там числа "резиновые" и растут автоматически. В Go ты сам выбираешь размер. Но для 99% случаев int — правильный выбор.

Целые числа

int — платформозависимый

Go 
count := 42   // int — размер зависит от системы
age := 25     // int
  • На 32-битной системе: int = 4 байта (±2 млрд)
  • На 64-битной системе: int = 8 байт (огромный диапазон)

Для свитчеров из Java: там int всегда 32 бита. В Go int адаптируется под процессор.

Все целые типы

Знаковые (могут быть отрицательными):

Тип Байт Диапазон
int8 1 -128...127
int16 2 ±32 тысячи
int32 4 ±2.1 млрд
int64 8 ±9 квинтиллионов

Беззнаковые (только положительные):

Тип Байт Диапазон Пример
uint8 / byte 1 0...255 Пиксели, байты
uint16 2 0...65535 Порты
uint32 4 0...4.2 млрд IPv4
uint64 8 0...огромное Хеши
Go 
var port uint16 = 443    // порт
var pixel byte = 255     // byte = uint8

Запись чисел

Go 
// Подчёркивания для читаемости (компилятор игнорирует)
billion := 1_000_000_000

// Разные системы счисления
dec := 255        // десятичная
hex := 0xFF       // шестнадцатеричная (0x)
bin := 0b11111111 // двоичная (0b)

// Все три равны 255

Float (дробные числа)

Go 
price := 19.99  // float64 — по умолчанию, используй везде
Тип Точность Когда
float64 ~15 цифр Везде по умолчанию
float32 ~7 цифр GPU, ML (экономия памяти)

Ловушка: деньги

Go 
// ❌ НЕ храни деньги в float!
balance := 0.10 + 0.20
fmt.Println(balance == 0.30)  // false!

// ✅ Храни в копейках как int64
balanceCents := int64(10 + 20)  // 30 копеек
fmt.Printf("%.2f руб\n", float64(balanceCents)/100)

Ловушка: NaN

Go 
nan := math.NaN()
fmt.Println(nan == nan)  // false! NaN не равен ничему

// Проверяй через функцию
math.IsNaN(nan)  // true

Переполнение

!
Important

Go НЕ предупреждает о переполнении! Число молча "переворачивается".

Как одометр в машине: после 999999 км идёт 000000.

Go 
var x int8 = 127   // максимум для int8
x++
fmt.Println(x)     // -128 (!)

var y uint8 = 255  // максимум для uint8
y++
fmt.Println(y)     // 0 (!)

Как защититься?

Go 
// Проверяй ПЕРЕД операцией
if x < math.MaxInt8 {
    x++  // безопасно
}

Деление на ноль

Go 
// int / 0 → panic!
a := 10 / 0  // программа упадёт

// float / 0 → Infinity
x := 10.0 / 0.0  // +Inf
!
Important

Всегда проверяй делитель перед делением!

Проверь себя

01 Почему диапазон int8 от -128 до 127, а не -128 до 128?
Показать ответ

256 комбинаций делятся поровну: 128 отрицательных (-128...-1) и 128 "положительных" (0...127). Ноль занимает место среди положительных.

02 Почему 0.1 + 0.2 != 0.3?
Показать ответ

0.1 в двоичной — бесконечная дробь (как 1/3 = 0.333... в десятичной). Компьютер округляет → накапливается погрешность.

Решение: math.Abs(a-b) < epsilon

03 Что будет: var x uint8 = 0; x--?
Показать ответ

x = 255. Это underflow — как одометр: 000000 - 1 = 999999. Go не предупреждает!

04 Какой тип для денег?
Показать ответ

int64 в копейках/центах. НЕ float — там погрешности.

Go 
priceCents := int64(1999)  // 19.99 руб

Задачи для практики

01 Что выведет?
Go 
var x int8 = 120
x = x + 10
fmt.Println(x)
Показать решение

Вывод: -126

120 + 10 = 130, но максимум int8 = 127. Переполнение → 130 - 256 = -126

02 Найди баг
Go 
func average(nums []int) int {
    var sum uint32 = 0
    for _, n := range nums {
        sum += uint32(n)
    }
    return int(sum) / len(nums)
}
Показать решение

3 бага:

  • Деление на ноль если len(nums) == 0
  • Переполнение если сумма > 4.2 млрд
  • uint32(n) для отрицательного n даст мусор

Исправление: проверь len > 0, используй int64 вместо uint32

03 Исправь сравнение
Go 
func isRightTriangle(a, b, c float64) bool {
    return a*a + b*b == c*c  // не работает!
}
Показать решение

Float нельзя сравнивать через == — есть погрешность.

Go 
const epsilon = 1e-9

func isRightTriangle(a, b, c float64) bool {
    return math.Abs(a*a + b*b - c*c) < epsilon
}
04 Какой тип?

Выбери типы для: возраст, порт, ID заказа, цена товара.

Показать решение
Go 
age := 25              // int
var port uint16 = 443  // uint16 (0-65535)
var orderID int64      // int64 (BIGSERIAL > 2 млрд)
priceCents := int64(1999)  // int64 в копейках!

Резюме

Key Числовые типы в Go

Целые: int по умолчанию, int64 для ID/денег, uint16 для портов.

Float: float64 по умолчанию. Деньги — в копейках как int64!

Ловушки:

  • Переполнение — Go не предупреждает
  • 0.1 + 0.2 != 0.3 — сравнивай через epsilon
  • int/0 = panic, float/0 = Inf
Золотое правило: используй int и float64, пока нет причины для другого
