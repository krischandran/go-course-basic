Тип bool и нулевые значения

Эти две темы связаны: false - нулевое значение для bool. Понимание обеих - ключ к идиоматичному Go коду.

Что спрашивают на собеседованиях

"Что такое truthy/falsy и есть ли они в Go?" - проверка на понимание строгой типизации.

"Какое нулевое значение у string? А у bool?" - базовые знания типов.

"Можно ли использовать int в условии if?" - понимание отличий от других языков.

Тип bool

Go 
var isReady bool = true
var isDone = false
active := true

В Go нет truthy/falsy - только true и false

Это главное отличие от JavaScript и Python.

Go 
count := 5
name := "Go"

// JavaScript/Python:
// if count:    ✓ (5 - truthy)
// if name:     ✓ ("Go" - truthy)

// Go:
// if count { }  ❌ Ошибка компиляции!
// if name { }   ❌ Ошибка компиляции!

// Нужны явные сравнения (изучим в модуле "Условия"):
// if count > 0 { }      ✅
// if name != "" { }     ✅

Почему Go отказался от truthy/falsy?

В JavaScript if (value) может означать шесть разных вещей:

  • value не null?
  • value не undefined?
  • value не пустая строка?
  • value не 0?
  • value не false?
  • value не NaN?

Слишком много вариантов. Читая чужой код, ты гадаешь, что проверяется.

В Go всегда нужны явные сравнения: count > 0, name != "", isActive == true. Подробнее об условиях - в модуле "Условия".

Trade-off: больше символов, но меньше багов и больше ясности.

Логические операции

Go 
a := true
b := false

fmt.Println(a && b)  // false (AND - оба должны быть true)
fmt.Println(a || b)  // true (OR - хотя бы один true)
fmt.Println(!a)      // false (NOT - инвертирует)

Сравнение возвращает bool

Go 
x := 10
y := 20

isEqual := x == y    // false
isGreater := x > y   // false
isLess := x < y      // true

fmt.Println(isEqual)    // false
fmt.Println(isGreater)  // false
fmt.Println(isLess)     // true
✓
Tip

Как использовать bool в условных конструкциях (if, switch) - изучим в модуле "Условия".

Нулевые значения (Zero Values)

В Go каждая переменная всегда имеет значение. Если не инициализировал - там будет нулевое значение типа.

Таблица нулевых значений

Тип Нулевое значение
int, int8, int64...0
float32, float640.0
boolfalse
string"" (пустая строка)
Go 
var count int       // 0
var price float64   // 0.0
var name string     // ""
var active bool     // false

Почему это важно?

C/C++: неинициализированная переменная содержит "мусор" - случайные байты из памяти. Источник трудноуловимых багов.

JavaScript: переменная может быть undefined или null - два разных способа сказать "ничего".

Go: переменная всегда инициализирована. Никаких сюрпризов, никакого мусора.

Практический пример

Go 
var total int
var message string
var found bool

// Можно сразу использовать без инициализации
total = total + 10  // 0 + 10 = 10

fmt.Println(total)   // 10
fmt.Println(message) // "" (пустая строка)
fmt.Println(found)   // false

Проверка на "пустоту"

Для базовых типов проверка - это сравнение с нулевым значением:

Тип Проверка "пустоты"
intcount == 0
stringname == ""
boolactive == false или !active
✓
Tip

Нулевые значения для более сложных типов (указатели, срезы, карты) изучим в соответствующих модулях. Как использовать проверки в условиях (if) - в модуле "Условия".

Проверь себя

Вопросы

1 Почему в Go нельзя написать if count { } где count - это int?
Показать ответ

В Go условие if требует только значения типа bool. В отличие от JavaScript/Python, числа и строки не приводятся автоматически к bool (нет truthy/falsy). Нужно писать явное сравнение: if count > 0 { }.

2 Какое нулевое значение у типа string?
Показать ответ

Пустая строка "". Не nil, не null, не undefined - именно пустая строка нулевой длины.

3 Чем отличается подход Go к неинициализированным переменным от C/C++?
Показать ответ

В C/C++ неинициализированная переменная содержит "мусор" - случайные байты из памяти. В Go переменная всегда инициализируется нулевым значением своего типа. Это исключает целый класс багов.

4 Что выведет код: var b bool; fmt.Println(b)?
Показать ответ

Выведет false. Это нулевое значение для типа bool.

Задачи

1 Что выведет? 
var i int
var b bool
var s string

fmt.Println(i, b, s)
Показать ответ

Вывод: 0 false

Почему: Все переменные объявлены без инициализации, поэтому содержат нулевые значения: 0 для int, false для bool, "" (пустая строка) для string. Пустая строка не видна в выводе, но она там есть.

2 Что выведет? 
a := 5 > 3
b := 10 == 10
c := "Go" != "go"

fmt.Println(a, b, c)
Показать ответ

Вывод: true true true

Почему:

5 > 3 = true

10 == 10 = true

"Go" != "go" = true (строки сравниваются с учётом регистра)

3 Что выведет? 
a := true
b := false

fmt.Println(a && b)
fmt.Println(a || b)
fmt.Println(!b)
Показать ответ

Вывод:

false

true

true

Почему:

true && false = false (AND требует оба true)

true || false = true (OR - хотя бы один true)

!false = true (NOT инвертирует)

4 Что выведет? 
var a, b bool
c := true

fmt.Println(a && c)
fmt.Println(b || c)
fmt.Println(!a && c)
Показать ответ

Вывод:

false

true

true

Почему:

1. a && c = false && true = false (AND требует оба true)

2. b || c = false || true = true (OR - хотя бы один true)

3. !a && c = !false && true = true && true = true

Резюме

  • bool принимает только true или false
  • В Go нет truthy/falsy - всегда явные проверки
  • Каждый тип имеет нулевое значение: 0, 0.0, "", false
  • Переменные в Go всегда инициализированы - никакого "мусора"
  • Явность > неявность - код читается однозначно
