Модуль 1: Переменные

Тип bool и нулевые значения

Зачем компьютеру знать, что переменная хранит "да или нет"? Потому что это позволяет проверять условия: пользователь авторизован? товар в наличии? платёж прошёл? Тип bool хранит только два значения: true или false. А нулевые значения решают проблему "мусора в памяти" из C/C++.

Тип bool

Что это такое

bool хранит логическое значение: истина или ложь.

Go 
var isReady bool = true
var isDone = false
active := true

Зачем отдельный тип? В реальных программах постоянно нужны флаги:

Go 
var isLoggedIn bool    // пользователь вошёл?
var hasDiscount bool   // есть скидка?
var isAdmin bool       // это админ?

Как работает

В Go нет "правдивых" и "ложных" значений как в JavaScript или Python. Только true и false.

Попробуем написать так:

Go 
count := 5
// if count { }  // хотим проверить "count не ноль"

Компилятор выдаст ошибку:

Ошибка 
non-boolean condition in if statement

Почему? Go требует явного сравнения. Не "если count", а "если count больше нуля":

Go 
// Когда изучим if в модуле "Условия":
// if count > 0 { }     // явно: count больше нуля?
// if name != "" { }    // явно: name не пустая?
// if isReady { }       // уже bool - можно напрямую

Почему Go отказался от truthy/falsy

В JavaScript if (value) может означать шесть разных вещей: value не null? не undefined? не пустая строка? не 0? не false? не NaN?

Читая чужой код, ты гадаешь что проверяется. Go убрал эту неоднозначность: хочешь проверить - напиши явно.

Trade-off: больше символов в коде, но читается однозначно.

Логические операции

Go поддерживает три логических оператора:

Go 
a := true
b := false

fmt.Println(a && b)  // false - AND: оба должны быть true
fmt.Println(a || b)  // true  - OR: хотя бы один true
fmt.Println(!a)      // false - NOT: инвертирует
false
true
false

Операторы && и || читаются как "и" и "или". Оператор ! читается как "не".

Tip: Подробнее о логических операциях, приоритетах и коротком вычислении - в модуле "Условия".

Сравнение возвращает bool

Любое сравнение в Go возвращает bool:

Go 
x := 10
y := 20

isEqual := x == y    // false
isGreater := x > y   // false
isLess := x < y      // true

fmt.Println(isEqual, isGreater, isLess)
false false true

Нулевые значения

Что это такое

В Go каждая переменная всегда имеет значение. Если не инициализировал - там будет нулевое значение типа.

Go 
var count int       // 0
var price float64   // 0.0
var name string     // "" (пустая строка)
var active bool     // false

Как это работает

Вот таблица нулевых значений для базовых типов:

Тип Нулевое значение
int, int8, int64... 0
float32, float64 0.0
bool false
string "" (пустая строка)

Практический пример - можно сразу использовать переменную:

Go 
var total int
var message string

total = total + 10  // 0 + 10 = 10

fmt.Println(total)    // 10
fmt.Println(message)  // (пустая строка - ничего не видно)
10

Почему это важно

Проблема C/C++: неинициализированная переменная содержит "мусор" - случайные байты из памяти. Это источник трудноуловимых багов: программа может работать на твоём компьютере и падать на сервере.

Проблема JavaScript: переменная может быть undefined или null - два разных способа сказать "ничего".

Решение Go: переменная всегда инициализирована. Никаких сюрпризов.

Проверка на "пустоту"

Для базовых типов проверка - это сравнение с нулевым значением:

Тип Проверка
int count == 0
string name == ""
bool !active или active == false

Tip: Нулевые значения для сложных типов (указатели, срезы, карты) изучим в соответствующих модулях.

Связь bool и zero values

Заметил закономерность? Нулевое значение bool - это false. Это не случайно.

Go 
var found bool      // false - "пока не нашли"
var isValid bool    // false - "пока не проверили"
var hasError bool   // false - "ошибок нет"

По умолчанию флаги "выключены". Это удобно: не нужно явно писать = false.

Проверь себя

Вопросы

01Почему в Go нельзя написать проверку if count где count - это int?
Показать ответ

В Go условие требует значения типа bool. Числа и строки не приводятся к bool автоматически (нет truthy/falsy). Нужно явное сравнение: count > 0 или count != 0.

02Какое нулевое значение у типа string?
Показать ответ

Пустая строка "". Не nil, не null, не undefined - именно пустая строка нулевой длины.

03Чем подход Go к неинициализированным переменным отличается от C/C++?
Показать ответ

В C/C++ неинициализированная переменная содержит "мусор" - случайные байты из памяти. В Go переменная всегда инициализируется нулевым значением своего типа. Это исключает целый класс багов.

04Что выведет код: var b bool; fmt.Println(b)?
Показать ответ

Выведет false. Это нулевое значение для типа bool.

Задачи

01Определи вывод

Что выведет этот код?

Go 
var i int
var b bool
var s string

fmt.Println(i, b, s)
Показать решение

Вывод: 0 false

Все переменные объявлены без инициализации, поэтому содержат нулевые значения. Пустая строка не видна в выводе, но она там есть (после false идёт пробел и пустая строка).

02Что в переменных?

Определи значения переменных и вывод:

Go 
a := 5 > 3
b := 10 == 10
c := "Go" != "go"

fmt.Println(a, b, c)
Показать решение

Вывод: true true true

  • 5 > 3 - true (5 больше 3)
  • 10 == 10 - true (равны)
  • "Go" != "go" - true (строки разные, сравнение с учётом регистра)
03Логические операции

Что выведет этот код?

Go 
a := true
b := false

fmt.Println(a && b)
fmt.Println(a || b)
fmt.Println(!b)
Показать решение

Вывод:

false
true
true
  • true && false - false (AND требует оба true)
  • true || false - true (OR - хотя бы один true)
  • !false - true (NOT инвертирует)
04Нулевые значения в действии

Что выведет этот код?

Go 
var a, b bool
c := true

fmt.Println(a && c)
fmt.Println(b || c)
fmt.Println(!a && c)
Показать решение

Вывод:

false
true
true

Переменные a и b не инициализированы, значит равны false.

  1. false && true - false
  2. false || true - true
  3. !false && true = true && true = true

Резюме

  • bool хранит только true или false
  • В Go нет truthy/falsy - всегда явные сравнения
  • Каждый тип имеет нулевое значение: 0, 0.0, "", false
  • Переменные в Go всегда инициализированы - никакого "мусора" из памяти
  • Нулевое значение bool = false - флаги по умолчанию "выключены"
