Преобразование типов

Преобразование типов (type conversion) - это явное превращение значения одного типа в другой. Go не выполняет автоматические преобразования даже между родственными типами: нельзя присвоить int переменной int64 без явного указания.

Go предлагает два вида преобразований: между числовыми типами через синтаксис тип(значение) и между строками и числами через пакет strconv. Первое работает на уровне битов, второе - разбирает или собирает текстовое представление числа.

Преобразование между числовыми типами

Зачем вообще преобразовывать числа? Представь: у тебя есть переменная типа int, а функция принимает int64. Или нужно получить целую часть от float. Go требует явно сказать, что ты хочешь.

Синтаксис простой - пишешь нужный тип и в скобках значение:

package main

import "fmt"

func main() {
    num := 200         // тип int
    var num64 int64

    num64 = int64(num) // явное преобразование
    fmt.Println(num64) // 200
}
200

Мы сказали: "представь num как int64". Компилятор проверил, что значение влезает, и выполнил преобразование.

А что если попробовать без преобразования?

package main

import "fmt"

func main() {
    num := 200
    var num64 int64

    num64 = num // ошибка!
    fmt.Println(num64)
}

Go скажет: "cannot use num (type int) as type int64". Типы разные - нужно явное преобразование.

Преобразование float в int

При преобразовании вещественного числа в целое дробная часть просто отбрасывается. Не округляется, а именно отбрасывается:

package main

import "fmt"

func main() {
    price := 199.99
    rounded := int(price)
    fmt.Println(rounded) // 199
}
199

Число 199.99 стало 199, а не 200. Если нужно округление - используй math.Round из пакета math.

Обратное преобразование всегда безопасно:

package main

import "fmt"

func main() {
    count := 7
    average := float64(count) / 2
    fmt.Println(average) // 3.5
}
3.5

Без преобразования 7 / 2 дало бы 3 (целочисленное деление). Преобразование в float64 даёт точный результат.

Переполнение при преобразовании типов

При преобразовании большого типа в маленький может произойти переполнение (overflow). Как работает переполнение на уровне битов - подробно разобрали в уроке про int.

Вот пример с преобразованием типов:

package main

import "fmt"

func main() {
    num64 := int64(200)
    num8 := int8(num64)  // 200 не влезает в int8!
    fmt.Println(num8)    // -56
}
-56

Число 200 не влезает в int8 (максимум 127). Go не выдаёт ошибку - следить должен программист.

Символ по коду: string()

Функция string() с числом возвращает символ с этим кодом Unicode:

package main

import "fmt"

func main() {
    fmt.Println(string(65))   // A
    fmt.Println(string(1055)) // П
}
A
П

Число 65 - код буквы 'A' в таблице ASCII. Число 1055 - код русской 'П' в Unicode. Функция находит символ по коду и возвращает его как строку.

Это работает и с переменными типа byte и rune:

package main

import "fmt"

func main() {
    var b byte = 72
    var r rune = 1055

    fmt.Println(string(b)) // H
    fmt.Println(string(r)) // П
}
H
П

Код символа

Чтобы получить код символа, используй одинарные кавычки:

package main

import "fmt"

func main() {
    fmt.Println('A')  // 65
    fmt.Println('П')  // 1055
    fmt.Println('0')  // 48 (код символа '0', не число 0!)
}
65
1055
48

Одинарные кавычки возвращают числовой код символа. Тип результата - rune (int32).

Строка в байты и руны

Строку можно преобразовать в последовательность байтов или рун. Это полезно, когда нужно работать с отдельными символами текста.

Для английского текста подходит []byte():

package main

import "fmt"

func main() {
    s := "Hi"
    bytes := []byte(s)
    fmt.Println(bytes) // [72 105]
}
[72 105]

Каждый английский символ занимает 1 байт: 'H' = 72, 'i' = 105.

Но с русским текстом []byte даёт неожиданный результат - русская буква занимает 2 байта:

package main

import "fmt"

func main() {
    s := "Привет"
    bytes := []byte(s)
    fmt.Println(bytes) // 12 чисел, не 6!
}

Для любых символов используй []rune() - она корректно разбивает на символы Unicode:

package main

import "fmt"

func main() {
    s := "Привет"
    runes := []rune(s)
    fmt.Println(runes) // 6 чисел - по одному на букву
}
Подробнее: Операции с байтами и рунами (изменение, индексирование) изучим в модуле "Срезы".

Число и строка: strconv

Как превратить строку "42" в число 42? Или число 42 в строку "42"? Для этого есть пакет strconv (от string conversion - преобразование строк).

strconv.Atoi - строка в число

Atoi означает "ASCII to integer" - текст в целое число:

package main

import (
    "fmt"
    "strconv"
)

func main() {
    s := "42"
    n, _ := strconv.Atoi(s)

    fmt.Println(n + 1) // 43
}
43

Функция возвращает два значения: число и ошибку. Пока мы игнорируем ошибку через _. Об обработке ошибок - в модуле про функции.

strconv.Itoa - число в строку

Itoa означает "integer to ASCII" - целое число в текст:

package main

import (
    "fmt"
    "strconv"
)

func main() {
    n := 42
    s := strconv.Itoa(n)

    fmt.Println("Ответ: " + s) // Ответ: 42
}
Ответ: 42

Важно не путать!

string(число) и strconv.Itoa(число) - разные вещи:

package main

import (
    "fmt"
    "strconv"
)

func main() {
    n := 65

    fmt.Println(string(n))       // A (символ с кодом 65!)
    fmt.Println(strconv.Itoa(n)) // 65 (строка "65")
}
A
65

string(65) ищет символ с кодом 65 - это буква 'A'. А strconv.Itoa(65) превращает число 65 в строку "65".

Проверь себя

Вопросы

Вопрос 1: Можно ли в Go присвоить значение типа int переменной типа int64 напрямую? +

Нет. Нужно явное преобразование: int64(num).

Вопрос 2: Чему равно int(9.9) в Go? +

9 - дробная часть отбрасывается, не округляется.

Вопрос 3: Что произойдёт при выполнении int8(200)? +

Переполнение: результат будет -56. Go не предупреждает об этом.

Вопрос 4: Чем отличается string(65) от strconv.Itoa(65)? +

string(65) возвращает символ с кодом 65 - букву "A". strconv.Itoa(65) возвращает строку "65".

Вопрос 5: Почему []byte("Привет") содержит 12 элементов, а []rune("Привет") - только 6? +

[]byte разбивает строку на байты. Русская буква занимает 2 байта, поэтому 6 букв = 12 байтов. []rune разбивает на символы Unicode, поэтому 6 букв = 6 рун.

Вопрос 6: Какая функция преобразует строку "100" в число? +

strconv.Atoi(str) - возвращает число и ошибку.

Задачи

Задача 1: Что выведет? + 
fmt.Println(string(72))

H - символ с кодом 72.

Задача 2: Что выведет? + 
fmt.Println(int(7.9) + int(2.1))

9 - дробная часть отбрасывается: int(7.9)=7, int(2.1)=2, сумма 7+2=9.

Задача 3: Что выведет? + 
n, _ := strconv.Atoi("100")
fmt.Println(n + 23)

123 - строка "100" превратилась в число 100, плюс 23 = 123.

Задача 4: Что выведет? + 
fmt.Println(string(65) + string(66))

AB - два символа склеились в строку.

Задача 5: Что выведет? + 
var x int8 = 127
x = x + 1
fmt.Println(x)

-128 - переполнение int8: 127 + 1 переходит на -128.

Задача 6: Что выведет? + 
fmt.Println(strconv.Itoa(7) + strconv.Itoa(3))

73 - строки "7" и "3" склеились (конкатенация, не сложение!).

Резюме

  • Go требует явного преобразования типов - тип(значение)
  • При float -> int дробная часть отбрасывается (не округляется)
  • При преобразовании в меньший тип возможно переполнение без предупреждения
  • string(код) - символ по коду Unicode
  • strconv.Atoi(s) - строка в int
  • strconv.Itoa(n) - int в строку
  • []byte(s) - для английского текста
  • []rune(s) - для любого текста (Unicode)
