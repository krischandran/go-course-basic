Целые числа

Целые числа - это числа без дробной части: 1, 42, -7, 0. Go предлагает целое семейство целочисленных типов: int и uint с размером, зависящим от платформы, плюс варианты с фиксированным размером - int8, int16, int32, int64 и их беззнаковые аналоги.

Тип int

В Go тип переменной фиксируется при создании и не меняется. Нельзя записать число в строковую переменную - компилятор остановит ещё до запуска программы. Это позволяет ловить ошибки на этапе компиляции, а не в продакшене.

Для целых чисел в Go есть тип int (от английского integer - целое число):

Go
package main

import "fmt"

func main() {
    age := 25
    fmt.Println(age)
}
Output
25

Переменная age автоматически получила тип int. Это самый удобный вариант для хранения целых чисел.

Размер int зависит от системы. На 32-битной платформе int занимает 4 байта, на 64-битной - 8 байт. У нас практически у всех 64-битная система, поэтому можем спокойно хранить очень большие числа.

Если нужен конкретный размер, используй типы с числом в названии. Цифры означают количество бит:

ТипРазмерДиапазон
int81 байт (8 бит)от -128 до 127
int162 байта (16 бит)от -32768 до 32767
int324 байта (32 бита)примерно +/- 2 миллиарда
int648 байт (64 бита)очень большие числа

Зачем нужны эти варианты? Представь, ты хочешь сэкономить память. У тебя число от 0 до 100, и выделять 8 байт для такого маленького числа как-то не хочется. Можно использовать int8 - он занимает всего 1 байт.

Go
package main

import "fmt"

func main() {
    var small int8 = 100
    fmt.Println(small)
}
Output
100

Но в обычных программах эта экономия будет "на спичках". Можно спокойно использовать int почти всегда.

Почему int8 хранит от -128 до 127, а не от -128 до 128? Давай разберёмся.

int8 занимает 8 бит. Один бит уходит на знак (плюс или минус), остаётся 7 бит для значения:

7 бит = 2^7 = 128 возможных значений Отрицательные: -128, -127, ..., -1 (128 значений) Неотрицательные: 0, 1, ..., 127 (128 значений, включая ноль)

Ноль считается положительным числом, поэтому положительных чисел на одно меньше: от 0 до 127.

Беззнаковые типы (uint)

Что если число никогда не будет отрицательным? Количество пользователей, размер файла - эти значения не могут быть меньше нуля.

Для этого есть uint (unsigned integer - беззнаковое целое):

Go
package main

import "fmt"

func main() {
    var users uint = 1500
    fmt.Println(users)
}
Output
1500

Беззнаковые типы хранят только положительные числа и ноль, зато их максимум в два раза больше:

ТипРазмерДиапазон
uint81 байтот 0 до 255
uint162 байтаот 0 до 65535
uint324 байтаот 0 до ~4 миллиардов
uint648 байточень большие числа

Сравни: int8 хранит от -128 до 127, а uint8 - от 0 до 255. Один бит, который шёл на знак, теперь используется для значения.

Бит знака больше не нужен - все 8 бит идут на значение. 2^8 = 256 возможных значений: от 0 до 255.

Нулевое значение

Если объявить переменную без начального значения, она получит нулевое значение. Для всех числовых типов это 0:

Go
package main

import "fmt"

func main() {
    var count int
    var price uint
    fmt.Println(count, price)
}
Output
0 0

Go гарантирует: переменная никогда не содержит "мусор". Всегда есть определённое начальное значение.

Подчёркивание для читаемости

Большие числа сложно читать. Сравни:

Go
population := 7900000000

и

Go
population := 7_900_000_000

Go позволяет использовать подчёркивание как разделитель разрядов. На результат это не влияет, зато читать удобнее - сразу видно, сколько миллионов и миллиардов.

Операции с целыми числами

Go поддерживает стандартные арифметические операции:

ОператорДействиеПример
+сложение5 + 3 = 8
-вычитание5 - 3 = 2
*умножение5 * 3 = 15
/деление5 / 3 = 1
%остаток5 % 3 = 2

Эти операторы называются бинарными (от лат. binarius - двойной), потому что работают с двумя значениями. Значения слева и справа от оператора называются операндами (от лат. operandum - то, над чем выполняется операция). В выражении 5 + 3 операнды - это 5 и 3.

Операции +, -, * работают как в математике. А вот с делением есть нюанс.

Целочисленное деление

При делении двух целых чисел результат тоже целое число - дробная часть отбрасывается:

Go
package main

import "fmt"

func main() {
    fmt.Println(12 / 3)
    fmt.Println(12 / 5)
    fmt.Println(12 / 14)
}
Output
4
2
0

Почему 12 / 14 равно 0? Потому что 12 меньше 14, и целая часть от деления равна нулю.

Остаток от деления

Оператор % возвращает остаток от деления:

Go
package main

import "fmt"

func main() {
    fmt.Println(12 % 3)
    fmt.Println(12 % 5)
    fmt.Println(12 % 14)
}
Output
0
2
12

Если делимое меньше делителя, остаток равен самому делимому: 12 % 14 = 12.

Если делимое делится нацело, остаток равен 0: 12 % 3 = 0.

Приоритет операций

Как в математике: сначала умножение и деление, потом сложение и вычитание:

Go
package main

import "fmt"

func main() {
    result := 2 + 2 * 2
    fmt.Println(result)
}
Output
6

Для изменения порядка используй скобки: (2 + 2) * 2 = 8.

Сокращённые операторы

Часто нужно изменить переменную и записать результат в неё же:

Go
package main

import "fmt"

func main() {
    price := 100

    price += 50  // то же что price = price + 50
    fmt.Println(price)

    price -= 30  // то же что price = price - 30
    fmt.Println(price)
}
Output
150
120

Есть также *=, /= и %=.

Инкремент и декремент

Для увеличения или уменьшения на единицу есть специальные операторы:

Go
package main

import "fmt"

func main() {
    count := 5
    count++
    fmt.Println(count)

    count--
    fmt.Println(count)
}
Output
6
5

Важная особенность Go: операторы ++ и -- изменяют значение переменной, но не возвращают результат. Нельзя написать fmt.Println(count++) или y := x++ - это вызовет ошибку компиляции.

Go
count++            // OK
fmt.Println(count) // OK

// fmt.Println(count++) // ошибка!
// y := x++             // ошибка!

Унарный минус

Унарный оператор (от лат. unus - один) работает с одним значением, в отличие от бинарных (+, -, *, /), которые работают с двумя.

Унарный минус меняет знак числа:

Go
package main

import "fmt"

func main() {
    a := 5
    b := -a
    fmt.Println(b)

    c := -10
    d := -c
    fmt.Println(d)
}
Output
-5
10

Не путай унарный минус -a с бинарным вычитанием a - b. Первый работает с одним значением, второй - с двумя.

Переполнение: что происходит на границе?

Что будет, если прибавить 1 к максимальному значению int8?

Go
package main

import "fmt"

func main() {
    var x int8 = 127
    x = x + 1
    fmt.Println(x)
}
Output
-128

Почему -128? Разберём по шагам.

int8 - это 8 бит. Число 127 в битах выглядит так: 01111111. Первый бит (0) означает "положительное".

Прибавляем 1:

01111111 (127) + 1 ---------- 10000000 (-128)

Первый бит стал 1 - теперь это отрицательное число. В двоичном представлении 10000000 = -128.

Вот как устроены числа int8 - они образуют замкнутый круг:

0 -1 1 -2 2 . . . . -127 127 -128 После 127 идёт -128 После -128 идёт -127

Это называется переполнение (overflow). Go не предупреждает о нём - проверка каждой операции замедлила бы программу.

С обычным int проблем не будет (диапазон в миллиарды). Но если используешь int8 для экономии памяти - помни о границах.

Напиши программу, которая вычисляет квадрат числа 7 и выводит результат.

Go

import "fmt"

    fmt.Println(x * x)

Переведи 3450 метров в полные километры. Выведи результат.

Go
package main

import "fmt"

func main() {
    meters := 3450
    km := meters / 1000
    fmt.Println(km)
}

Результат: 3

03Раздели яблоки

У тебя 17 яблок и 5 друзей. Сколько яблок получит каждый? Сколько останется?

Показать решение
Go
package main

import "fmt"

func main() {
    apples := 17
    friends := 5
    each := apples / friends
    left := apples % friends
    fmt.Println("Каждому:", each)
    fmt.Println("Осталось:", left)
}

Результат:

Output
Каждому: 3
Осталось: 2
04Пересчёт времени

Дано 150 минут. Вычисли, сколько это полных часов и минут.

Показать решение
Go
package main

import "fmt"

func main() {
    minutes := 150
    hours := minutes / 60
    mins := minutes % 60
    fmt.Println(hours, "ч", mins, "мин")
}

Результат: 2 ч 30 мин

Output

10

Три товара стоят 199, 249 и 179 рублей. Вычисли общую сумму.

  • int - основной тип для целых чисел, размер зависит от системы
  • int8, int16, int32, int64 - фиксированный размер (цифра = биты)
  • uint - беззнаковые (только положительные и ноль)

  • Целочисленное деление отбрасывает дробную часть: 5 / 2 = 2
  • Оператор % даёт остаток от деления
  • ++ и -- не возвращают значение - только отдельной строкой
  • Переполнение происходит без предупреждения
  • Подчёркивание _ улучшает читаемость больших чисел
