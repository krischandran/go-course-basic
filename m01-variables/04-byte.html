Байты (byte)

Байт (от англ. byte - единица информации из 8 бит) - это тип для работы с отдельными символами: английскими буквами, цифрами, знаками препинания.

Как это работает? Компьютер не понимает буквы напрямую - он работает только с числами. Каждому символу присвоен числовой код от 0 до 255. Буква 'A' = 65, буква 'a' = 97, цифра '0' = 48. Тип byte хранит именно эти коды. Эта система кодирования называется ASCII (American Standard Code for Information Interchange) - таблица из 128 символов: только английские буквы, цифры и базовые знаки. Других языков и эмодзи в ASCII нет - для них нужен Unicode (об этом в уроке про rune).

Зачем это нужно? Понимание кодов символов открывает возможности: превратить 'A' в 'B' (просто прибавить 1), проверить заглавная ли буква (сравнить коды), извлечь цифру из символа ('5' - '0' = 5). Это основа работы с текстом на низком уровне.

Что такое byte

Тип byte - это алиас (от англ. alias - другое имя) для uint8. А что такое uint8? Это unsigned int 8 bit - беззнаковое целое число размером 8 бит, хранит значения от 0 до 255. byte и uint8 - один и тот же тип с двумя именами:

Go
package main

import "fmt"

func main() {
    var b byte = 72
    fmt.Println(b)         // 72
    fmt.Println(string(b)) // H
}
Output
72
H

Число 72 - это код буквы 'H' в таблице ASCII. Функция string() преобразует код обратно в символ.

Зачем два имени? Разница в намерении:

  • uint8 говорит: "здесь просто число от 0 до 255"
  • byte говорит: "здесь код символа"

Вот почему byte удобнее для работы с текстом:

Go
package main

import "fmt"

func main() {
    var code uint8 = 65  // Непонятно - это что?
    var letter byte = 65 // Понятно - это символ

    fmt.Println(string(code))   // A
    fmt.Println(string(letter)) // A
}
Output
A
A

Результат одинаковый, но byte сразу говорит читателю кода: "это символ".

Таблица ASCII и одинарные кавычки

Как узнать код символа, не заглядывая в таблицу? В Go одинарные кавычки возвращают код символа:

Go
package main

import "fmt"

func main() {
    fmt.Println('A')     // 65
    fmt.Println('a')     // 97
    fmt.Println('0')     // 48
    fmt.Println('A' + 1) // 66 (код 'B')
}
Output
65
97
48
66

Одинарные кавычки 'A' - это не строка, а число 65. Двойные кавычки "A" - это строка.

ASCII (American Standard Code for Information Interchange) - таблица, где каждому символу присвоен номер. Вот основные диапазоны:

СимволыКодыПример
Цифры '0'-'9'48-57'0' = 48, '5' = 53
Заглавные 'A'-'Z'65-90'A' = 65, 'Z' = 90
Строчные 'a'-'z'97-122'a' = 97, 'z' = 122

Важное свойство: символы идут подряд. Буква 'b' на 1 больше 'a'. Цифра '5' на 5 больше '0'. Это позволяет делать арифметику с символами.

Когда объявляешь byte через символ, Go автоматически подставляет код:

Go
package main

import "fmt"

func main() {
    var b byte = 'H'       // То же самое что var b byte = 72
    fmt.Println(b)         // 72
    fmt.Println(string(b)) // H
}
Output
72
H

Писать 'H' удобнее, чем запоминать число 72.

Ограничение byte: только ASCII

byte хранит числа от 0 до 255, но ASCII использует только коды 0-127. Что насчёт остальных символов?

Русские буквы, китайские иероглифы, эмодзи - всё это НЕ влезает в byte. Русская буква 'П' имеет код 1055 - это больше 255.

Go
package main

import "fmt"

func main() {
    // var b byte = 'П'  // Ошибка! 1055 не влезает в byte
    fmt.Println('П')      // 1055 - код русской буквы
    fmt.Println('A')      // 65 - код английской буквы
}
Output
1055
65

byte подходит только для английского текста, цифр и базовых символов. Для работы с любыми символами мира есть тип rune - он хранит коды до 1 114 111.

Арифметика с байтами

Раз символы идут подряд в таблице, можно делать арифметику. Хочешь следующую букву? Прибавь 1:

Go
package main

import "fmt"

func main() {
    var b byte = 'a'
    fmt.Println(string(b))     // a
    fmt.Println(string(b + 1)) // b
    fmt.Println(string(b + 2)) // c
    fmt.Println(string(b + 3)) // d
}
Output
a
b
c
d

Код 'a' = 97. Добавляем 1, получаем 98 = 'b'. Добавляем 2, получаем 99 = 'c'.

Получение цифры из символа

Часто нужно превратить символ цифры в число. Например, '5' превратить в 5. Как это сделать?

Символы цифр идут подряд: '0' = 48, '1' = 49, ..., '9' = 57. Значит, разница между любой цифрой и '0' даёт само число:

Go
package main

import "fmt"

func main() {
    digit := '5'
    number := digit - '0'
    fmt.Println(number) // 5
}
Output
5

'5' - '0' = 53 - 48 = 5. Вычитая код нуля, получаем само число.

Это работает для любой цифры:

Go
package main

import "fmt"

func main() {
    fmt.Println('0' - '0') // 0
    fmt.Println('3' - '0') // 3
    fmt.Println('9' - '0') // 9
}
Output
0
3
9

Сравнение символов

Можно ли сравнивать символы? Да, сравниваются их коды:

Go
package main

import "fmt"

func main() {
    fmt.Println('a' < 'b') // true (97 < 98)
    fmt.Println('A' < 'a') // true (65 < 97)
    fmt.Println('0' < '9') // true (48 < 57)
}
Output
true
true
true

Заглавные буквы "меньше" строчных, потому что их коды меньше: A-Z (65-90) идут раньше a-z (97-122).

Почему разница между 'A' и 'a' всегда 32? Это не случайность:

'A' = 65 = 01000001
'a' = 97 = 01100001
Отличается 1 бит (разница = 32)

ASCII спроектирована для быстрого переключения регистра - достаточно изменить один бит.

Показать ответ

Ничем технически. byte - алиас к uint8. Разница в намерении: byte говорит "это символ", uint8 - "это число".

02Почему 'A' меньше 'a' при сравнении?
Показать ответ

Потому что сравниваются коды: 'A' = 65, 'a' = 97. 65 < 97.

03Что вернёт выражение '5' - '0'?
Показать ответ

Число 5. Код '5' = 53, код '0' = 48. 53 - 48 = 5.

04Чем отличаются "A" и 'A' в Go?
Показать ответ

"A" - строка (тип string). 'A' - код символа (тип int32/rune, значение 65).

05Почему русская буква 'П' не влезает в byte?
Показать ответ

Код 'П' = 1055, а byte вмещает только 0-255. Для русских букв нужен rune.
