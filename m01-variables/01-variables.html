Переменные в Go

На собеседованиях по переменным редко задают прямые вопросы - обычно дают код и просят найти ошибку или предсказать поведение. Чтобы уверенно отвечать, нужно понимать как Go работает с памятью.

"Что выведет этот код?" - и дальше shadowing или переопределение переменной.

Что такое переменная на самом деле

Переменная - это именованная область памяти. Когда ты пишешь x := 42, происходит следующее:

  1. Go выделяет участок памяти (для int - 8 байт на 64-bit системе)
  2. Записывает туда значение 42
  3. Связывает имя x с адресом этой памяти
Память:
...
42 8 байт
↑ имя "x"
...

Когда ты пишешь fmt.Println(x), Go идёт по адресу, связанному с x, и читает значение оттуда.

Почему это важно понимать?

Потому что многие ошибки новичков связаны с непониманием того, что именно копируется при передаче переменных. Когда ты передаёшь x в функцию - копируется значение, а не сама переменная:

Go 
func double(n int) {
    n = n * 2  // меняем копию, не оригинал!
}

func main() {
    x := 10
    double(x)
    fmt.Println(x)  // 10 - не изменился!
}

В Go всё передаётся по значению (by value). Это фундаментальный принцип языка.

Статическая типизация

Go - статически типизированный язык. Это значит, что тип переменной:

  1. Определяется при объявлении
  2. Не может измениться за всё время жизни переменной
  3. Проверяется на этапе компиляции, а не выполнения

Сравнение с динамической типизацией

Python 
# Python: переменная как ярлык
x = 42          # x указывает на число
x = "hello"     # теперь на строку
x = [1, 2, 3]   # теперь на список
# Всё работает
Go 
// Go: переменная - это типизированная память
x := 42          // выделено 8 байт под int
// x = "hello"   // Ошибка: строка не влезет в 8 байт int'а!
!
Important

Go выбрал статическую типизацию ради надёжности и скорости. Если код скомпилировался - целый класс ошибок уже исключён.

Объявление переменных: три способа

Способ 1: var с типом

Go 
var age int = 25
var name string = "Alice"
var isActive bool = true

Когда использовать: когда хочешь явно указать тип, особенно если он отличается от выведенного автоматически:

Go 
var bigNumber int64 = 100  // без int64 будет просто int
var smallFloat float32 = 3.14  // без float32 будет float64

Способ 2: var с выводом типа

Go 
var age = 25        // int (выведен автоматически)
var name = "Alice"  // string
var pi = 3.14       // float64 (по умолчанию для дробных)

Когда использовать: на уровне пакета (вне функций), где := не работает.

Способ 3: короткое объявление :=

Go 
age := 25
name := "Alice"
isActive := true

Оператор := делает две вещи сразу:

  1. Создаёт новую переменную
  2. Присваивает ей значение

Способ Где работает Когда

var x = 10 Везде На уровне пакета

Нулевые значения

В Go каждая переменная автоматически инициализируется нулевым значением своего типа. Никакого "мусора" из памяти:

Go 
var i int      // 0
var f float64  // 0.0
var s string   // "" (пустая строка)
var b bool     // false
var p *int     // nil

Таблица нулевых значений

Тип Нулевое значение
int, int8, int64... 0
float32, float64 0.0
bool false
string ""
Указатели, слайсы, map, chan nil

Изменение значений: = vs :=

Это частый источник ошибок у новичков. Запомни:

  • := - создаёт новую переменную
  • = - изменяет существующую
Go 
x := 10      // создаём x со значением 10
x = 20       // меняем значение на 20
// x := 30   // ошибка: x уже существует!

Исключение: новые переменные в группе

Если хотя бы одна переменная в := новая - всё работает:

Go 
x := 10
x, y := 20, 30  // OK - y новая, x переиспользуется

Область видимости (scope)

Переменная "живёт" только в том блоке, где объявлена:

Go 
func main() {
    x := 10  // x видна во всей функции main

    if true {
        y := 20  // y видна только внутри if
        fmt.Println(x, y)  // OK
    }

    // fmt.Println(y)  // ошибка: y не определена
}

Shadowing - главная ловушка

*
Tip

Мнемоника "ТЕНЬ": Переменная во внутреннем блоке Так же названа - Её Не видно из внешнего, она в тени!

Что такое shadowing?

Shadowing (затенение) - когда переменная во внутреннем блоке "перекрывает" переменную с тем же именем из внешнего:

Go 
x := 10
fmt.Println(x)  // 10

if true {
    x := 20  // Это НОВАЯ переменная x!
    fmt.Println(x)  // 20
}

fmt.Println(x)  // 10 (!) - внешняя x не изменилась

Типичная ошибка с err

Go 
func getConfig() (Config, error) {
    cfg := Config{Default: true}

    if needsDB {
        cfg, err := loadFromDB()  // Создали НОВУЮ cfg!
        if err != nil {
            return Config{}, err
        }
        // cfg из if - не та же, что cfg из функции
    }

    return cfg  // Возвращается пустая Config{}!
}

Как исправить?

Go 
func getConfig() (Config, error) {
    cfg := Config{Default: true}

    if needsDB {
        var err error
        cfg, err = loadFromDB()  // Используем внешнюю cfg
        if err != nil {
            return Config{}, err
        }
    }

    return cfg  // Теперь правильно!
}

Проверь себя

01 Что такое переменная на уровне памяти?
Показать ответ

Переменная - это именованная область памяти определённого размера. Имя переменной связано с адресом в памяти, где хранится значение.

02 В чём разница между var x = 10 и x := 10?
Показать ответ

Функционально - разницы нет, обе создают переменную x типа int со значением 10.

Различия:

  • var работает везде (и в функциях, и на уровне пакета)
  • := работает только внутри функций
03 Что такое shadowing и почему это опасно?
Показать ответ

Shadowing - когда переменная во внутреннем блоке "перекрывает" переменную с тем же именем из внешнего.

Опасно тем, что изменения во внутреннем блоке не влияют на внешнюю переменную.

Задачи

01 Что выведет?
Go 
func main() {
    x := 10
    if true {
        x := 20
        x = 30
    }
    fmt.Println(x)
}
Показать решение

Вывод: 10

Разбор:

  1. x := 10 - создаём x = 10 в main
  2. x := 20 - создаём новую x = 20 внутри if (shadowing!)
  3. x = 30 - меняем внутреннюю x на 30
  4. После if внутренняя x уничтожается
  5. fmt.Println(x) - выводим внешнюю x, которая всё ещё 10
02 Что выведет?
Go 
func main() {
    x := 1
    x, y := 2, 3
    fmt.Println(x, y)
}
Показать решение

Вывод: 2 3

Разбор:

  1. x := 1 - создаём x = 1
  2. x, y := 2, 3 - y новая, поэтому := разрешён. x переиспользуется и получает 2

Это исключение из правила "нельзя объявить повторно" - если хотя бы одна переменная новая.
