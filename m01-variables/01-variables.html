Переменные в Go

Переменная - это именованный ящик для хранения данных. Научимся создавать переменные тремя способами и понимать, как Go работает с памятью.

Что такое переменная

Переменная - это именованный ящик для хранения данных. Представь коробку с этикеткой: положил что-то внутрь - можешь достать по имени.

У каждой переменной три характеристики:

  • Имя - этикетка на ящике (age, userName, price)
  • Тип - что можно положить (int для чисел, string для текста)
  • Значение - что лежит внутри прямо сейчас

Вот минимальный рабочий пример:

Go
package main

import "fmt"

func main() {
    userName := "Олег"
    age := 25
    fmt.Println(userName, age)  // Олег 25
}

Две переменные: userName хранит имя пользователя, age - его возраст. Обе созданы через :=.

Зачем нужны переменные? Программа без переменных может только выводить один и тот же результат. С переменными ты работаешь с разными данными: сегодня обрабатываешь заказ на 1500 рублей, завтра - на 3200.

Как работать с переменными

Три способа объявления

Go предлагает три способа создать переменную. Разберём каждый.

Способ 1: короткое объявление :=

Самый частый способ внутри функций. Пишешь имя, двоеточие с равно, значение:

Go
age := 25              // int (от integer - целое число)
userName := "Олег"     // string (строка текста)
isActive := true       // bool (от boolean - логическое значение)
price := 199.99        // float64 (от floating point - число с плавающей точкой)

Оператор := делает две вещи сразу: создаёт переменную и кладёт в неё значение. Тип Go определяет автоматически по тому, что справа.

Способ 2: var с выводом типа

Тип определяется автоматически, но пишем ключевое слово var:

Go
var age = 25           // int
var userName = "Олег"  // string
var price = 199.99     // float64

Зачем нужен var, если есть :=? Короткое объявление := работает только внутри функций. На уровне пакета нужен var:

Go
package main

var appVersion = "1.0.0"  // var - на уровне пакета
// appVersion := "1.0.0"  // ошибка: := не работает вне функций

func main() {
    userName := "Олег"  // := - внутри функции, OK
}

Способ 3: var с явным типом

Иногда нужно указать тип явно. Например, когда хочешь конкретный размер числа:

Go
var userID int64 = 9223372036854775807  // без int64 было бы просто int
var temperature float32 = 36.6          // без float32 было бы float64

Сравнение способов:

Способ Где работает Когда использовать
x := 10 Только в функциях Локальные переменные (90% случаев)
var x = 10 Везде На уровне пакета
var x int = 10 Везде Нужен конкретный тип

Изменение значений: = vs :=

Это важное различие. Запомни:

  • := создаёт новую переменную
  • = изменяет существующую
Go
price := 100     // создаём price = 100
price = 150      // меняем на 150
// price := 200  // ошибка: price уже существует!

Почему Go запрещает повторное :=? Защита от опечаток. Если случайно написать := вместо =, компилятор поймает ошибку. Иначе ты бы создал новую переменную вместо изменения старой и долго искал баг.

Но есть исключение. Если хотя бы одна переменная в группе новая - всё работает:

Go
price := 100
price, discount := 150, 10  // OK - discount новая, price переиспользуется

Это удобно при работе с функциями, которые возвращают несколько значений:

Go
data, err := loadConfig("config.json")
// ... обработка ...
settings, err := loadConfig("settings.json")  // OK - settings новая

Множественное объявление

Go позволяет работать с несколькими переменными одновременно:

Go
// Объявление нескольких переменных
width, height := 1920, 1080

// Обмен значений - элегантно в одну строку!
a, b := 1, 2
a, b = b, a  // теперь a=2, b=1

Как работает обмен? Go сначала вычисляет ВСЕ значения справа (b, a = 2, 1), потом присваивает их слева. Поэтому не нужна временная переменная.

Для группировки связанных переменных на уровне пакета используй var ():

Go
var (
    dbHost = "localhost"
    dbPort = 5432
    dbUser = "admin"
)

Правила именования

Несколько правил для имён переменных:

  1. Только латинские буквы a-z, A-Z, цифры, подчёркивание _
  2. Имя не может начинаться с цифры
  3. Go регистрочувствителен: name и Name - разные переменные
  4. Имя должно отражать назначение
Go
// Хорошие имена - понятно что хранят
userName := "Олег"
orderTotal := 1500
isDelivered := false

// Плохие имена - непонятно
x := "Олег"       // что это?
n := 1500         // какое число?
flag := false     // какой флаг?

Почему Go работает именно так

Статическая типизация

Go - статически типизированный язык (static typing). Это значит:

  1. Тип переменной определяется при создании
  2. Тип не меняется за всё время жизни переменной
  3. Компилятор проверяет типы ДО запуска программы
Go
price := 100        // price имеет тип int
// price = "сто"    // ошибка компиляции: нельзя присвоить string к int

Зачем такие ограничения?

Производительность. Компилятор знает точный размер каждой переменной и размещает их в памяти оптимально.

Безопасность. Ошибки типов ловятся при компиляции, а не у пользователя в продакшене.

Читаемость. Тип переменной понятен из кода.

i

Для знакомых с Python/JS: там переменная может хранить что угодно - сегодня число, завтра строку. В Go тип фиксирован. Это не ограничение, а защита от целого класса ошибок.

Как устроена память

Когда пишешь price := 100, Go делает следующее:

  1. Выделяет участок памяти (для int - 8 байт на 64-bit системе)
  2. Записывает туда значение 100
  3. Связывает имя price с адресом этой памяти

Вот как это выглядит:

Память:
...
100
...
↑ имя "price"

Важный момент: в Go переменные копируются при присваивании:

Go
originalPrice := 100
discountedPrice := originalPrice  // копия значения
discountedPrice = 80              // меняем копию

fmt.Println(originalPrice)    // 100 - не изменился!
fmt.Println(discountedPrice)  // 80

Это называется передача по значению - фундаментальный принцип Go.

Проверь себя

Вопросы

01Что такое переменная?
Показать ответ

Переменная - это именованная область памяти определённого типа. У неё есть имя (для обращения), тип (что можно хранить) и значение (текущее содержимое).

02В чём разница между var x = 10 и x := 10?
Показать ответ

Функционально - обе создают переменную x типа int со значением 10.

Различие: var работает везде (включая уровень пакета), а := только внутри функций. Для локальных переменных := предпочтительнее - короче.

03Можно ли объявить переменную на уровне пакета через :=?
Показать ответ

Нет. Оператор := работает только внутри функций. На уровне пакета нужно использовать var:

Go
var appVersion = "1.0.0"  // OK
// appVersion := "1.0.0"  // ошибка!
04Почему Go запрещает повторное объявление через :=?
Показать ответ

Защита от опечаток. Если случайно написать := вместо =, компилятор поймает ошибку. Без этой проверки можно создать новую переменную вместо изменения существующей и долго искать баг.

05Что означают названия типов int, float64, bool?
Показать ответ

int = integer (целое число)

float64 = floating point, 64 бита (число с плавающей точкой)

bool = boolean (логическое значение, по имени математика Джорджа Буля)

06Что выведет этот код?
Go
var a string = "25"
var b string = "4"
fmt.Println(a + b)
Показать ответ

Вывод: 254

Для строк + означает склеивание (конкатенацию), а не сложение чисел. Строки "25" и "4" склеиваются в "254".

Задачи

01Какое значение в b?
Go
var a int = 10
var b int = a * 8
Показать решение

Ответ: 80

a равно 10, умножаем на 8, получаем 80.

02Какое значение в a?
Go
a := 10
a = a + 1
Показать решение

Ответ: 11

Сначала a = 10. Потом берём значение a (10), прибавляем 1, записываем обратно в a.

03Swap в одну строку

Поменяй местами значения a и b без третьей переменной:

Go
a := 1
b := 2
// твой код
fmt.Println(a, b)  // должно вывести: 2 1
Показать решение
Go
a := 1
b := 2
a, b = b, a  // множественное присваивание
fmt.Println(a, b)  // 2 1

Go сначала вычисляет все значения справа (b, a = 2, 1), потом присваивает их слева.

04Исправь ошибку
Go
package main

globalCounter := 0  // ошибка компиляции!

func main() {
    fmt.Println(globalCounter)
}
Показать решение

Проблема: := не работает на уровне пакета.

Исправление:

Go
package main

import "fmt"

var globalCounter = 0  // используем var

func main() {
    fmt.Println(globalCounter)  // 0
}
05Что выведет?
Go
var a int = 6
var b int = a * 8
a = b - 8
var c int = b + a
fmt.Println(c)
Показать решение

Пошаговый разбор:

  1. a = 6
  2. b = 6 * 8 = 48
  3. a = 48 - 8 = 40 (перезаписываем a)
  4. c = 48 + 40 = 88

Ответ: 88

06Найди ошибку
Go
var a int = 6
var b int = a * 8
var a int = b - 8  // что не так?
var c int = b + a
fmt.Println(c)
Показать решение

Ошибка: Переменная a объявляется дважды через var. Нельзя создавать переменную с тем же именем в той же области видимости.

Исправление:

Go
var a int = 6
var b int = a * 8
a = b - 8          // убираем var - просто присваиваем
var c int = b + a
fmt.Println(c)     // 88
