Переменные в Go

Что спрашивают на собеседованиях

На собеседованиях по переменным редко задают прямые вопросы — обычно дают код и просят найти ошибку или предсказать поведение. Типичные ловушки:

"Что выведет этот код?" — и дальше shadowing или переопределение переменной.

Чтобы уверенно отвечать, нужно понимать как Go работает с памятью и почему язык устроен именно так.

Переменная — это ящик

Переменную можно представить как ящик для хранения. У ящика есть:

  • Имя — наклейка, по которой ты его находишь (age, userName, isActive)
  • Тип — размер и форма ящика (что туда можно положить)
  • Значение — то, что лежит внутри
42
значение
"age"
тип: int — определяет размер и допустимые значения

Под каждый тип данных нужен свой ящик: ты не будешь хранить воду в картонной коробке, а документы — в ведре. Так и в Go — int хранит числа, string хранит текст, и смешивать нельзя.

Что происходит в памяти

Когда ты пишешь x := 42, происходит следующее:

  1. Go выделяет участок памяти (для int — 8 байт на 64-bit системе)
  2. Записывает туда значение 42
  3. Связывает имя x с адресом этой памяти
Память:
...
42 (8b)
^ имя "x"
...

Когда ты пишешь fmt.Println(x), Go идёт по адресу, связанному с x, и читает значение оттуда.

Почему это важно понимать?

Потому что в Go переменные копируются при присваивании — изменение копии не влияет на оригинал:

Go 
x := 10
y := x    // y получает КОПИЮ значения x
y = 20    // меняем y

fmt.Println(x)  // 10 — x не изменился!
fmt.Println(y)  // 20

Это называется передача по значению (by value) — фундаментальный принцип Go.

*
Tip

Подробнее о передаче в функции изучим в модуле функций.

Статическая типизация: почему Go строгий

Go — статически типизированный язык. Это значит, что тип переменной:

  1. Определяется при объявлении
  2. Не может измениться за всё время жизни переменной
  3. Проверяется компилятором (программа, которая переводит Go-код в исполняемый файл) до запуска

Сравнение с динамической типизацией

В Python или JavaScript переменная — это просто "ярлык", который можно прицепить к любому объекту:

Python 
# Python: переменная как ярлык
x = 42          # x указывает на число
x = "hello"     # теперь на строку
x = [1, 2, 3]   # теперь на список
# Всё работает — Python не проверяет типы

В Go переменная — это конкретная область памяти определённого размера:

Go 
// Go: переменная — это типизированная память
x := 42          // выделено 8 байт под int
// x = "hello"   // Ошибка: строка не влезет в 8 байт int'а!

Зачем такие ограничения?

1. Производительность. Компилятор знает точный размер каждой переменной и может оптимально разместить их в памяти. В Python каждая переменная — это объект с метаданными, что замедляет работу.

2. Безопасность. Ошибки типов ловятся при компиляции, а не у пользователя в продакшене:

Go 
// Go гарантирует типы при компиляции:
var age int = 25
var name string = "Alice"

// age = name  // Ошибка компиляции! Нельзя присвоить string к int
// Ошибка найдена ДО запуска программы

3. Читаемость. Типы переменных сразу видны в объявлении:

Go 
var count int = 10
var message string = "Hello"
var ratio float64 = 0.75
// Сразу понятно: число, текст, дробное число
!
Important

Go выбрал статическую типизацию ради надёжности и скорости. Если код скомпилировался — целый класс ошибок уже исключён.

Объявление переменных: три способа

Способ 1: var с типом

Самый явный способ — указываем и тип, и значение:

Go 
var age int = 25
var name string = "Alice"
var isActive bool = true

Когда использовать: когда хочешь явно указать тип, особенно если он отличается от выведенного автоматически:

Go 
var bigNumber int64 = 100  // без int64 будет просто int
var smallFloat float32 = 3.14  // без float32 будет float64

Способ 2: var с выводом типа

Компилятор сам определит тип по значению:

Go 
var age = 25        // int (выведен автоматически)
var name = "Alice"  // string
var pi = 3.14       // float64 (по умолчанию для дробных)

Когда использовать: на уровне пакета (вне функций), где := не работает.

Способ 3: короткое объявление :=

Самый частый способ внутри функций:

Go 
age := 25
name := "Alice"
isActive := true

Оператор := делает две вещи сразу:

  1. Создаёт новую переменную
  2. Присваивает ей значение

Когда использовать: везде внутри функций, если не нужен особый тип.

Сравнение способов

Способ Где работает Когда использовать
var x int = 10 Везде Нужен конкретный тип
var x = 10 Везде На уровне пакета
x := 10 Только в функциях Локальные переменные
Go 
package main

var globalConfig = "production"  // var — на уровне пакета
// globalConfig := "production"  // := не работает вне функций

func main() {
    localVar := 42  // := — внутри функции (предпочтительно)
    var anotherVar = 42  // var — тоже работает, но длиннее
}

Нулевые значения: Go не оставляет мусор

В Go каждая переменная автоматически инициализируется нулевым значением своего типа. Никакого "мусора" из памяти:

Go 
var i int      // 0
var f float64  // 0.0
var s string   // "" (пустая строка)
var b bool     // false

Почему это важно?

В C/C++ неинициализированная переменная содержит случайный мусор — то, что было в этой памяти раньше. Это источник багов:

C 
// C: опасно!
int x;
printf("%d", x);  // Может напечатать что угодно: 0, -12345, 42...

В Go такой проблемы нет:

Go 
// Go: безопасно
var x int
fmt.Println(x)  // Всегда 0

Таблица нулевых значений

Тип Нулевое значение
int, int8, int16, int32, int64 0
uint, uint8, ... 0
float32, float64 0.0
bool false
string "" (пустая строка)
*
Tip

Нулевые значения для указателей, срезов, map и структур — в соответствующих уроках.

Множественное объявление

Несколько переменных в одной строке

Удобно при работе с функциями, возвращающими несколько значений:

Go 
// Объявление
x, y := 10, 20

// Из функции
file, err := os.Open("config.json")

// Обмен значений (swap) — элегантно!
a, b := 1, 2
a, b = b, a  // теперь a=2, b=1

Группировка с var ()

Для связанных переменных на уровне пакета:

Go 
var (
    // Настройки базы данных
    dbHost     = "localhost"
    dbPort     = 5432
    dbUser     = "admin"

    // Таймауты
    readTimeout  = 30 * time.Second
    writeTimeout = 30 * time.Second
)

Это чисто визуальная группировка — помогает читать код, но технически эквивалентна отдельным var.

Изменение значений: = vs :=

Это частый источник ошибок у новичков. Запомни:

  • := — создаёт новую переменную
  • = — изменяет существующую
Go 
x := 10      // создаём x со значением 10
x = 20       // меняем значение на 20
// x := 30   // ошибка: x уже существует!

Почему Go запрещает повторное :=?

Это защита от опечаток. Представь:

Go 
user := getCurrentUser()
// ... 100 строк кода ...
user := getAdminUser()  // Опечатка! Хотел =, написал :=

В Python такой код молча сработает — и ты потеряешь первого user'а. В Go компилятор скажет: "user уже объявлен. Ты уверен?"

Исключение: новые переменные в группе

Если хотя бы одна переменная в := новая — всё работает:

Go 
x := 10
x, y := 20, 30  // OK — y новая, x переиспользуется

Это удобно при работе с ошибками:

Go 
data, err := readFile("config.json")
// ... обработка ...
settings, err := readFile("settings.json")  // OK — settings новая

Область видимости (scope)

Переменная "живёт" только в том блоке, где объявлена. Блок — это код между { и }:

Go 
func main() {
    x := 10  // x видна во всей функции main

    {  // начало вложенного блока
        y := 20  // y видна только внутри этого блока
        fmt.Println(x, y)  // OK — видны обе переменные
    }  // конец блока, y уничтожается

    fmt.Println(x)   // OK — x всё ещё видна
    // fmt.Println(y)  // ошибка: y не определена
}

Визуализация областей видимости

Уровень пакета (видна везде в пакете)
var globalVar = 100
Уровень функции (видна во всей функции)
localVar := 200
Уровень блока (только внутри {})
blockVar := 300
Здесь видны: globalVar, localVar, blockVar
blockVar не видна
*
Tip

В модуле условий изучим if, который тоже создаёт блок со своей областью видимости.

Shadowing — главная ловушка

*
Tip

Мнемоника "ТЕНЬ": Переменная во внутреннем блоке Так же названа — Её Не видно из внешнего, она в тени! Ь — мягкий знак, мягко напоминает быть осторожным.

Что такое shadowing?

Shadowing (затенение) — когда переменная во внутреннем блоке "перекрывает" переменную с тем же именем из внешнего:

Go 
x := 10
fmt.Println(x)  // 10

{
    x := 20  // Это НОВАЯ переменная x!
    fmt.Println(x)  // 20
}

fmt.Println(x)  // 10 (!) — внешняя x не изменилась

Визуализация shadowing

Внешний блок:
x = 10
|
shadow (новая x)
x := 20
x = 20
Новая x "закрывает" внешнюю
|
После блока:
x = 10
Внешняя x никогда не менялась!

Почему shadowing опасен?

Ты думаешь, что меняешь переменную, а на самом деле создаёшь новую:

Go 
counter := 0
fmt.Println("Начало:", counter)  // 0

{
    counter := 100  // Новая переменная counter!
    counter = counter + 1
    fmt.Println("Внутри блока:", counter)  // 101
}

fmt.Println("После блока:", counter)  // 0 (!) — не 101!

Правильный вариант — без := во внутреннем блоке:

Go 
counter := 0
fmt.Println("Начало:", counter)  // 0

{
    counter = 100  // Меняем внешнюю переменную
    counter = counter + 1
    fmt.Println("Внутри блока:", counter)  // 101
}

fmt.Println("После блока:", counter)  // 101

Как защититься от shadowing?

  1. Используй линтер — автоматически находит shadowing
  2. Называй переменные уникально — innerCounter вместо counter
  3. Будь внимателен с := — если переменная уже существует, используй =
Bash 
# Установка линтера для проверки shadowing
go install golang.org/x/tools/go/analysis/passes/shadow/cmd/shadow@latest

# Проверка кода
go vet -vettool=$(which shadow) ./...
*
Tip

Shadowing особенно коварен с переменной err в функциях — изучим это в модуле функций.

Проверь себя

Вопросы

01 Что такое переменная на уровне памяти?
Показать ответ

Переменная — это именованная область памяти определённого размера. Имя переменной связано с адресом в памяти, где хранится значение. Размер области зависит от типа (например, int занимает 8 байт на 64-bit системе).

02 Почему Go статически типизирован? Какие преимущества это даёт?
Показать ответ

Преимущества статической типизации:

  1. Производительность — компилятор знает точные размеры и может оптимизировать
  2. Безопасность — ошибки типов ловятся при компиляции, а не в runtime
  3. Читаемость — сигнатуры функций сразу показывают типы
  4. IDE поддержка — автодополнение и рефакторинг работают лучше
03 В чём разница между var x = 10 и x := 10?
Показать ответ

Функционально — разницы нет, обе создают переменную x типа int со значением 10.

Различия:

  • var работает везде (и в функциях, и на уровне пакета)
  • := работает только внутри функций
  • := короче и предпочтительнее для локальных переменных
04 Какое нулевое значение у string, bool, float64?
Показать ответ
  • string — "" (пустая строка)
  • bool — false
  • float64 — 0.0
05 Почему Go запрещает повторное объявление переменной через :=?
Показать ответ

Это защита от опечаток. Если ты случайно написал := вместо =, компилятор поймает ошибку. Без этой проверки ты мог бы случайно создать новую переменную вместо изменения существующей, потеряв предыдущее значение.

06 Что такое shadowing и почему это опасно?
Показать ответ

Shadowing — когда переменная во внутреннем блоке "перекрывает" переменную с тем же именем из внешнего. Опасно тем, что:

  • Изменения во внутреннем блоке не влияют на внешнюю переменную
  • Легко ошибиться, думая что меняешь внешнюю переменную
  • Особенно коварно с err — типичная ловушка Go

Задачи

01 Что выведет?
Go 
func main() {
    x := 10

    {
        x := 20
        x = 30
        fmt.Println("Внутри:", x)
    }

    fmt.Println("Снаружи:", x)
}
Показать решение

Вывод:

Внутри: 30
Снаружи: 10

Разбор:

  1. x := 10 — создаём x = 10 в main
  2. x := 20 — создаём новую x = 20 внутри блока (shadowing!)
  3. x = 30 — меняем внутреннюю x на 30
  4. После блока внутренняя x уничтожается
  5. Внешняя x всё ещё 10
02 Что выведет?
Go 
func main() {
    a := 1

    {
        a = 10  // = не :=
        b := 20
        fmt.Println(a, b)
    }

    fmt.Println(a)
}
Показать решение

Вывод:

10 20
10

Разбор:

  1. a := 1 — создаём a = 1
  2. a = 10 — меняем ту же a (используем =, не :=)
  3. b := 20 — создаём b внутри блока
  4. Внутри блока: a = 10, b = 20
  5. После блока b уничтожается, a осталась 10

Ключ: Здесь нет shadowing, потому что a = 10 использует = вместо :=.

03 Что выведет?
Go 
func main() {
    x := 1
    x, y := 2, 3
    fmt.Println(x, y)
}
Показать решение

Вывод: 2 3

Разбор:

  1. x := 1 — создаём x = 1
  2. x, y := 2, 3 — y новая, поэтому := разрешён. x переиспользуется и получает 2
  3. Итого: x = 2, y = 3

Это исключение из правила "нельзя объявить повторно" — если хотя бы одна переменная новая.

04 Исправь код
Go 
package main

globalCounter := 0  // ошибка компиляции!

func main() {
    fmt.Println(globalCounter)
}
Показать решение

Проблема: := не работает на уровне пакета.

Исправление:

Go 
package main

var globalCounter = 0  // используем var на уровне пакета

func main() {
    fmt.Println(globalCounter)  // 0
}
05 Swap без временной переменной

Поменяй местами значения переменных a и b без использования третьей переменной. Подсказка: в Go это делается элегантно в одну строку.

Go 
a := 1
b := 2
// Твой код — поменяй a и b местами
fmt.Println(a, b)  // должно вывести: 2 1
Показать решение
Go 
a := 1
b := 2
a, b = b, a  // swap в одну строку!
fmt.Println(a, b)  // 2 1

Go поддерживает множественное присваивание. Сначала вычисляются все правые части (b, a = 2, 1), потом присваиваются левым (a, b).
