Переменные в Go

Переменная - это именованный ящик для хранения данных. Научимся создавать переменные тремя способами и избегать типичных ловушек.

Что такое переменная

Переменная - это именованный ящик для хранения данных. У каждого ящика есть:

  • Имя - по нему ты находишь ящик (age, userName, isActive)
  • Тип - что можно положить внутрь (int для чисел, string для текста)
  • Значение - то, что лежит внутри прямо сейчас

Вот минимальный рабочий пример:

Go
package main

import "fmt"

func main() {
    name := "Alice"
    age := 25
    fmt.Println(name, age)  // Alice 25
}

Две переменные: name хранит текст, age хранит число. Обе созданы через := - это самый частый способ.

Как работать с переменными

Три способа объявления

В Go есть три способа создать переменную. Давай разберём каждый.

Способ 1: короткое объявление :=

Самый частый способ внутри функций:

Go
age := 25           // int
name := "Alice"     // string
isActive := true    // bool
price := 19.99      // float64

Оператор := делает две вещи: создаёт переменную и присваивает значение. Тип определяется автоматически по значению справа.

Способ 2: var с выводом типа

Компилятор сам определяет тип:

Go
var age = 25        // int
var name = "Alice"  // string
var pi = 3.14       // float64

Зачем нужен var, если есть :=? Потому что := работает только внутри функций. На уровне пакета нужен var:

Go
package main

var globalConfig = "production"  // var - на уровне пакета
// globalConfig := "production"  // ошибка: := не работает вне функций

func main() {
    localVar := 42  // := - внутри функции
}

Способ 3: var с явным типом

Иногда нужно указать тип явно:

Go
var bigNumber int64 = 100    // без int64 было бы просто int
var smallFloat float32 = 3.14  // без float32 было бы float64

Вот сравнение всех способов:

Способ Где работает Когда использовать
x := 10 Только в функциях Локальные переменные (90% случаев)
var x = 10 Везде На уровне пакета
var x int = 10 Везде Нужен конкретный тип

Изменение значений: = vs :=

Это частая ловушка для новичков. Запомни:

  • := создаёт новую переменную
  • = изменяет существующую
Go
x := 10      // создаём x
x = 20       // меняем значение
// x := 30   // ошибка: x уже существует!

Почему Go запрещает повторное :=? Это защита от опечаток. Представь, что ты хотел написать =, а написал := - и случайно создал новую переменную вместо изменения старой. Go такое не пропустит.

Но есть исключение. Если хотя бы одна переменная в группе новая - всё работает:

Go
x := 10
x, y := 20, 30  // OK - y новая, x переиспользуется

Это удобно при работе с функциями, которые возвращают несколько значений:

Go
data, err := readFile("config.json")
// ... обработка ...
settings, err := readFile("settings.json")  // OK - settings новая

Нулевые значения

В Go каждая переменная автоматически получает нулевое значение своего типа:

Go
var i int      // 0
var f float64  // 0.0
var s string   // "" (пустая строка)
var b bool     // false

Это гарантия безопасности: в Go не бывает "мусора" в памяти. Переменная всегда имеет предсказуемое начальное значение.

Множественное объявление

Go позволяет работать с несколькими переменными одновременно:

Go
// Объявление нескольких переменных
x, y := 10, 20

// Обмен значений - элегантно в одну строку!
a, b := 1, 2
a, b = b, a  // теперь a=2, b=1

Для группировки связанных переменных на уровне пакета используй var ():

Go
var (
    dbHost = "localhost"
    dbPort = 5432
    dbUser = "admin"
)

Область видимости (scope)

Переменная "живёт" только в том блоке, где объявлена. Блок - это код между { и }:

Go
func main() {
    x := 10  // x видна во всей функции main

    {  // начало вложенного блока
        y := 20  // y видна только внутри этого блока
        fmt.Println(x, y)  // OK - видны обе
    }  // конец блока, y уничтожается

    fmt.Println(x)   // OK - x всё ещё видна
    // fmt.Println(y)  // ошибка: y не определена
}

Ловушка: shadowing (затенение)

Это главная ловушка Go с переменными. Смотри внимательно:

Go
x := 10
fmt.Println(x)  // 10

{
    x := 20  // это НОВАЯ переменная x!
    fmt.Println(x)  // 20
}

fmt.Println(x)  // 10 - внешняя x не изменилась!

Во внутреннем блоке x := 20 создаёт новую переменную, которая "затеняет" внешнюю. Изменения внутренней не влияют на внешнюю.

Если хочешь изменить внешнюю переменную - используй = без двоеточия:

Go
x := 10
{
    x = 20  // меняем внешнюю x (без двоеточия!)
}
fmt.Println(x)  // 20

Как защититься от shadowing:

  1. Внимательно смотри на := vs =
  2. Называй переменные уникально во вложенных блоках
  3. Используй линтер - он автоматически находит shadowing
✓

Установи линтер shadow: go install golang.org/x/tools/go/analysis/passes/shadow/cmd/shadow@latest

Правила именования

Несколько правил для имён переменных:

  1. Только латинские буквы a-z, A-Z, цифры, подчёркивание _
  2. Имя не может начинаться с цифры
  3. Go регистрочувствителен: name и Name - разные переменные
  4. Имя должно отражать назначение: userAge лучше чем x
Go
// Хорошие имена
userName := "Alice"
totalCount := 42
isActive := true

// Плохие имена
x := "Alice"      // непонятно что хранит
a1 := 42          // бессмысленно
flag := true      // какой flag?

Почему Go работает именно так

Статическая типизация

Go - статически типизированный язык. Это значит:

  1. Тип переменной определяется при объявлении
  2. Тип не может измениться за всё время жизни переменной
  3. Компилятор проверяет типы до запуска программы
Go
x := 42          // x имеет тип int
// x = "hello"   // ошибка компиляции: нельзя присвоить string к int

Зачем такие ограничения?

Производительность. Компилятор знает точный размер каждой переменной и оптимально размещает их в памяти.

Безопасность. Ошибки типов ловятся при компиляции, а не у пользователя в продакшене.

Читаемость. Типы переменных сразу видны в коде.

✓

Для свитчеров из Python/JS: там переменные "резиновые" - можно присвоить что угодно. В Go тип фиксирован навсегда. Это не баг, а фича - защита от целого класса ошибок.

Как устроена память

Когда ты пишешь x := 42, Go делает следующее:

  1. Выделяет участок памяти (для int - 8 байт на 64-bit системе)
  2. Записывает туда значение 42
  3. Связывает имя x с адресом этой памяти

Вот как это выглядит в памяти:

Память:
...
42
...
↑ имя "x"

Почему это важно? Потому что в Go переменные копируются при присваивании:

Go
x := 10
y := x    // y получает КОПИЮ значения x
y = 20    // меняем y

fmt.Println(x)  // 10 - x не изменился!
fmt.Println(y)  // 20

Это называется передача по значению - фундаментальный принцип Go.

✓

Как передать переменную так, чтобы изменения были видны снаружи? Через указатели - изучим в модуле "Указатели".

Проверь себя

Вопросы

01Что такое переменная?
Показать ответ

Переменная - это именованная область памяти определённого типа. У неё есть имя (для обращения), тип (что можно хранить) и значение (текущее содержимое).

02В чём разница между var x = 10 и x := 10?
Показать ответ

Функционально - обе создают переменную x типа int со значением 10.

Различие: var работает везде, а := только внутри функций. Для локальных переменных := предпочтительнее - короче.

03Какое нулевое значение у string, bool, int?
Показать ответ

string - "" (пустая строка)

bool - false

int - 0

04Почему Go запрещает повторное объявление через :=?
Показать ответ

Защита от опечаток. Если случайно написать := вместо =, компилятор поймает ошибку. Без этой проверки можно случайно создать новую переменную вместо изменения существующей.

05Что такое shadowing и почему это опасно?
Показать ответ

Shadowing - когда переменная во внутреннем блоке "перекрывает" переменную с тем же именем из внешнего. Опасно тем, что изменения внутренней переменной не влияют на внешнюю - легко получить баг, думая что меняешь одну переменную, а на самом деле создаёшь другую.

Задачи

01Что выведет?
Go
func main() {
    x := 10

    {
        x := 20
        x = 30
        fmt.Println("Внутри:", x)
    }

    fmt.Println("Снаружи:", x)
}
Показать решение

Вывод:

Output
Внутри: 30
Снаружи: 10

Разбор:

  1. x := 10 - создаём x = 10 в main
  2. x := 20 - создаём новую x = 20 внутри блока (shadowing!)
  3. x = 30 - меняем внутреннюю x на 30
  4. После блока внутренняя x уничтожается
  5. Внешняя x всё ещё 10
02Что выведет?
Go
func main() {
    a := 1

    {
        a = 10  // = не :=
        b := 20
        fmt.Println(a, b)
    }

    fmt.Println(a)
}
Показать решение

Вывод:

Output
10 20
10

Разбор: Здесь a = 10 использует = (не :=), поэтому меняется внешняя переменная a. Нет shadowing.

03Что выведет?
Go
func main() {
    x := 1
    x, y := 2, 3
    fmt.Println(x, y)
}
Показать решение

Вывод: 2 3

Разбор: x, y := 2, 3 - переменная y новая, поэтому := разрешён. Переменная x переиспользуется и получает значение 2.

04Исправь ошибку
Go
package main

globalCounter := 0  // ошибка компиляции!

func main() {
    fmt.Println(globalCounter)
}
Показать решение

Проблема: := не работает на уровне пакета.

Исправление:

Go
package main

var globalCounter = 0  // используем var

func main() {
    fmt.Println(globalCounter)  // 0
}
05Swap в одну строку

Поменяй местами значения a и b без третьей переменной:

Go
a := 1
b := 2
// твой код
fmt.Println(a, b)  // должно вывести: 2 1
Показать решение
Go
a := 1
b := 2
a, b = b, a  // множественное присваивание
fmt.Println(a, b)  // 2 1

Go сначала вычисляет все значения справа (b, a = 2, 1), потом присваивает их слева.

06Какое значение в b?
Go
var a int = 10
var b int = a * 8
Показать решение

Ответ: 80

a равно 10, умножаем на 8, получаем 80.
