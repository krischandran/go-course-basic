Переменные в Go

Переменная (от англ. variable - изменяемый) - это именованная область памяти для хранения данных. Go предлагает два способа создания переменных: классический var и короткий :=. При этом тип фиксируется при создании и не меняется - Go статически типизирован. Ещё одна особенность: Go сам инициализирует переменные нулевыми значениями, если ты не указал значение явно.

Создание переменных

Представь переменную как ящик с наклейкой. На наклейке - имя, внутри - значение, а форма ящика определяет, что можно положить (тип).

Go предлагает три способа создать переменную.

Короткое объявление :=

Самый частый способ внутри функций. Оператор := делает две вещи сразу: создаёт переменную и кладёт в неё значение:

Go
package main

import "fmt"

func main() {
    userName := "Олег"
    age := 25
    fmt.Println(userName, age)
}
Output
Олег 25

Тип Go определяет автоматически по значению справа. "Олег" - это текст, значит userName получит тип string (строка). 25 - целое число, значит age получит тип int (от integer - целое число).

var с выводом типа

Иногда нужно ключевое слово var. Тип по-прежнему определяется автоматически:

Go
var age = 25           // int
var userName = "Олег"  // string
var price = 199.99     // float64 (число с плавающей точкой)

Зачем var, если есть :=? Короткое объявление работает только внутри функций. На уровне пакета нужен var:

Go
package main

var appVersion = "1.0.0"  // var - на уровне пакета
// appVersion := "1.0.0"  // ошибка: := не работает вне функций

func main() {
    userName := "Олег"  // := - внутри функции, OK
}

var с явным типом

Когда нужен конкретный тип, указывай его явно:

Go
var userID int64 = 9223372036854775807  // без int64 было бы просто int
var temperature float32 = 36.6          // без float32 было бы float64

Вот сравнение трёх способов:

Способ Где работает Когда использовать
x := 10 Только в функциях Локальные переменные (90% случаев)
var x = 10 Везде На уровне пакета
var x int = 10 Везде Нужен конкретный тип

Изменение значений

Важное различие: := создаёт новую переменную, = изменяет существующую.

Go
package main

import "fmt"

func main() {
    price := 100     // создаём price = 100
    price = 150      // меняем на 150
    fmt.Println(price)
}
Output
150

А вот что будет при попытке повторного объявления:

Go
price := 100
// price := 200  // ошибка: no new variables on left side of :=

Почему Go запрещает повторное :=? Защита от опечаток. Если случайно написать := вместо =, компилятор поймает ошибку. Иначе ты бы создал новую переменную вместо изменения старой и долго искал баг.

Но есть исключение. Если хотя бы одна переменная в группе новая - работает:

Go
price := 100
price, discount := 150, 10  // OK - discount новая, price переиспользуется

Множественное объявление

Go позволяет работать с несколькими переменными одновременно:

Go
package main

import "fmt"

func main() {
    width, height := 1920, 1080
    fmt.Println(width, height)
}
Output
1920 1080

Особенно удобен обмен значений - в одну строку, без временной переменной:

Go
package main

import "fmt"

func main() {
    a, b := 1, 2
    a, b = b, a  // обмен!
    fmt.Println(a, b)
}
Output
2 1

Как это работает? Go сначала вычисляет ВСЕ значения справа (b, a = 2, 1), потом присваивает их слева.

Для группировки связанных переменных на уровне пакета используй var ():

Go
var (
    dbHost = "localhost"
    dbPort = 5432
    dbUser = "admin"
)

Правила именования

Несколько правил для имён переменных:

  1. Только латинские буквы a-z, A-Z, цифры, подчёркивание _
  2. Имя не может начинаться с цифры
  3. Go регистрочувствителен: name и Name - разные переменные
  4. Имя должно отражать назначение
Go
// Хорошие имена - понятно что хранят
userName := "Олег"
orderTotal := 1500
isDelivered := false

// Плохие имена - непонятно
x := "Олег"       // что это?
n := 1500         // какое число?
flag := false     // какой флаг?

Нулевые значения

Что лежит в переменной, если мы её создали, но ничего не положили?

Go
package main

import "fmt"

func main() {
    var count int
    var name string
    var active bool
    fmt.Println(count, name, active)
}
Output
0  false

Go автоматически инициализирует переменные нулевым значением (zero value) типа:

  • Числа: 0
  • Строки: "" (пустая строка)
  • Логические: false

Это не случайность, а осознанное решение. В Go нет "мусора" в памяти - переменная всегда имеет определённое значение.

Статическая типизация

Go - статически типизированный язык. Это значит:

  1. Тип переменной определяется при создании
  2. Тип не меняется за всё время жизни переменной
  3. Компилятор проверяет типы ДО запуска программы
Go
price := 100        // price имеет тип int
// price = "сто"    // ошибка компиляции: нельзя присвоить string к int

Зачем такие ограничения?

Производительность. Компилятор знает точный размер каждой переменной и размещает их в памяти оптимально.

Безопасность. Ошибки типов ловятся при компиляции, а не у пользователя в продакшене.

Читаемость. Тип переменной понятен из кода.

✓

Для знакомых с Python/JS: там переменная может хранить что угодно - сегодня число, завтра строку. В Go тип фиксирован. Это не ограничение, а защита от целого класса ошибок.

Как устроена память

Когда пишешь price := 100, Go делает следующее:

  1. Выделяет участок памяти (для int - 8 байт на 64-bit системе)
  2. Записывает туда значение 100
  3. Связывает имя price с адресом этой памяти

Вот как это выглядит:

Память:
... 100 (8b) ...
^ имя "price"

Как видишь, переменная - это просто имя, указывающее на участок памяти.

Передача по значению

В Go переменные копируются при присваивании:

Go
package main

import "fmt"

func main() {
    originalPrice := 100
    discountedPrice := originalPrice  // копия значения
    discountedPrice = 80              // меняем копию

    fmt.Println(originalPrice)    // 100 - не изменился!
    fmt.Println(discountedPrice)  // 80
}
Output
100
80

Это называется передача по значению - фундаментальный принцип Go.

Проверь себя

Вопросы

01Что такое переменная?
Показать ответ

Переменная - это именованная область памяти определённого типа. У неё есть имя (для обращения), тип (что можно хранить) и значение (текущее содержимое).

02В чём разница между var x = 10 и x := 10?
Показать ответ

Функционально обе создают переменную x типа int со значением 10. Различие: var работает везде (включая уровень пакета), а := только внутри функций. Для локальных переменных := предпочтительнее - короче.

03Можно ли объявить переменную на уровне пакета через :=?
Показать ответ

Нет. Оператор := работает только внутри функций. На уровне пакета нужно использовать var.

04Почему Go запрещает повторное объявление через :=?
Показать ответ

Защита от опечаток. Если случайно написать := вместо =, компилятор поймает ошибку. Без этой проверки можно создать новую переменную вместо изменения существующей и долго искать баг.

05Какое нулевое значение у типа int? У типа string? У типа bool?
Показать ответ

int: 0, string: "" (пустая строка), bool: false.

06Что выведет этот код? Определи без запуска:
Go
x := 10
y := x
y = 20
fmt.Println(x)
Показать ответ

Вывод: 10. При y := x создаётся копия значения x. Изменение y не влияет на оригинал - это передача по значению.

Задачи

01Какое значение в b?
Go
var a int = 10
var b int = a * 8
Показать решение

b = 80 (10 * 8 = 80)

02Какое значение в a?
Go
a := 10
a = a + 1
Показать решение

a = 11 (10 + 1 = 11)

03Swap в одну строку

Поменяй местами значения a и b без третьей переменной:

Go
a := 1
b := 2
// твой код
fmt.Println(a, b)  // должно вывести: 2 1
Показать решение
Go
a, b = b, a  // множественное присваивание
04Исправь ошибку
Go
package main

globalCounter := 0  // ошибка компиляции!

func main() {
    fmt.Println(globalCounter)
}
Показать решение
Go
package main

import "fmt"

var globalCounter = 0  // используем var вместо :=

func main() {
    fmt.Println(globalCounter)  // 0
}
05Что выведет?
Go
var a int = 6
var b int = a * 8
a = b - 8
var c int = b + a
fmt.Println(c)
Показать решение

88. Пошагово: a=6, b=48, a=40, c=48+40=88.

06Найди ошибку
Go
var a int = 6
var b int = a * 8
var a int = b - 8  // что не так?
var c int = b + a
fmt.Println(c)
Показать решение

Переменная a объявляется дважды через var. Нужно убрать var в третьей строке:

Go
a = b - 8  // просто присваиваем, без var
