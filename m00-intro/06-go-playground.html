Go Playground и запуск программ

Go предоставляет несколько способов запуска программ: от онлайн-песочницы до полноценной компиляции. Разберём основные инструменты.

Go Playground: песочница в браузере

Что такое Go Playground

Go Playground - это онлайн-редактор, где можно писать и запускать Go-код без установки компилятора. Работает прямо в браузере.

Найти его просто: введите в поисковике "Go Playground" - первая ссылка откроет сайт play.golang.org (или go.dev/play).

Когда использовать

Сценарий Подходит
Быстро проверить идею Да
Поделиться кодом с другими Да
Задать вопрос и показать проблему Да
Разработка реального проекта Нет

Как поделиться кодом

Одна из главных фич Playground - возможность поделиться кодом:

  1. Напишите код
  2. Нажмите Share
  3. Скопируйте ссылку
  4. Отправьте тому, кому нужна помощь

Человек откроет ссылку и увидит ваш код. Это удобно для вопросов в чатах и форумах.

!
Important

Всегда прикрепляйте код, когда задаёте вопрос. Без кода невозможно понять проблему.

Ограничения

  • Нет доступа к файловой системе
  • Ограничено время выполнения
  • Не все пакеты доступны
  • Для серьёзной разработки используйте локальное окружение

Запуск программы: go run

Основы go run

Команда go run - самый быстрый способ запустить программу:

Bash 
go run main.go

Что происходит внутри:

  1. Go компилирует исходный код
  2. Создаёт временный исполняемый файл
  3. Запускает его
  4. После выполнения удаляет временный файл

Навигация в терминале

Стрелка вверх в терминале показывает предыдущую команду. Не нужно печатать go run main.go каждый раз - просто нажмите стрелку вверх и Enter.

Запуск нескольких файлов

Если проект состоит из нескольких файлов в одном пакете:

Bash 
go run .

Точка означает "все Go-файлы в текущей директории".

Компиляция: go build

Зачем компилировать

go run удобен для разработки, но созданный файл - временный. Если хотите:

  • Передать программу на другой компьютер
  • Запускать без установленного Go
  • Распространять программу

...нужна компиляция через go build.

Базовая компиляция

Bash 
go build

Создаст исполняемый файл с именем модуля (из go.mod). Если модуль называется your-go, появится файл your-go (или your-go.exe на Windows).

Компиляция с указанием имени

Bash 
go build -o myprogram main.go

Флаг -o (output) позволяет задать имя:

Команда Результат
go build your-go (имя модуля)
go build -o prog main.go prog
go build -o app.exe main.go app.exe

Запуск скомпилированного файла

Bash 
./myprogram

На Linux/Mac нужен ./ перед именем файла, чтобы указать "файл в текущей директории".

На Windows:

Bash 
myprogram.exe

Преимущество компиляции

Скомпилированный файл не требует Go на целевом компьютере. Это готовая программа - можно передать кому угодно, и она запустится.

Кросс-компиляция

Сборка под другую ОС

Go умеет компилировать программы для другой операционной системы. Это называется кросс-компиляция.

Работая на Linux, можно собрать программу для Windows:

Bash 
GOOS=windows GOARCH=amd64 go build -o prog.exe main.go
Переменная Значение Описание
GOOS windows Целевая ОС
GOARCH amd64 Архитектура (64-бит)

Популярные комбинации

GOOS GOARCH Результат
windows amd64 Windows 64-bit
linux amd64 Linux 64-bit
darwin amd64 macOS Intel
darwin arm64 macOS Apple Silicon

Список всех платформ

Bash 
go tool dist list

Покажет все поддерживаемые комбинации ОС и архитектур. Их очень много - от Android до Plan 9.

Стиль кода и форматирование

Единый стиль в Go

В отличие от многих языков, в Go стиль кода регламентирован. Разработчики Go решили: "Пишем так, и никак иначе".

Нельзя перенести фигурную скобку на новую строку:

Go 
// ОШИБКА - так нельзя в Go
func main()
{
    fmt.Println("Hello")
}

// ПРАВИЛЬНО - скобка на той же строке
func main() {
    fmt.Println("Hello")
}

Почему это хорошо

В других языках люди спорят о стилях: "Скобка на той же строке или на новой?", "Пробелы или табы?". В Go споров нет - есть один правильный способ.

Преимущества:

  • Весь Go-код выглядит одинаково
  • Легче читать чужой код
  • Код выглядит так, будто его писал один человек

Автоформатирование

При сохранении файла в VS Code (Ctrl+S) Go-расширение автоматически форматирует код:

До сохранения:

Go 
func main(){
fmt.Println(   "Hello"   )
    }

После сохранения:

Go 
func main() {
    fmt.Println("Hello")
}

Это делает утилита gofmt, встроенная в Go. Не нужно следить за пробелами и отступами - форматирование происходит автоматически.

Ручное форматирование

Если хотите отформатировать файл из командной строки:

Bash 
gofmt -w main.go

Флаг -w (write) перезаписывает файл. Без него gofmt только выводит отформатированный код, не изменяя файл.

Проверь себя

01 Для чего нужен Go Playground?
Показать ответ

Go Playground - онлайн-песочница для написания и запуска Go-кода в браузере. Удобен для быстрых экспериментов и для того, чтобы делиться кодом с другими (например, при вопросах на форумах).

02 Чем go run отличается от go build?
Показать ответ

go run компилирует и сразу запускает программу, создавая временный файл, который удаляется после выполнения. go build создаёт постоянный исполняемый файл, который можно запускать без Go и передавать на другие компьютеры.

03 Как собрать программу для Windows, работая на Linux?
Показать ответ

Использовать кросс-компиляцию с переменными окружения:

Bash 
GOOS=windows GOARCH=amd64 go build -o prog.exe main.go
04 Что делает gofmt?
Показать ответ

gofmt - утилита для автоматического форматирования Go-кода по единому стандарту. Выравнивает отступы, расставляет пробелы, приводит код к стандартному стилю Go.

Задачи

01 Какая команда?

Вам нужно создать исполняемый файл с именем calculator из файла main.go. Какую команду использовать?

Показать решение
Bash 
go build -o calculator main.go

Флаг -o задаёт имя выходного файла.

02 Что не так?

Код не компилируется. Найдите ошибку:

Go 
package main

import "fmt"

func main()
{
    fmt.Println("Hello")
}
Показать решение

Ошибка: Фигурная скобка на новой строке.

Исправление:

Go 
func main() {
    fmt.Println("Hello")
}

В Go открывающая фигурная скобка должна быть на той же строке, что и объявление функции.

03 Кросс-компиляция

Напишите команду для сборки программы под macOS с процессором Apple Silicon (arm64).

Показать решение
Bash 
GOOS=darwin GOARCH=arm64 go build -o myapp main.go

darwin - это внутреннее имя macOS в Go.

04 Что произойдёт?

Вы выполнили:

Bash 
go build -o app
./app

Программа выводит "Hello". Затем вы изменили код на "Hello, World!" и снова выполнили:

Bash 
./app

Что выведется?

Показать решение

Выведется: Hello

Почему: Вы не перекомпилировали программу. Файл app содержит старую версию. Нужно снова выполнить go build -o app, чтобы обновить исполняемый файл.

05 Порядок действий

Расставьте действия в правильном порядке для создания и запуска программы:

A) go mod init myproject
B) Создать файл main.go
C) Написать код
D) go run main.go

Показать решение

Правильный порядок: A -> B -> C -> D

  1. A) go mod init myproject - инициализировать модуль
  2. B) Создать файл main.go
  3. C) Написать код
  4. D) go run main.go - запустить

Резюме

Key Go Playground и запуск программ
  • Go Playground - онлайн-песочница для экспериментов и обмена кодом
  • go run - компилирует и запускает (временный файл)
  • go build - создаёт постоянный исполняемый файл
  • go build -o name - задаёт имя выходного файла
  • Кросс-компиляция: GOOS=os GOARCH=arch go build
  • gofmt автоматически форматирует код по стандарту Go
  • Стиль кода в Go единый - скобки на той же строке
!
Главная ловушка

Забыть перекомпилировать после изменения кода. Правило: Изменил код - перекомпилируй перед запуском.
