Первая программа: Hello, World!

Разбирая простую программу, изучим фундамент всех будущих программ на Go.

АНАТОМИЯ GO-ПРОГРАММЫ

Вот как выглядит простейшая программа на Go:

Go 
package main

import "fmt"

func main() {
    fmt.Println("Привет, мир!")
}

Разберем каждую строку - это фундамент всех будущих программ.

Строка 1: package main

Эта строка указывает пакет, к которому относится код.

Что такое пакет? Пакет - это способ организовать код в программе. Как папки на компьютере группируют файлы по смыслу, так пакеты группируют код по назначению.

В Go каждый файл должен начинаться с указания пакета. Имя main - особенное: оно означает, что это исполняемая программа, а не библиотека.

main = точка входа
Компилятор ищет: 1. Пакет main
2. Функцию main внутри него
3. Запускает эту функцию
!

Если хотите создать библиотеку (код для других программ), используйте другое имя пакета. Но для исполняемых программ всегда package main.

Строка 2: import "fmt"

Эта строка подключает пакет fmt из стандартной библиотеки Go.

Что такое fmt? Сокращение от "format" (форматирование). Пакет fmt предоставляет инструменты для работы с текстом - вывод на экран, форматирование строк, чтение ввода.

import "fmt"
import - подключение пакета
"fmt" - имя пакета в кавычках
!

Go не позволяет импортировать неиспользуемые пакеты. Если написать import "fmt", но не использовать ничего из него, компилятор выдаст ошибку.

Строка 3: func main()

Это объявление функции с именем main.

Часть Значение
func Ключевое слово - объявляем функцию
main Имя функции (особенное - точка входа)
() Скобки для параметров (пустые)
{ Начало тела функции

Функция main в пакете main - это точка входа в программу. Когда вы запускаете программу, выполнение начинается именно с нее.

Строка 4: fmt.Println()

Вызываем функцию Println из пакета fmt.

Часть Значение
fmt Имя пакета
. Обращение к содержимому пакета
Println Имя функции (Print Line)
("...") Аргумент функции - что выводить

Аргумент (или параметр) - это данные, которые мы передаем функции в скобках.

PRINT VS PRINTLN

В пакете fmt есть две похожие функции для вывода:

fmt.Print - вывод без перевода строки

Go 
package main

import "fmt"

func main() {
    fmt.Print("Hello")
    fmt.Print("World!")
}
// Вывод: HelloWorld!

После Print курсор остается на той же строке. Следующий вывод продолжится там же.

fmt.Println - вывод с переводом строки

Go 
package main

import "fmt"

func main() {
    fmt.Println("Hello")
    fmt.Println("World!")
}
// Вывод:
// Hello
// World!

После Println курсор переходит на новую строку. Буква ln в названии означает "line" - строка.

Print("A") Print("B") - Вывод: AB (курсор на той же строке)
Println("A") Println("B") - Вывод: A и B на разных строках

Пустой Println

Println без аргументов просто вставляет пустую строку:

Go 
fmt.Println("Строка 1")
fmt.Println()
fmt.Println("Строка 3")
// Вывод:
// Строка 1
//
// Строка 3

КОММЕНТАРИИ В КОДЕ

В Go есть способ оставлять заметки в коде - комментарии. Компилятор их полностью игнорирует.

Однострочные комментарии

Go 
fmt.Println("Go") // это комментарий - текст после //

Все после // до конца строки - комментарий.

Практический пример

Go 
package main

import "fmt"

func main() {
    // fmt.Println("Java")
    // fmt.Println("Swift")
    fmt.Println("Go") // основной вывод
    fmt.Println("Python")
    fmt.Println("other")
}
// Вывод: Go, Python, other

Закомментированные строки не выполняются. Это удобно для временного отключения кода.

РЕГИСТРОЗАВИСИМОСТЬ

Go различает заглавные и строчные буквы. Это называется регистрозависимость.

Go 
// Правильно
fmt.Println("Hello")

// ОШИБКА - такой функции нет
fmt.println("Hello")  // p маленькая
fmt.PRINTLN("Hello")  // все заглавное

Имена функций, переменных, пакетов нужно писать точно так, как они определены.

i

В Go имена функций из пакетов начинаются с заглавной буквы (Println, Print). Это не случайность - в следующих уроках узнаете почему.

СОЗДАНИЕ ПЕРВОЙ ПРОГРАММЫ

Создадим программу пошагово.

  1. Откройте VS Code (или другой редактор)
  2. Создайте файл main.go
  3. Введите код программы
  4. Сохраните файл (Ctrl+S)
  5. Откройте терминал и выполните:
Bash 
go run main.go

Результат:

Output 
Привет, мир!

Команда go run компилирует и сразу запускает программу.

ЛОВУШКИ НОВИЧКОВ

Ловушка 1: Неиспользуемый импорт

Go 
package main

import "fmt"  // импортировали

func main() {
    // но ничего не используем из fmt
}

Ошибка: imported and not used: "fmt"

Go не позволяет "мертвый" код. Если импортировали - используйте.

Ловушка 2: Нет функции main

Go 
package main

import "fmt"

func hello() {
    fmt.Println("Hello")
}

Ошибка: function main is undeclared in the main package

В пакете main обязательно должна быть функция main.

Ловушка 3: Неправильный регистр

Go 
fmt.println("Hello")  // маленькая p

Ошибка: undefined: fmt.println

Функция называется Println с большой буквы P.

ПРОВЕРЬ СЕБЯ

Вопросы

01 Почему пакет называется main?
Показать ответ

Имя main зарезервировано в Go для исполняемых программ. Компилятор ищет пакет main и функцию main внутри него как точку входа. Если использовать другое имя, получится библиотека, а не запускаемая программа.

02 Что делает строка import "fmt"?
Показать ответ

Подключает стандартный пакет fmt (format), который содержит функции для форматированного ввода-вывода. Без этой строки нельзя использовать fmt.Println и другие функции пакета.

03 В чем разница между Print и Println?
Показать ответ

Print выводит текст и оставляет курсор на той же строке. Println выводит текст и переводит курсор на новую строку. Буквы ln означают "line" (строка).

04 Что произойдет, если импортировать пакет, но не использовать его?
Показать ответ

Компилятор выдаст ошибку imported and not used. Go запрещает неиспользуемые импорты - это помогает держать код чистым.

Задачи

01 Что выведет?
Go 
fmt.Print("A")
fmt.Print("B")
fmt.Println("C")
fmt.Print("D")
Показать решение

Вывод: ABC на первой строке, D на второй

Print("A") и Print("B") - курсор остается. Println("C") - выводит C и переводит строку. Print("D") - выводит D на новой строке.

02 Что выведет?
Go 
fmt.Println("row1")
fmt.Print("row2")
fmt.Println("row3")
fmt.Print("row4")
Показать решение

Вывод:

row1

row2row3

row4

Println("row1") переводит строку. Print("row2") оставляет курсор, Println("row3") выводит рядом и переводит строку.

03 Что выведет?
Go 
// fmt.Println("Java")
fmt.Println("Go") // fmt.Println("C++")
fmt.Println("Python") // комментарий
// fmt.Println("Ruby")
Показать решение

Вывод:

Go

Python

Строки с Java и Ruby закомментированы. Комментарии после кода на той же строке игнорируются.

04 Что выведет?
Go 
fmt.Print("row1")
fmt.Println("row2")
fmt.Println()
fmt.Print("row3")
fmt.Println("row4")
Показать решение

Вывод:

row1row2

(пустая строка)

row3row4

Print и Println рядом склеиваются. Println() без аргументов создает пустую строку.

05 Напиши программу

Напиши программу, которая выводит:

Output 
Hello,
World!
Показать решение
Go 
package main

import "fmt"

func main() {
    fmt.Println("Hello,")
    fmt.Println("World!")
}

Используем Println дважды, чтобы каждая строка выводилась на отдельной строке.

РЕЗЮМЕ

Key Ключевые идеи
  • Любая Go-программа начинается с package main и функции main()
  • import "fmt" подключает пакет для вывода текста
  • fmt.Println() выводит текст и переводит строку
  • fmt.Print() выводит текст без перевода строки
  • Комментарии начинаются с // и игнорируются компилятором
  • Go регистрозависим - Println и println это разные имена

Правило: Нужен перевод строки - используй Println. Нужно склеить - используй Print.
